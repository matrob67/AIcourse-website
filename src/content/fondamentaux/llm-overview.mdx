## La formule simple qui marche: prédire le prochain mot

Un **Large Language Model** (LLM) est un réseau de neurones — presque toujours un Transformer — entraîné sur **une seule tâche simple**:

**Prédire le prochain token connaissant tous les précédents.**

```
Donnée: "Le chat noir"
Objectif: Prédire ce qui vient après
Réponse: "dort", "saute", "miaule" (avec probabilités)
```

Ce qui le rend "Large"?
- **Milliards de paramètres**: GPT-1 (117M) < GPT-3 (175B) < PaLM (540B) < Llama 3.1 (405B)
- **Trillions de tokens d'entraînement**: corpus gigantesque de texte web, livres, code, articles scientifiques
- **Compute massif**: des milliers de GPU pendant des mois

### Le miracle: une tâche banale crée l'intelligence

La tâche est **triviale**. Prédire le prochain mot. Juste ça.

Et pourtant... à échelle suffisante, émergent:
- Connaissance du monde (savoir que Paris est capitale de France)
- Logique mathématique (résoudre des équations)
- Raisonnement multi-étapes (décomposer des problèmes)
- Code (générer du Python fonctionnel)
- Traduction (sans données d'entraînement de traduction!)

Aucune de ces capacités n'est "programmée". Elles **émergent** simplement de la tâche de prédiction.



<Callout type="tip" title="Tokenization: pas des mots, des morceaux">
Les LLM ne traitent pas "mots entiers". Ils découpent en **tokens** (morceaux de mots ou sous-mots):

```
"extraordinaire" → tokens ["extra", "ordinaire"] ou ["ext", "raord", "inaire"]
"le" → token ["le"]
"!" → token ["!"]

Ratio moyen: 1.3 tokens par mot en anglais
```

Algorithmes courants: BPE (Byte-Pair Encoding), WordPiece, SentencePiece.
</Callout>

## Les trois leviers du progrès: le scaling au-delà des limites

L'histoire entière des LLM (2018-2024) c'est: **augmenter trois variables et voir le modèle devenir plus intelligent**. C'est pas compliqué, mais ça marche à chaque fois.

### Levier 1: Paramètres (taille du réseau)

```
GPT-1   (2018): 117 millions
GPT-2   (2019): 1.5 milliards
GPT-3   (2020): 175 milliards
PaLM    (2022): 540 milliards
Llama 3.1 (2024): 405 milliards
```

**Observation**: plus gros = plus intelligent (avec du scaling up et du temps).

### Levier 2: Données (volume de texte d'entraînement)

```
GPT-1:   1 milliard de tokens
GPT-3:   570 milliards de tokens (et beaucoup plus en dedup)
Llama 3.1: 15 trillions de tokens
```

**Observation**: plus de texte varié = mieux apprendre les patterns.

### Levier 3: Compute (puissance GPU)

```
GPT-3:     3.6 × 10^23 opérations = $4.6M en GPU
PaLM:      ~10x plus d'opérations
Llama 3:   estimé $100M+ en compute
```

**Observation**: plus de GPU-jours = meilleur entraînement.

### Le secret: c'est prévisible

Kaplan et al. (OpenAI, 2020) découvrent que **la performance suit une courbe mathématique régulière**:

```
Performance ∝ (Parametres × Data × Compute)^(exponent négatif)
```

En log-log, c'est une droite. Pas de surprises, pas de seuils magiques. Juste une loi de puissance.

**Conséquence**: tu peux planifier. "Je veux performance de 85%. Combien de GPU me faut-il?" → formule mathématique.

Ça a changé tout. Avant: "Innove et hope". Après: "Scale et win".

<Callout type="info" title="Le débat: peut on scalerer infiniment?">
Les lois de scaling décrissent la perte de pré-entraînement (métrique statistique). Mais certaines tâches peuvent avoir des "plataux": amélioration rapide jusqu'à 85%, puis flat. Débat ouvert en 2026.
</Callout>

## Les capacités "émergentes": le puzzle du scaling

À certains seuils de taille, des capacités complètement **nouvelles** apparaissent. Pas d'entraînement spécialisé, juste: "Le modèle est plus grand et soudain il peut faire X."

Exemples:

### In-context learning
GPT-2 ne peut pas faire ça. GPT-3 (175B)? Donne 3 exemples dans le prompt et ça marche.

```
Few-shot: "Sentiment: 'J'adore!' -> positif
         Sentiment: 'Bof...' -> neutre
         Sentiment: 'Excellent!' -> ?"
GPT-3 répond correctement ("positif") sans fine-tuning.
```

### Chain-of-thought
GPT-2 donne une réponse. GPT-3 + instruction "Let's think step by step"?

```
Question: Si j'ai 3 pommes et mon ami m'en donne 5, puis je partage la moitié avec ma mère...
GPT-3 sort: "Étape 1: 3 + 5 = 8. Étape 2: 8 / 2 = 4. Réponse: 4 pommes"
Raisonnement explicite! Et ça aide la performance.
```

### Code generation
Pas entraîné sur du code différent des LLM génériques. GPT-3.5?

```
Prompt: "Écris une fonction Python qui vérifie si un nombre est premier"
Sortie: Code correct, effiace, commenté.
```

### Traduction zero-shot
Jamais entraîné sur chinois-vers-français. GPT-3 le fait.

```
"Traduis du chinois au français: 你好 ->"
GPT-3: "Bonjour"
```

### Le débat philosophique

**Question**: ces capacités **émergent-elles vraiment** (comportement radicalement nouveau), ou **se révèlent-elles** (étaient latentes, deviennent visibles avec le scaling)?

**Débat actif en 2026** :
- Côté "émergence": certaines tâches passent de 0% à 80% d'un coup quand tu doubles la taille
- Côté "révélation": peut-être que c'est juste une question de métrique. Rendre visible ≠ émerger

Nous ne savons pas. C'est fascinant et débattu.

<Callout type="warning" title="Les capacités sont parfois instables">
GPT-3 peut faire de la traduction. GPT-4 beaucoup mieux. Mais "faire" peut signifier "parfois juste", pas "toujours". Les capacités émergentes peuvent être fragiles.
</Callout>

## Comment un LLM génère du texte (la mécanique concrète)

Étape par étape, quand tu poses une question à un LLM:

### 1. Tokenization
```
Entrée: "Explique la gravité"
Tokens: ["Expl", "ique", "la", "grav", "ité"]
```

Le tokenizer découpe en morceaux gérables (pas des mots entiers, des sub-unités).

### 2. Embedding
```
Token "Expl" → vecteur numérique [0.2, -0.5, 0.8, ...] (2048 dimensions)
Token "ique" → vecteur numérique [0.1, 0.3, -0.2, ...]
...
```

Chaque token devient un vecteur dense dans un espace continu.



### 3. Passage dans le Transformer
```
Couche 1: self-attention + MLP
Couche 2: self-attention + MLP
...
Couche 32: self-attention + MLP (pour GPT-3.5)

Output: vecteur mis à jour pour chaque token
```

À chaque couche, chaque token "parle" aux autres tokens via l'attention. Le réseau affine les représentations.

### 4. Prédiction du prochain token
```
Vecteur du dernier token → couche linéaire → 50,000 scores
(50,000 = taille du vocabulaire)

softmax() → probabilités qui somment à 1

Résultat: P("la") = 0.6, P("dans") = 0.2, P("entre") = 0.1, ...
```

### 5. Sampling
Tu **tires un token** selon cette distribution de probabilité.

```
Température = 0:    Toujours le token le plus probable ("la")
Température = 1:    Tire selon les probas exactes
Température = 2:    Moins déterministe, plus de créativité

Température = 0 → réponses reproductibles
Température = 1+ → réponses variées et créatives
```

### 6. Auto-régression (le loop)
```
Prompt initial: "Explique la gravité"

Itération 1:
  Entrée: "Explique la gravité"
  Sortie: "C'est"
  Nouveau prompt: "Explique la gravité C'est"

Itération 2:
  Entrée: "Explique la gravité C'est"
  Sortie: "une"
  Nouveau prompt: "Explique la gravité C'est une"

Itération 3: "Explique la gravité C'est une force"
...
(continue jusqu'à </s> ou max_tokens)
```

Chaque token généré devient input pour la prédiction suivante. C'est **auto-régressif** — auto = self, régress = regress/reference à.

<Callout type="tip" title="Prédiction != compréhension">
Techniquement, un LLM ne "comprend" rien. Il:
1. Transforme des vecteurs via matrices de poids
2. Produit une distribution de probabilité
3. Tire un token

Est-ce de la compréhension? Philosophiquement ouvert. Pratiquement: ça marche.
</Callout>

## Les limites fortes

- **Hallucinations**: génère des faits faux avec confiance
- **Fenêtre de contexte**: limite de tokens qu'il peut traiter à la fois (s'améliore)
- **Pas de mémoire persistante**: chaque conversation repart de zéro
- **Raisonnement fragile**: échoue sur variations mineures
- **Coût computationnel**: entraînement énorme, inférence aussi

## Pourquoi les LLM gagnent

- **Architecture**: le Transformer scale bien
- **Données**: texte abondant sur le web
- **Compute**: GPU massifs disponibles
- **Flexibilité**: un modèle pour 100 tâches (via prompting)

## Pourquoi c'est important

Un LLM = un réseau entraîné sur **une tâche triviale** (prédire le prochain token) à **une échelle folle** (milliards de paramètres, trillions de tokens).

Ce simple combo produit un outil d'une **flexibilité remarquable** qui peut:
- Répondre à des questions
- Générer du code
- Traduire
- Résumer
- Débattre
- Et bien d'autres (presque tout qui s'énonce comme "générer du texte")

Les trois axes du scaling (paramètres, données, compute) gouvernent directement le progrès. Pas de breakthrough magique. Juste: plus gros = plus intelligent.

Les capacités émergentes (in-context learning, chain-of-thought, code) révèlent que les patterns du langage humain contiennent beaucoup plus que nous ne le pensions — logique, raisonnement, connaissance du monde — tout encodé implicitement dans les statistiques texte.
