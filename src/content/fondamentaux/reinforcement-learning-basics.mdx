## Introduction : Apprendre par essai-erreur et récompense

Ton enfant veut apprendre à marcher. Personne ne lui code les muscles. Il essaie, tombe (ouch, négatif), se relève, réussit un pas (oui, positif !). Petit à petit, il apprend la séquence de muscles qu'il faut contracter.

C'est le **Reinforcement Learning**. Un agent interagit avec un environnement. Il essaie une action. L'environnement dit "bien joué, voilà une récompense" ou "raté, pénalité". Après des milliers de tentatives, l'agent trouve la stratégie qui maximise la récompense cumulée.



<Callout type="info" title="C'est pas supervisé, c'est feedback réel">
Supervisé : "Voilà la bonne réponse." RL : "Voilà une récompense, débrouille-toi à apprendre d'où elle vient."
</Callout>

## Les acteurs du RL

### Agent

L'entité qui apprend. C'est le "cerveau" qu'on entraîne. Peut être un robot, une IA qui joue aux échecs, un chatbot.

### Environnement

Tout ce avec quoi l'agent interagit. Réagit aux actions. Détermine les récompenses.

### La boucle d'interaction

```
1. Agent observe l'état (position, situation)
2. Agent choisit une action
3. Environnement change et produit une récompense
4. Retour à 1
```

Imagine AlphaGo : agent = l'IA, environnement = l'échiquier. AlphaGo joue un coup. L'environnement valide si c'est légal et produit une récompense (meilleure position ? victoire éventuelle ?)

## Les concepts clés



### État (State)

La description actuelle de la situation.

Exemples :
- Jeu d'échecs : position de toutes les pièces
- Robot qui marche : positions et vitesses de ses articulations
- Jeu vidéo : pixels de l'écran, ou position du joueur + ennemis + objets

### Action (Action)

Ce que l'agent peut faire.

**Discret** : aller gauche, droite, sauter, tirer (jeux vidéo)

**Continu** : appliquer une force continue sur un moteur (robotique)

### Récompense (Reward)

Feedback immédiat de l'environnement, un nombre.

Exemples :
- Jeu d'échecs : +1 pour une victoire, -1 pour une défaite, 0 sinon
- Robot marcheur : +1 pour chaque mètre avancé, -100 pour une chute
- Pac-Man : +10 pour manger une pastille, -10 pour se faire attraper

<Callout type="warning" title="Concevoir la récompense est un art">
Une mauvaise récompense = agent bizarre. Exemple classique : agent chargé "de ramasser des objets". Il apprend à tourner en cercle à côté du point de réapparition au lieu d'explorer (reward hacking). Il maximise la récompense, mais pas ce qu'on voulait vraiment.
</Callout>

### Politique (Policy)

La stratégie du l'agent. Une fonction qui dit "dans cet état, fais cette action".

**Déterministe** : même état = même action toujours.

**Stochastique** : même état = probabilité sur différentes actions.

L'objectif du RL : trouver la **politique optimale** qui maximise la récompense à long terme.

### Fonction de Valeur (Value Function)

Répond : "Combien cet état vaut-il à long terme ?"

```
V(état) = récompense immédiate + (facteur d'actualisation) * récompense future
```

Le **facteur d'actualisation** (gamma, entre 0 et 1) contrôle l'importance du futur.
- Gamma proche de 0 : l'agent est myope, focus sur récompense immédiate
- Gamma proche de 1 : l'agent planifie à long terme

### Fonction Q (Q-Function)

Variante de la valeur. Répond : "Si je prends cette action dans cet état, quelle récompense j'attends ?"

```
Q(état, action) = récompense de cette action + récompense future optimale
```

Si on connaît la Q optimale, la politique optimale est simple : chaque état, prendre l'action avec la plus grande Q.

## Exploration vs Exploitation : le dilemme fondamental

Tu connais un bon restaurant (exploitation). Mais peut-être qu'il en existe un meilleur à côté (exploration). Si tu ne fais qu'exploiter, tu ne découvres jamais le meilleur. Si tu ne fais qu'explorer, tu ne profites jamais de tes découvertes.

C'est le compromis central du RL.

**Exploitation** : fais ce que tu sais être bon

**Exploration** : essaie de nouvelles choses pour découvrir potentiellement mieux

Stratégies :

**Epsilon-greedy** : avec probabilité epsilon (disons 10%), choisis une action aléatoire (exploration). Sinon, choisis la meilleure action connue (exploitation). On réduit epsilon au fil du temps.

**Softmax** : choisis les actions proportionnellement à leur valeur Q estimée. Bonnes actions = plus probables, mais les autres ont aussi une chance.

**UCB (Upper Confidence Bound)** : équilibre entre valeur et incertitude. Favorise les actions mal connues.

## Q-Learning : l'algo fondateur

L'un des algorithmes de RL les plus célèbres. Apprend la fonction Q optimale sans modèle de l'environnement.

Fonctionnement :

1. Observe l'état actuel s
2. Choisis action a (epsilon-greedy par exemple)
3. Environnement donne récompense r et nouvel état s'
4. Mets à jour Q :

```
Q(s, a) = Q(s, a) + alpha * [r + gamma * max Q(s', a') - Q(s, a)]
```

Où alpha c'est le taux d'apprentissage.

La mise à jour pousse Q vers la valeur cible "récompense immédiate + meilleure valeur future".

### Deep Q-Network (DQN) : quand les espaces sont énormes

Quand l'espace d'états est infini (les pixels d'un jeu Atari = des milliards d'états), une table Q ne marche pas.

Solution : remplace la table Q par un **réseau de neurones** qui approxime Q(s, a).

DQN a permis à une IA d'apprendre des jeux Atari au niveau surhumain. Deux innovations clés :

**Experience Replay** : stocke les transitions (s, a, r, s') dans un buffer. Ré-échantillonne aléatoirement pour l'entraînement. Ça brise les corrélations temporelles.

**Target Network** : utilise une copie figée du réseau pour calculer la cible. Stabilise l'apprentissage.

## Policy Gradient : optimiser la politique directement

Au lieu d'apprendre Q et d'en déduire la politique, optimise **directement la politique**.

Théorème de policy gradient :

```
gradient = E[gradient log(probabilité action|état) * Q(état, action)]
```

En français : augmente la probabilité des actions à forte valeur Q.

Variations :

**REINFORCE** : le plus simple, très basique.

**Actor-Critic** : utilise deux réseaux. L'acteur = la politique. Le critique = la fonction de valeur. L'un apprend à agir mieux, l'autre apprend à évaluer.

**PPO (Proximal Policy Optimization)** : la plus populaire maintenant. Stable, robuste. Utilisée pour RLHF (alignment de ChatGPT).

## Applications spectaculaires

### Jeux

**AlphaGo** (2016) : battait le champion du monde de Go, humainement considéré impossible. Combinaison de réseaux de neurones + recherche + RL.

**AlphaZero** (2017) : apprend les échecs à partir de zéro en quelques heures, bat tous les engines classiques.

**OpenAI Five** (2019) : bat les meilleurs joueurs de Dota 2 (jeu complexe, coopératif).

### Robotique

Entraîne des robots à marcher, saisir, naviguer. **Sim-to-real transfer** : entraîne en simulation (fast, safe), puis transfert sur un vrai robot.

### RLHF : aligner les LLMs avec les humains

L'une des applications les plus impactantes. ChatGPT, Claude, Gemini l'utilisent.

<ArxivLink id="2203.02155" title="Training language models to follow instructions with human feedback (InstructGPT)" />

Processus :

1. **Pré-entraînement supervisé** : LLM entraîné classiquement sur du texte (déjà utile)

2. **Modèle de récompense** : humains comparent paires de réponses, indiquent laquelle est meilleure. Entraîne un modèle à prédire ces préférences.

3. **Optimisation PPO** : utilise PPO pour optimiser le LLM. Maximise le score du modèle de récompense, tout en restant proche du modèle original (évite le "reward hacking").

Résultat : ChatGPT capable de suivre des instructions, être utile, éviter les contenus nuisibles.

<Callout type="info" title="DPO : alternative moderne au RLHF">
Le Direct Preference Optimization simplifie le processus. Pas besoin de modèle de récompense explicite ni PPO. Juste un problème de classification direct sur les paires de préférences. Plus simple, souvent aussi bon.
</Callout>

## Les défis du RL

**Sample efficiency** : le RL demande souvent des millions d'interactions. En simulation c'est rapide. Avec un vrai robot ? Très coûteux.

**Stabilité** : l'entraînement par RL est notoirement instable. Un léger changement d'hyperparamètre = effondrement.

**Reward design** : concevoir la bonne récompense est très difficile. Facile de se faire tromper par l'agent qui trouve des failles.

**Exploration vs exploitation** : trouver le bon équilibre demande de l'expérience.

**Généralisation** : un agent entraîné sur un environnement peut échouer si l'environnement change légèrement.

<Callout type="warning" title="Le problème d'alignement avec les LLMs">
Si tu optimises un LLM pour plaire à un modèle de récompense imparfait, il peut le "hacker". Génère du texte qu'un humain n'aimerait pas réellement mais qui score bien sur la récompense. C'est un défi central de la sécurité de l'IA.
</Callout>

## Pourquoi c'est important

Le RL est comment les systèmes modernes apprennent de l'interaction réelle. AlphaGo, ChatGPT, les voitures autonomes, les robots. Partout où tu dois apprendre à partir du feedback du monde réel (pas juste du texte statique), c'est le RL qui domaine. C'est aussi le plus difficile, mais le plus puissant.
