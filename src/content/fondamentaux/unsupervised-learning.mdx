## Introduction : Apprendre sans prof

Imagine : tu arrives dans une ville inconnue sans map, sans guide. Personne ne te dit "c'est le quartier riche" ou "c'est la gare". Mais tu explores et tu remarques naturellement : "Ces bâtiments sont tous neufs et grands, les gens habillés cher. C'est riche ici." "Ici c'est vieux, bruit, odeur... c'est le vieux port."

C'est l'apprentissage non-supervisé. Pas d'étiquettes. Pas de prof. Juste les données brutes. Tu dois découvrir la structure par toi-même.



<Callout type="info" title="Pourquoi c'est important ?">
99% des données du monde ne sont pas étiquetées. Étiqueter coûte cher : il faut des humains, du temps, de l'expertise. L'apprentissage non-supervisé te laisse exploiter des millions de données brutes sans jamais payer un humain pour les labéliser.
</Callout>

## Clustering : regrouper les similaires

**Clustering** = partitionner les données en groupes où chaque groupe contient des choses similaires.

Exemple : un site e-commerce a 1 million de clients. Aucune étiquette. Mais le clustering découvre :
- Groupe A : achètent souvent, petits montants, fidèles (les acheteurs fréquents)
- Groupe B : achètent rarement, énormes montants (les acheteurs rares mais dépensiers)
- Groupe C : juste des visiteurs, 95% ne payent jamais (les window shoppers)

Chaque groupe reçoit une stratégie marketing différente. Pas besoin de les labéliser à la main.

### K-Means : le classique facile

L'algo de clustering le plus populaire. Fonctionnement :

1. Décide le nombre de clusters K (2, 3, 5, peu importe).
2. Place K points aléatoires (les "centres" ou centroïdes).
3. Assigne chaque point au centre le plus proche.
4. Recalcule les centres comme la moyenne des points de chaque groupe.
5. Répète 3-4 jusqu'à que rien ne change (convergence).

```
Les centroïdes bougent vers les "cœurs de gravité" de chaque groupe.
```

**Forces** :
- Super rapide (complexité linéaire)
- Simple à comprendre et coder
- Marche bien quand les clusters sont à peu près sphériques

**Limites** :
- Il faut choisir K à l'avance (qu'est-ce que c'est le "bon" nombre de clusters ?)
- Sensible à l'initialisation (on peut se bloquer dans un mauvais plateau)
- Suppose des clusters de forme similaire (sphériques)

<Callout type="tip" title="Choisir K avec la méthode du coude">
Trace un graphique : X = nombre de clusters K, Y = inertie (distance totale). La courbe descent abruptement puis s'aplatit. Le "coude" (la transition) est le K optimal. C'est visuellement une courbe qui ressemble à un bras plié.
</Callout>

### DBSCAN : trouver les vraies structures

Approche complètement différente : au lieu de chercher des centre, on cherche les régions **denses**.

Principe :
- Un point qui a beaucoup de voisins proches (ex: au moins 5 points dans un rayon de 0.5) c'est un "point noyau"
- Connecte les points noyaux qui sont proches -> c'est un cluster
- Les points isolés (pas assez de voisins) -> c'est du bruit/anomalies

**Forces** :
- Pas besoin de spécifier K
- Détecte les clusters de **n'importe quelle forme** (pas juste sphériques)
- Détecte automatiquement les anomalies (les points isolés)

**Limites** :
- Problématique si les clusters ont des densités très différentes
- Sensible aux paramètres epsilon (rayon) et minPts (nombre minimum de voisins)

### Clustering hiérarchique : l'arbre des groupes

Construis une hiérarchie représentée en "dendrogramme" (un arbre).

**Agglomérative (bottom-up)** : chaque point commence seul. Fusionne itérativement les deux clusters les plus proches.

**Divisive (top-down)** : tout commence ensemble. Divise itérativement.

**Avantage** : tu vois la structure à toutes les échelles. Tu peux couper l'arbre n'importe où pour obtenir le nombre de clusters que tu veux.

## Réduction de dimensionnalité : voir le monde en 2D

Tes données ont 1000 dimensions ? 10 000 features ? C'est trop pour visualiser ou même pour calculer efficacement.

La réduction de dimension projette les données dans un espace plus petit (2D ou 3D généralement) en **préservant l'information importante**.

### Pourquoi réduire ?

**Visualisation** : impossible de voir 1000 dimensions. Réduire à 2D te laisse explorer les données visuellement.

**Accélération** : moins de dimensions = algorithmes plus rapides.

**Débruitage** : les dimensions les moins importantes sont souvent du bruit pur. Les supprimer aide.

**La malédiction de la dimensionnalité** : en très haute dimension, tous les points deviennent très éloignés les uns des autres. K-Means, KNN, etc. deviennent inutiles.

### PCA : suivre la variance maximale

**Principal Component Analysis** : cherche les directions où tes données varient le plus.

Imagine : 10 000 points dans l'espace 3D. Mais ils sont tous alignés le long d'une ligne. La vraie dimension est 1, pas 3. PCA trouve cette ligne.

Comment ça marche :
1. Normalise les données (soustrais la moyenne)
2. Calcule la matrice de covariance (comment chaque feature varie avec les autres)
3. Trouve les "axes principaux" (les directions de variance maximale)
4. Projette sur les 2-3 premiers axes

**Forces** : rapide, déterministe, mathématiquement clair.

**Limites** : capture que les relations linéaires.

### t-SNE : pour les belles visualisations

**t-distributed Stochastic Neighbor Embedding** : conçu **pour la visualisation seulement**, pas pour le preprocessing.

Idée : préserve les **structures locales** (les points proches restent proches).

Résultat : des visualisations très lisibles où on voit clairement les clusters. Magique.

**Mais attention** :
- C'est lent (O(n²))
- Non-déterministe (chaque exécution est différente)
- N'utilise jamais t-SNE comme input d'un algorithme ML (ça change ses propriétés). Juste pour regarder.



### UMAP : t-SNE mais mieux

**Uniform Manifold Approximation and Projection** : alternative plus récente et meilleure.

Avantages :
- 10-100x plus rapide que t-SNE
- Préserve mieux la structure globale
- Scalable à des millions de points
- Peut être utilisé même en preprocessing (pas juste visualisation)

Maintenant préféré à t-SNE pour explorer les grandes données.

## Détection d'anomalies

Identifier les observations bizarres qui dérogent à la normale.

Exemples d'utilisation :
- Cybersécurité : détecte les intrusions
- Finance : détecte les fraudes
- Maintenance : détecte les machines près de la panne
- Santé : détecte les cas rares/pathologies
- Monitoring : détecte les serveurs qui meurent

Techniques courantes :

**Isolation Forest** : construit des arbres aléatoires. Mesure combien de coupures sont nécessaires pour isoler un point. Les anomalies s'isolent plus vite (moins de coupures).

**Autoencoders** : réseau de neurones qui compresse puis reconstruit les données. Entraîné que sur données normales. Une anomalie (jamais vue) sera mal reconstruite (erreur de reconstruction élevée).

**One-Class SVM** : apprend une frontière autour des données normales. Ce qui est dehors = anomalie.

## Cas d'usage concrets

**Segmentation client** : E-commerce groupe ses 10M de clients en 5-10 segments. Chaque segment reçoit une campagne marketing personnalisée. ROI augmente de 40%.

**Exploration de données** : Avant de construire un modèle supervisé, utilise PCA ou UMAP pour visualiser. Ça t'aide à voir : les classes se chevauchent-elles ? Y a-t-il des outliers bizarres ? Mes features sont-elles utiles ?

**Compression** : PCA réduit les images 1000x1000 à 50 dimensions. Ça occupe 1000x moins d'espace. Puis tu peux reconstruire l'image (95% de qualité).

**Systèmes de recommandation** : Factorisation de matrices découvre les préférences cachées. Netflix groupe les films en "caractéristiques latentes" (action-level, romance-level, etc.). Puis recommande films similaires.

## Supervisé vs Non-supervisé : le tableau

| Aspect | Supervisé | Non-supervisé |
|---|---|---|
| Données | Étiquetées (X, y) | Brutes (X seulement) |
| Objectif | Prédire y | Découvrir structure |
| Évaluation | Claires (accuracy, MSE) | Subjectives, visuelles |
| Exemples | Classification, régression | Clustering, réduction |
| Défi principal | Obtenir les labels | Interpréter les résultats |

## Pourquoi c'est important

L'apprentissage non-supervisé t'ouvre à l'analyse des données brutes du monde réel. C'est comment Netflix trouve que tu aimes Sci-Fi. Comment les banques détectent les fraudes. Comment la science découvre des structures cachées dans les données. Presque aucune entreprise ne peut étiqueter tous les données. Non-supervisé c'est la clé.
