## Ministral : l'IA qui tient dans votre poche

Vous avez besoin d'IA, mais pas sur le cloud. Un robot qui doit penser en local. Un vieux laptop. Un téléphone. Ministral, c'est pour vous.

La famille Ministral (3B, 8B, 14B paramètres) est spécifiquement optimisée pour tourner sur du **matériel léger** : NVIDIA Jetson, GPU RTX grand public, même des CPUs puissants.

### Pourquoi l'edge ?

Cloud n'est pas toujours l'option :

**Latence** : un véhicule autonome qui attend 500ms de réponse cloud, c'est un accident. L'IA doit être en local.

**Confidentialité** : les données sensibles ne quittent pas votre réseau. Jamais.

**Pas d'internet** : l'usine n'a pas de fibre. L'IA embarquée marche hors ligne.

**Coût** : déployer de l'IA sur 10 000 appareils via cloud = millions par an. En local = amortissement du matériel en 2 ans.

<Callout type="info" title="L'explosion du edge computing">
L'IA embarquée c'est un marché énorme : robots, véhicules, usines connectées, appareils IoT. Ministral est positionné pour le capturer.
</Callout>

### Ministral 3B : le minimalist

**3 milliards de paramètres** = petite taille.

En quantification INT4 (compression intelligent), ça prend ~1,5-2 Go de RAM. Fonctionne sur :
- Jetson Nano
- Téléphones haut de gamme
- Vieilles machines

**Cas d'usage** : classification de texte, extraction simple, genération courte, assistants vocaux embarqués.

**Limitation** : pas de raisonnement complexe, pas de long-form generation, pas de multilingue avancé.

### Ministral 8B : le sweet spot

**8 milliards de paramètres** = l'équilibre parfait.

Tourne sur un RTX 4070 seul. Performances similaires aux petits modèles cloud, mais en local.

**Cas d'usage** : chatbot embarqué, analyse de documents locaux, résumé, traduction, aide à la rédaction.

**Bonus** : les développeurs adorent pour l'itération locale rapide.

<Callout type="tip" title="Le préféré des devs">
Pouvoir prototyper et tester sur sa machine dev (sans dépendre du cloud) = cycle itératif rapide. Ministral 8B est devenu le favori des développeurs IA.
</Callout>

### Ministral 14B : le pointy

**14 milliards de paramètres** = la limite haute du edge.

Capacités proches de Small cloud. Un RTX 4090 ou un bon serveur = easy.

**Cas d'usage** : déploiements on-premise, serveurs d'infra légers, stations de travail IA.

### Denses, pas MoE

Contrairement aux grands modèles Mistral (qui utilisent MoE, plus efficace), les Ministral sont **denses** (tous les paramètres participent toujours).

Pourquoi ?

**Simplicité d'inférence** : pas de routage complexe. Le matériel edge n'aime pas les tricks.

**Prévisibilité** : la RAM et le CPU utilisés sont garantis. Pas de surprises.

**Efficacité locale** : en dessous de 15B paramètres, les modèles denses sont souvent plus rapides que MoE.

### Optimisés NVIDIA

Mistral travaille étroitement avec NVIDIA.

**Jetson** : profils d'optimisation TensorRT prêts à l'emploi.

**RTX** : exploite pleinement les Tensor Cores. Gains massivement plus rapides qu'avec PyTorch pur.

**TensorRT** : compilez vos modèles pour la perf max.

### Formats de distribution

| Format | Usage | Avantage |
|--------|-------|----------|
| Hugging Face (safetensors) | Dev + fine-tuning | Écosystème HF |
| GGUF | CPU/GPU via llama.cpp | Quantification intégrée |
| TensorRT | GPU NVIDIA | Performance max |
| ONNX | Multi-plateforme | Portabilité |

Pick le format qui match votre use case.

<Callout type="warning" title="Attentes réalistes">
3B/8B != GPT-4. Ne vous attendez pas aux mêmes capacités. Mais pour du travail edge spécifique, c'est excellent.
</Callout>

Une fois que le hardware est payé, les coûts de token approchent zéro.

Pour des déploiements massifs (10 000 robots, chaîne de production entière), l'économie vs. cloud est **colossale**.
