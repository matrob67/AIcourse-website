## Mistral Large 3 : le flagship

**Mistral Large 3** = le meilleur modèle de Mistral. Lancé en 2025. **675 milliards de paramètres totaux**, mais seulement **41 milliards actifs** par token (grâce à MoE).

Contexte de **256K tokens**. Multimodal (texte + images). Classé #2 sur LMArena (le classement humain le plus réputé) parmi les modèles ouverts.

C'est le modèle qui représente tout ce que Mistral a appris.

### Architecture

| Paramètre | Valeur |
|-----------|--------|
| Totaux | 675B |
| Actifs par token | 41B |
| Architecture | MoE |
| Contexte max | 256 000 tokens |
| Multimodal | Oui (texte + image) |
| GPU entraînement | NVIDIA Hopper (H100/H200) |

La ratio totals/actifs = 16:1. Compare à Mixtral 8x7B (3,6:1). Beaucoup plus d'experts, routage plus ciblé.

<Callout type="info" title="Le pouvoir du MoE à échelle massive">
675B params totaux = capacité de connaissance énorme (comme une biblio de 675B livres).
41B actifs = coût de calcul lean (consulte seulement 41B livres par question).
C'est l'avantage du MoE : capacité sans le coût computationnnel full.
</Callout>

### Contexte : 256K tokens

**256K tokens** = ~200 000 mots = **2-3 livres complets**.

Un projet logiciel entier (des dizaines de fichiers). La doc complète de RGPD. Un rapport financier de 500 pages. Tout en contexte.

Possible grâce à : Sliding Window Attention, attention hiérarchique, compression KV cache, entraînement sur long sequences.

### Multimodal

Mistral Large 3 comprend les **images**, pas juste le texte.

- Analyser des photos
- Extraire du texte (OCR)
- Interpréter des graphiques
- Résoudre des maths visuels

**Magic** : tout en gardant les perfs texte au top. Beaucoup de modèles multimodaux régressent sur le texte.

### Performances

**LMArena** : classé #2 parmi open-weights. Jugement humain en aveugle.

**Benchmarks** : MMLU, HumanEval, MATH = compétitif vs GPT-4o et Claude 3.5 Sonnet. L'écart open vs proprietary = closing.

### Entraînement : NVIDIA Hopper

Entraîné sur GPU H100/H200 avec HBM3e (4,8 TB/s de bandwidth).

Pour 675B paramètres, faut du parallélisme massif : data parallelism + tensor parallelism + pipeline parallelism + expert parallelism (spécifique au MoE).

<Callout type="paper" title="Coût d'entraînement">
Estimé 50-150M$ en GPU hours. Chiffre non-official, community estimates.
</Callout>

### Cas d'usage

**Docs complexes** : avec 256K contexte + multimodal, analyser des rapports financiers complets avec graphiques et tableaux en une shot.

**Raisonnement** : tâches multi-étapes, synthèse d'infos complexes.

**Agents** : function calling avancé + long context = agents sophistiqués.

**Multilingue** : excelle dans beaucoup de langues, déploiements internationaux.

### Disponibilité

- **La Plateforme** : accès API
- **Le Chat** : interface grand public
- **Clouds** : Azure, AWS Bedrock, GCP
- **Open-weight** : sur Hugging Face pour self-hosted (mais faut du hardware sérieux)

<Callout type="warning" title="Déploiement self-hosted">
675B parameters = infrastructure massive. Même quantifié, faut un cluster de GPUs haut de gamme. Pour la plupart, l'API c'est plus pratique.
</Callout>
