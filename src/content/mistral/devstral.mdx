## Devstral : de l'IA copilote à l'IA agent développeur

De Codestral à Devstral, il y a un bond : passer d'un modèle qui complète du code ligne par ligne à un modèle qui **comprend un projet entier** et peut le refactoriser d'un coup.

Imaginez : au lieu d'un copilote qui murmure à votre oreille, c'est un junior developer qui peut lire tout le codebase et proposer des améliorations architecturales.



### Ce qui change

**Compréhension structurelle** : Devstral voit la hiérarchie des fichiers, les dépendances, les patterns de design, les conventions du projet.

**Refactoring multi-fichier** : modifier 5 fichiers en une passe pour corriger une architecture brisée, c'est faisable.

**Débogage contextuel** : vous donnez une stack trace, Devstral navigue le code complet pour trouver la root cause.

<Callout type="info" title="Le saut qualitatif">
Codestral = « complète ma fonction »
Devstral = « comprends mon projet et améliore-le »
C'est le saut de l'autocomplétion (niveau fonction) à l'agentique (niveau architecture).
</Callout>

### Le résultat ? Devstral 2 surpasse Qwen 3 Coder Flash 30B

Qwen a 6 milliards de paramètres de **plus**. Mais sur **SWE-bench** (le benchmark qui compte vraiment — des vrais problèmes GitHub), Devstral 2 gagne.

SWE-bench n'évalue pas des fonctions isolées. C'est du vrai travail de développeur : lire un projet, identifier un bug, le corriger. Devstral 2 le fait mieux avec 24B qu'Alibaba avec 30B.

<Callout type="tip" title="SWE-bench : le test du réel">
HumanEval teste des functions en isolation. SWE-bench teste des bugs réels dans des projets réels. C'est le benchmarking qui fait mal mais qui compte.
</Callout>

### Devstral Small 2

Pour les développeurs qui veulent rapidité et efficacité en local.

**Latence quasi-instantanée** : l'autocomplétion doit être invisible. Small 2 y arrive.

**Déploiement local** : un seul GPU grand public (RTX 4070) suffit. Pas besoin du cloud.

**Coût minimal** : pour les entreprises, chaque token coûte 3-4x moins cher qu'avec Devstral 2.

### Licence MIT modifiée

Attention : différente de la licence Apache 2.0 des modèles Mistral précédents.

**Autorisé** : recherche, développement, évaluation
**À vérifier** : les restrictions commerciales peuvent différer de l'Apache 2.0

<Callout type="warning" title="Avant de commercialiser">
Si vous voulez vendre un produit basé sur Devstral, lisez les termes exacts de la licence MIT modifiée. Mistral peut avoir ajouté des restrictions.
</Callout>

### Capacités avancées

**Génération de tests unitaires** : Devstral lit votre fonction et pond des cas de test couvrant les cas limites.

**Migration de code** : Python 2 → 3, JavaScript → TypeScript, class components React → hooks. Devstral guide le refactoring.

**Documentation auto** : il lit votre code et génère des docstrings, des README, des guides API.

**Revue de code** : style, bugs potentiels, optimisations, violations des bonnes pratiques.

### Dans votre workflow

- **IDE** : autocomplétion + refactoring en temps réel (VS Code, JetBrains)
- **CI/CD** : revue de code auto, génération de tests, détection de régressions
- **Ligne de commande** : modifications batch, refactoring de projets entiers
- **Chat** : discussion interactive sur l'architecture
