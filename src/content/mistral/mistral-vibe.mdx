## Mistral Vibe : Mistral dans ton terminal

Les devs vivent dans le terminal. `cd`, `git`, `npm`, `vim`. Pourquoi sortir du terminal pour appeler Mistral? Vibe? L'IA vient à toi, dans ton terminal.

"Je veux refactoriser ce fichier" → Tu le dis, Mistral le fait. Pas d'interface web, pas de sortir du terminal.

### Pourquoi?

Les vrais devs: Neovim, tmux, zero GUI. Vibe c'est pour eux. CLI-first.

**Le nom « Vibe »** fait référence au concept de « vibe coding » — cette approche de développement où le développeur décrit ce qu'il veut accomplir en langage naturel et laisse l'IA s'occuper de l'implémentation.

<Callout type="info" title="CLI-first">
L'approche CLI-first de Vibe est complémentaire des intégrations IDE (comme Codestral dans VS Code). Les développeurs qui préfèrent le terminal (utilisateurs de Neovim, tmux, systèmes headless) bénéficient d'une assistance IA sans quitter leur environnement de travail.
</Callout>

### Fonctionnalités principales

#### Manipulation de fichiers

Vibe peut créer, modifier et organiser des fichiers en suivant des instructions en langage naturel.

**Création de fichiers :** « Crée un composant React pour un formulaire de connexion avec validation. »

**Modification :** « Ajoute la gestion des erreurs dans le fichier utils/api.ts. »

**Refactoring :** « Refactorise les fichiers du dossier src/services pour utiliser des classes TypeScript. »

**Opérations batch :** « Renomme tous les fichiers .jsx en .tsx et mets à jour les imports. »

#### Recherche de code

Vibe utilise la compréhension sémantique des modèles Mistral pour effectuer des recherches de code plus intelligentes que le grep traditionnel.

**Recherche sémantique :** « Trouve toutes les fonctions qui font des appels API externes. »

**Recherche par intention :** « Où est-ce que la validation des emails est implémentée ? »

**Analyse de dépendances :** « Quels fichiers seraient impactés si je modifie la structure de User ? »

#### Compréhension de projet

Vibe peut analyser un projet dans son ensemble et répondre à des questions sur son architecture.

**Architecture :** « Explique-moi la structure de ce projet et les principaux patterns utilisés. »

**Documentation :** « Génère une documentation des API publiques de ce module. »

**Onboarding :** « Guide-moi à travers les principaux fichiers pour comprendre comment fonctionne l'authentification. »

<Callout type="tip" title="Contexte du projet">
Vibe analyse automatiquement la structure du projet (fichiers, dossiers, configuration, dépendances) pour fournir des réponses contextualisées. Il comprend les conventions du projet et s'y conforme lors de la génération ou modification de code.
</Callout>

#### Gestion de version

Vibe s'intègre avec Git pour assister dans les opérations de gestion de version.

**Commits :** « Crée un commit avec un message décrivant les changements actuels. »

**Branches :** « Crée une branche feature pour l'ajout du système de notifications. »

**Review :** « Analyse les changements du dernier commit et suggère des améliorations. »

**Résolution de conflits :** « Aide-moi à résoudre les conflits de merge dans ce fichier. »

### Architecture technique

Vibe est une application CLI qui communique avec les modèles Mistral via La Plateforme.

**Client local.** L'outil CLI tourne localement et a accès au système de fichiers du développeur. Il peut lire les fichiers du projet, analyser la structure des répertoires et exécuter des commandes shell.

**Communication API.** Les requêtes sont envoyées à l'API Mistral (La Plateforme) pour le traitement par les modèles de langage. Le contexte du projet est injecté dans les prompts pour obtenir des réponses pertinentes.

**Exécution locale.** Les modifications de fichiers sont effectuées localement par le CLI. Le développeur peut prévisualiser les changements avant de les appliquer.

### Installation et configuration

Vibe est distribué comme un package installable via les gestionnaires de packages standard.

**Installation :** via npm, pip ou un binaire précompilé selon la plateforme.

**Configuration :** une clé API Mistral est nécessaire (configurable via variable d'environnement ou fichier de configuration).

**Personnalisation :** le comportement de Vibe peut être personnalisé via un fichier de configuration dans le projet (modèle préféré, conventions de code, fichiers à ignorer).

### Modes d'interaction

Vibe propose plusieurs modes d'interaction.

#### Mode conversationnel

Un chat interactif dans le terminal où le développeur pose des questions et reçoit des réponses, avec le contexte du projet chargé automatiquement.

#### Mode commande

Des commandes directes pour des actions spécifiques : `vibe fix "corrige le bug dans auth.py"`, `vibe explain "qu'est-ce que fait cette regex ?"`, `vibe create "un middleware Express pour le rate limiting"`.

#### Mode pipe

Vibe peut être intégré dans des pipelines shell : `cat error.log | vibe "explique ces erreurs"`, `git diff | vibe "résume ces changements"`.

<Callout type="warning" title="Sécurité et données">
Vibe envoie des fragments de code à l'API Mistral pour le traitement. Pour les projets contenant du code propriétaire sensible, il est important de comprendre que le code est transmis à l'API (même si Mistral ne l'utilise pas pour l'entraînement). Les entreprises ayant des politiques strictes de confidentialité du code peuvent préférer un déploiement on-premise des modèles Mistral pour utiliser Vibe sans transmission externe.
</Callout>

### Cas d'usage avancés

#### Automatisation de tâches répétitives

Les tâches de développement répétitives (ajout de tests, mise à jour de documentation, migration de patterns) peuvent être automatisées via des scripts utilisant Vibe.

#### Revue de code automatisée

Intégré dans un pipeline CI/CD, Vibe peut effectuer une revue de code automatisée sur les pull requests, identifiant les problèmes potentiels et suggérant des améliorations.

#### Pair programming en solo

Vibe permet à un développeur solo de bénéficier d'une forme de pair programming, avec un « partenaire » qui peut questionner les choix, suggérer des alternatives et identifier des problèmes potentiels.

Mistral Vibe s'inscrit dans la vision de Mistral AI de fournir des outils d'IA intégrés à chaque étape du workflow de développement, du terminal à l'IDE en passant par la production.
