## Mistral sur AWS et GCP : partout, pas juste Azure

Mistral n'est pas limité à Azure. Les modèles Mistral tournent aussi sur **AWS Bedrock** et **Google Cloud Model Garden**. Si vous êtes sur l'un de ces clouds, vous avez accès à Mistral sans changer d'infrastructure.

### AWS Bedrock

Amazon Bedrock est le « app store » des modèles de fondation chez AWS. Mistral y est disponible.

**Modèles dispo** :
- Mistral Large
- Mistral Small
- Mixtral 8x7B
- Mistral 7B

Tous via une API managée (pas de serveurs à gérer).

**Intégrations AWS** :
- **Lambda** : invoquez Mistral depuis des fonctions sans serveur
- **S3** : les données d'entrée/sortie directement depuis S3
- **SageMaker** : pour du fine-tuning ou de l'évaluation avancée
- **IAM** : contrôle d'accès granulaire par modèle
- **CloudWatch** : monitoring des appels et des coûts

**Knowledge Bases Bedrock** : RAG clé en main. Documents uploadés → chunked/vectorisés → indexés. Requêtes enrichies automatiquement avec le contexte pertinent.

<Callout type="info" title="Bedrock Knowledge Bases">
Si vous avez des doc propriétaires et que vous voulez faire du RAG, Bedrock Knowledge Bases automatise tout. C'est du RAG sans pipelines ETL compliqués.
</Callout>

**Guardrails Bedrock** : filtrage de contenu, détection de PII, deny lists. Superpose avec les guardrails natifs de Mistral.

### Google Cloud Model Garden

Google Cloud propose Mistral via Model Garden avec un accent sur **Vertex AI** (leur plateforme ML).

**Modèles dispo** :
- Mistral Large (API managée)
- Mistral Small (API managée)
- Mixtral 8x7B (API managée + self-deployed sur GKE)
- Modèles open-weight (déployables sur GKE)

**Intégrations GCP** :
- **Vertex AI** : fine-tuning, évaluation, déploiement custom
- **BigQuery** : appelez Mistral **directement depuis des requêtes SQL**. À l'échelle.
- **Cloud Functions / Cloud Run** : serverless
- **GKE** : deployer les modèles open-weight sur vos clusters Kubernetes

**BigQuery ML + Mistral** : la killer feature. Une query SQL qui enrichit des millions de lignes avec du LLM sans pipeline complexe. Exemple : classer des millions d'avis clients en une requête.

<Callout type="tip" title="BigQuery SQL + Mistral">
Vous avez 10 millions de commentaires clients dans BigQuery. Une seule requête SQL avec Mistral les classe et les résume. Pas de script ETL, pas de pipeline. Just SQL.
</Callout>

### Comparaison des trois clouds

| Aspect | Azure | AWS Bedrock | GCP Model Garden |
|--------|-------|-------------|------------------|
| Partenariat Mistral | Fort | Standard | Standard |
| Modèles dispo | Large sélection | Bonne sélection | Bonne sélection |
| Fine-tuning | Azure ML | Bedrock Custom | Vertex AI |
| RAG intégré | Azure AI Search | Knowledge Bases | Vertex AI Search |
| Guardrails | Content Safety | Bedrock Guardrails | Intégrés |
| Régions EU | France, Pays-Bas | Irlande, Frankfurt | Belgique, Pays-Bas |
| OpenAI compatible | Oui | Oui | Partiel |

### Comment choisir ?

**Vous êtes Microsoft-first ?** Azure. C'est là que le partenariat est le plus profond.

**Vous êtes AWS-first ?** Bedrock. Knowledge Bases pour le RAG, intégration lambda/SageMaker solide.

**Vous êtes GCP-first ?** Model Garden + Vertex AI. BigQuery ML pour les cas data-intensifs.

**Vous êtes multi-cloud ?** Utilisez La Plateforme Mistral directe. Neutre, souvent meilleur prix.

<Callout type="warning" title="Lock-in cloud">
Plus vous intégrez Mistral aux services spécifiques d'un cloud (guardrails, RAG, monitoring), plus vous êtes locked-in. Si portabilité = importante, restez sur La Plateforme directe et outils cloud-agnostiques.
</Callout>

### Tarification

Mais :

- Intégration écosystème
- Support cloud
- Crédits existants potentiels

Souvent ça compense.
