## La Plateforme : votre porte d'entrée à Mistral

**La Plateforme**, c'est l'API de Mistral. Lancée en février 2024, elle permet à n'importe quel développeur d'utiliser les modèles Mistral via du code, avec un tier gratuit et une tarification juste.

L'API est compatible OpenAI, donc si vous avez un code qui utilise ChatGPT, changer vers Mistral = changer deux lignes.

### Modèles disponibles

| Modèle | Endpoint | Contexte | Multimodal |
|--------|----------|----------|------------|
| Mistral Small | `mistral-small-latest` | Variable | Non |
| Mistral Medium | `mistral-medium-latest` | Variable | Non |
| Mistral Large | `mistral-large-latest` | 256K | Oui |
| Codestral | `codestral-latest` | Variable | Non |
| Magistral | `magistral-latest` | Variable | Non |
| Mistral Embed | `mistral-embed` | 8K | Non |
| Pixtral | `pixtral-large-latest` | Variable | Oui |

Tous les modèles, une seule API. Les noms en `-latest` vous donnent toujours la version courante.

<Callout type="info" title="Compatibilité OpenAI">
L'API Mistral suit le même format qu'OpenAI. Migrer depuis ChatGPT ? Changez l'URL et la clé API, c'est tout. LangChain et LlamaIndex supportent Mistral nativement.
</Callout>

### Tier gratuit

Vous pouvez **tester gratuitement** sans carte bancaire.

- Limites : moins de requêtes/jour qu'un tier payant
- Accès : modèles d'entrée de gamme
- Pas de SLA, pas de support prioritaire

Parfait pour les prototypes, les étudiants, les petits projets.

**Setup = 2 minutes** : console.mistral.ai → créer un compte → générer une clé API → commencer.

### Tarification

Payez ce que vous consommez, au token.

**Logique** :
- Tokens d'entrée < tokens de sortie (payer plus pour ce qu'on génère)
- Modèles puissants = plus chers
- Batch processing = réductions

Les prix changent, mais Mistral est généralement compétitif. La Plateforme directe est souvent moins cher que passer par les clouds publics.

### Rate limits

Combien de requêtes par minute vous pouvez faire ?

- **Gratuit** : limites strictes
- **Payant** : plus élevées, ajustables sur demande
- **Enterprise** : sans limites pratiques

Les headers HTTP vous disent combien il vous reste.

### Fonctionnalités clés

**Chat Completions** : l'endpoint principal. Vous envoyez un historique de messages, Mistral génère la suite.

Paramètres courants :
- `temperature` : créativité (0=déterministe, 1=créatif)
- `max_tokens` : limite de longueur
- `top_p` : nucleus sampling
- `tools` : définissez des fonctions que le modèle peut appeler
- `response_format` : forcez du JSON

**Streaming** : recevez la réponse token par token via SSE, parfait pour l'UX temps réel.

**Embeddings** : convertir du texte en vecteurs pour la recherche sémantique. Mistral Embed = 1024 dimensions.

**Fine-tuning** : créez des modèles personnalisés sur vos données.



### SDKs

- **Python** : `pip install mistralai`
- **JavaScript/TypeScript** : `npm install @mistralai/mistralai`
- **Autres** : Go, Rust, Java (communauté)

<Callout type="warning" title="Gardez vos clés API secrètes">
Jamais d'API key en public. Pas dans le HTML du frontend, pas dans GitHub. Variables d'environnement. Rotation régulière.
</Callout>

### Monitoring

**Dashboard console.mistral.ai** :
- Vue temps réel de la consommation (tokens, coûts)
- Historique des appels (latence, erreurs)
- Alertes configurables

### Bonnes pratiques

**Retry avec backoff** : si vous hit un rate limit, attendez avant de retry (exponentiellement).

**Caching** : même question deux fois = même réponse ? Mettez en cache.

**Timeout** : adaptez à votre cas d'usage. Chat court = timeout court.

**Fallback** : si Large timeout, basculez sur Small. La résilience, c'est important.
