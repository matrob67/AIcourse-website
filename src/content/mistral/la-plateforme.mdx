## La Plateforme : votre porte d'entree a Mistral

**La Plateforme** est l'API de Mistral. Lancee en fevrier 2024, elle permet a n'importe quel developpeur d'acceder a tous les modeles Mistral via du code, avec un tier gratuit pour demarrer et une tarification au token pour scaler.

Le point crucial : l'API est **compatible avec le format OpenAI**. Si vous avez deja du code qui appelle ChatGPT, passer a Mistral revient a changer deux lignes (l'URL de l'endpoint et la cle API). Le reste de votre code ne bouge pas.

### Modeles disponibles

| Modele | Endpoint | Contexte | Multimodal |
|--------|----------|----------|------------|
| Mistral Small | `mistral-small-latest` | Variable | Non |
| Mistral Medium | `mistral-medium-latest` | Variable | Non |
| Mistral Large | `mistral-large-latest` | 256K | Oui |
| Codestral | `codestral-latest` | Variable | Non |
| Magistral | `magistral-latest` | Variable | Non |
| Mistral Embed | `mistral-embed` | 8K | Non |
| Pixtral | `pixtral-large-latest` | Variable | Oui |

Tous les modeles, une seule API, un seul compte. Les endpoints en `-latest` pointent toujours vers la version courante du modele, ce qui vous evite de gerer les versions manuellement.

<Callout type="info" title="Compatibilite OpenAI : migration en 2 minutes">
L'API Mistral suit le meme format qu'OpenAI (meme structure de messages, memes parametres). Migrer depuis ChatGPT ? Changez l'URL de base et la cle API, c'est tout. Les frameworks comme LangChain et LlamaIndex supportent Mistral nativement, donc la migration est encore plus simple si vous les utilisez.
</Callout>

### Tier gratuit : demarrez sans carte bancaire

Vous pouvez **tester gratuitement** tous les modeles d'entree de gamme sans fournir de carte bancaire :

- Limites : moins de requetes par jour qu'un tier payant
- Acces : modeles d'entree de gamme (Small, embeddings)
- Pas de SLA, pas de support prioritaire

C'est parfait pour les prototypes, les etudiants, les projets personnels, et la decouverte de l'ecosysteme.

**Demarrage en 2 minutes** : allez sur console.mistral.ai → creez un compte → generez une cle API → lancez votre premiere requete. C'est aussi simple que ca.

### Tarification : payez ce que vous consommez

Le modele economique est simple : vous payez au token consomme, sans abonnement fixe.

**La logique tarifaire** :
- Les tokens d'entree (votre prompt) coutent moins cher que les tokens de sortie (la reponse generee). C'est logique : generer du texte est plus couteux que le lire.
- Les modeles plus puissants coutent plus cher par token (Large > Medium > Small)
- Le batch processing (traitement en lot) beneficie de reductions de 25 a 50%

Les prix evoluent, mais Mistral est generalement competitif par rapport a OpenAI et Anthropic. Et La Plateforme directe est presque toujours moins chere que de passer par les clouds publics (Azure, AWS, GCP) qui ajoutent leur marge.

### Rate limits : combien de requetes par minute

- **Gratuit** : limites strictes (suffisantes pour le dev, pas pour la production)
- **Payant** : limites plus elevees, ajustables sur demande
- **Enterprise** : pratiquement sans limites

Les headers HTTP de chaque reponse indiquent votre consommation restante, ce qui facilite la gestion cote code.

### Fonctionnalites cles de l'API

**Chat Completions** : l'endpoint principal. Vous envoyez un historique de messages (system, user, assistant), et Mistral genere la suite de la conversation.

Parametres courants que vous utiliserez :
- `temperature` : controle la creativite (0 = deterministe et factuel, 1 = creatif et varie)
- `max_tokens` : limite la longueur de la reponse
- `top_p` : nucleus sampling pour controler la diversite
- `tools` : definissez des fonctions que le modele peut appeler (function calling)
- `response_format` : forcez le modele a repondre en JSON structure

**Streaming** : recevez la reponse token par token via Server-Sent Events (SSE). Indispensable pour l'experience utilisateur en temps reel : l'utilisateur voit la reponse apparaitre progressivement au lieu d'attendre la fin de la generation.

**Embeddings** : convertissez du texte en vecteurs pour la recherche semantique. L'endpoint Mistral Embed produit des vecteurs de 1024 dimensions.

**Fine-tuning** : creez des modeles personnalises entraines sur vos propres donnees, directement depuis l'API.



### SDKs officiels et communautaires

- **Python** : `pip install mistralai` (SDK officiel)
- **JavaScript/TypeScript** : `npm install @mistralai/mistralai` (SDK officiel)
- **Go, Rust, Java** : SDKs communautaires disponibles

<Callout type="warning" title="Gardez vos cles API secretes">
Regle d'or : jamais de cle API dans du code public. Pas dans le HTML du frontend, pas dans un depot GitHub, pas dans un fichier de config commite. Utilisez des variables d'environnement. Faites tourner vos cles regulierement. Une cle API compromise peut generer des milliers d'euros de consommation en quelques heures.
</Callout>

### Dashboard et monitoring

Le dashboard sur console.mistral.ai offre une vue complete de votre utilisation :
- **Consommation en temps reel** : tokens consommes, couts par modele, par jour
- **Historique des appels** : latence, taux d'erreur, volumes
- **Alertes configurables** : soyez prevenu si votre consommation depasse un seuil

### Bonnes pratiques en production

**Retry avec backoff exponentiel** : si vous atteignez un rate limit (erreur 429), n'envoyez pas immediatement une nouvelle requete. Attendez 1 seconde, puis 2, puis 4, puis 8. L'exponential backoff evite de saturer l'API.

**Caching intelligent** : si la meme question revient regulierement avec la meme reponse attendue, mettez le resultat en cache. Ca reduit les couts et la latence.

**Timeout adapte** : configurez le timeout en fonction du cas d'usage. Un chat court = timeout court (10 secondes). Une generation longue = timeout plus genereux (60 secondes).

**Fallback entre modeles** : si Large est trop lent ou indisponible, basculez automatiquement sur Small. La resilience de votre application ne doit pas dependre d'un seul modele.
