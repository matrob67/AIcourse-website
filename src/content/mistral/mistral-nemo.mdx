## Mistral NeMo : la collaboration Mistral-NVIDIA

**Mistral NeMo 12B** = co-développé par Mistral + NVIDIA. **12 milliards de paramètres**. **128K tokens de contexte** (énorme pour la taille).

Multilingue. Apache 2.0 (libre). C'est le modèle polyvalent idéal : puissant, pas trop lourd, contexte massif.

### Pourquoi « NeMo » ?

NeMo = framework NVIDIA pour les LLM. Le fait que le modèle porte ce nom = intégration profonde avec l'écosystème NVIDIA.

**Partenariat** : pas juste une vente de GPUs. Co-optimisation. Accès prioritaire au hardware. Distribution via NVIDIA.

<Callout type="info" title="NVIDIA NeMo Framework">
NeMo = framework open-source pour train/fine-tune/deploy des LLM sur NVIDIA hardware. NeMo 12B s'intègre naturellement.
</Callout>

### Contexte : 128K tokens

**128K = ~100 000 mots = un livre complet de 300 pages.**

Pour un modèle de 12B, c'est extraordinaire. Typiquement, plus petit = contexte plus court.

Comment ? Attention avancée (RoPE scaling), training sur long sequences, Tekken tokenizer qui économise les tokens.

**Résultat** : analyse de docs complets sans RAG.

<Callout type="tip" title="128K sans RAG">
Avant : doc long → chunking → retrieval → RAG pipeline compliqué.
Maintenant : fit le doc entier en contexte.
Architectures plus simples.
</Callout>

### Tokenizer Tekken

**Tekken** = nouveau tokenizer multilingue.

- Vocab plus gros = meilleur support de langues non-anglaises
- Efficiency multilingue : moins de tokens pour français, allemand, espagnol, chinois, etc.
- Byte-level BPE avec optimisations pour scripts non-latins

**Impact** : un texte en français ≈ moins de tokens qu'avant → contexte effectif plus grand pour EU.

### Multilingue

NeMo excelle dans multiple langues.

**EU languages** : français, allemand, espagnol, italien, portugais = super solides.

**Asian languages** : chinois, japonais, coréen = significant improvement vs. previous Mistral.

**Code-switching** : mélanger les langues dans une conversation = pas de problème.

<Callout type="info" title="Multilingue = critère clé pour EU">
Une startup EU opérant en France, Allemagne, Espagne, Italie → besoin de multilingue. NeMo + Apache 2.0 = solution attractive.
</Callout>

### Performances

**vs LLaMA 3 8B** : NeMo 12B (un peu plus gros) gagne sur benchmark, surtout multilingue et long context.

**Rapport taille/perf** : rivalise avec des modèles de 13-15B.

### Cas d'usage

**Docs longs** : rapports annuels, contrats, manuels → 128K contexte = handle en une shot.

**Multilingual apps** : chatbots, assistants pour multiples pays.

**Prototypage** : taille manageable + Apache 2.0 = bon pour dev.

**Fine-tuning** : 12B assez gros pour avoir des capacités, assez petit pour fine-tune sans bcp de data.

### Disponibilité

- **Hugging Face** : open-weight, Apache 2.0
- **La Plateforme** : API Mistral
- **NVIDIA NeMo ecosystem** : training/deploy optimisé
- **Quantized versions** : GGUF, AWQ, GPTQ (pour hardware grand public)

<Callout type="warning" title="Contexte long = mémoire">
128K tokens = beaucoup de RAM. Cache KV proportionnel à longueur. Infrastructure doit match la longueur de contexte réelle.
</Callout>
