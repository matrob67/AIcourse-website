## Mistral Compute : l'IA européenne sur sol français

Mistral a construit son propre **datacenter** en Essonne (Île-de-France). Pas AWS, pas Azure, pas GCP. Leurs serveurs, leurs GPU, leur territoire.

Pourquoi ? Souveraineté. Les données (et les modèles) ne transitent **jamais** par les États-Unis.

### Pourquoi construire votre propre infra ?

**Souveraineté juridique** : le Cloud Act américain permet aux feds de fouiller dans les données même hébergées ailleurs. Mistral Compute = zéro ambiguïté.

**Coûts** : louer des GPU dans le cloud coûte énormément. À long terme, posséder ses GPUs est **drastiquement** moins cher. Pour de l'entraînement 24/7, l'amortissement = 2-3 ans.

**Garantie de disponibilité** : le marché GPU est crazy. Posséder sa capacité = pas de waiting lists.

**Performance optimale** : un datacenter conçu pour les LLM peut être optimisé spécifiquement (networking, refroidissement, alimentation).

<Callout type="info" title="L'économie des GPU">
GPU H100 en cloud = 2-3$/heure. Un cluster de 2000 GPUs = 4-6M$/mois. Acheter les GPUs (~30k chacun) + construire le DC = 60-70M$. Mais après 2-3 ans, les coûts opérationnels sont bien moins chers.
</Callout>

### Localisation : Essonne/Île-de-France

**Proximité Paris** : HQ Mistral = Paris. Ops faciles.

**Électricité** : France = 75% nucléaire. Décarboné, stable, pas cher. Un avantage énorme pour l'empreinte carbone.

**Connectivity** : région parisienne = principal hub internet EU. Faible latence vers le reste de l'Europe.

**Écosystème** : Saclay, pôles de recherche, startups. Talent pool.



### Partenariat NVIDIA

NVIDIA fournit les GPU, collabore sur l'architecture du DC.

**Accès prioritaire** : Mistral obtient les dernières générations (Hopper H100/H200, bientôt Blackwell B200).

**Co-optimisation** : les équipes NVIDIA aident à optimiser la config (topologie réseau, refroidissement, placement).

**Support technique** : accès direct au support NVIDIA pour debugging et optimisation.

**DGX/HGX** : Mistral utilise probablement les serveurs pré-configurés de NVIDIA optimisés pour l'entraînement.

<Callout type="tip" title="L'avantage du partenariat">
Mistral n'utilise pas juste du hardware NVIDIA. Il collabore avec l'équipe NVIDIA pour exploiter pleinement les capacités. Accès anticipé aux nouvelles techs, formations spécialisées, partnerships.
</Callout>

### Spécifications estimées

**GPU** : plusieurs milliers de dernière génération NVIDIA (assez pour entraîner des modèles de 675B+ comme Mistral Large).

**Réseau** : InfiniBand NDR 400Gb/s entre les nœuds. NVLink 4.0 intra-nœud. Fat-tree topology pour max bandwidth.

**Stockage** : NVMe haute perf + distributed storage pour les datasets d'entraînement.

**Refroidissement** : adapté à la densité thermique énorme des GPU modernes.

**Redondance** : alimentation redondante, générateurs de secours, failover auto.

### Souveraineté : presque complète

Mistral Compute offre la **souveraineté logicielle et de données** (les données ne quittent jamais l'UE).

Mais : les GPU sont conçus aux US (NVIDIA) et fabriqués à Taïwan (TSMC). L'Europe n'a pas encore de production de chips haute performance.

Dépendance résiduelle, mais on ne peut pas l'éviter (aucun pays n'a la capacity).

### Impact environnemental

**Consommation** : des milliers de GPU = plusieurs mégawatts. Énorme.

**Avantage** : énergie française = 75% nucléaire = low carb vs. du charbon/gaz.

**Efficacité** : le PUE (Power Usage Effectiveness) du DC mesure l'efficacité énergétique. Mistral vise les meilleurs standards.

### Alternative EU aux clouds US

C'est un signal fort : tu **peux** construire de l'infrastructure EU-grade IA, sans dépendre de Silicon Valley.

Pour les clients des secteurs réglementés (défense, santé, finance), c'est une alternative crédible.
