## Pourquoi la simulation : un million d'essais par jour vs quelques par jour

Collecter données robotiques en monde réel = **lent, coûteux, dangereux**. Un robot physique : quelques centaines de tentatives par jour. En simulation : **millions de trajectoires en quelques heures**, parallèles sur GPUs.

La main robotique Dactyl d'OpenAI a résolu le Rubik's Cube après l'équivalent de 13 000 ans d'expérience. Impossible collecter ça dans monde réel. **Sim-to-real transfer** = entraîner en simulation, déployer sur robot physique.



<Callout type="info" title="L'échelle par la simulation">
13 000 années d'expérience équivalent. Impossible en monde réel. La simulation fournit l'échelle de données nécessaire pour des politiques robustes. C'est fondamental.
</Callout>

## Les simulateurs principaux

**NVIDIA Isaac Sim** : plateforme NVIDIA construite sur **Omniverse**.
- **Rendu photoréaliste** : ray-tracing temps réel, images quasi indistinguibles réalité
- **Physique GPU-accélérée** : milliers environnements parallèles via PhysX
- **Isaac Lab** : framework entraînement RL pour manipulation/locomotion
- **Intégration ROS** : connexion écosystème Robot Operating System
- **Digital twins** : réplication fidèle environnements réels

<Callout type="tip" title="Isaac Lab pour foundation models">
Isaac Lab spécialement conçu pour entraîner foundation models robotiques. Support RL massif, imitation learning, évaluation VLA.
</Callout>

**MuJoCo** (Multi-Joint dynamics with Contact) : simulateur référence recherche.
- Développé Emo Todorov, racheté DeepMind 2022, rendu gratuit open-source
- **Physique haute fidélité** : contacts, frottements, contraintes précises
- **Rapidité** : ultra-rapide CPU, idéal RL millions épisodes
- **Stabilité numérique** : intégrateur robuste contacts complexes
- **Standard académique** : utilisé majorité publications RL robotique

**Habitat** (Meta) : spécialisé **navigation + interaction** environnements intérieurs.
- Scans 3D vrais appartements/maisons
- **Habitat 3.0** : humains virtuels interagissant robots
- Navigation sociale : robot navigue présence humains
- Rearrangement : tâches rangement objets

Autres : **PyBullet** (léger), **Gazebo** (intégré ROS), **SAPIEN** (objets articulés), **Genesis** (nouveau, génératif).

## Le reality gap : le grand défi non résolu

**Reality gap** = différence simulation-réalité :
- Physique imparfaite : contacts, frictions, déformations jamais parfaitement modélisés
- Visuels différents : même ray-tracing diffère réalité
- Dynamique robot : moteurs réels ont retards, jeux mécaniques, usure
- Objets imprévisibles : objet mouillé/déformable se comporte différemment

Une politique réussissant 95% en simulation peut tomber 20% monde réel si reality gap mal gérée.

<Callout type="warning" title="Échec classique sim-to-real">
Une politique réussissant 95% simulation peut échouer 80% monde réel. Simulation n'est utile que si transfert fonctionne. C'est l'un des problèmes durcs de la robotique.
</Callout>

## Techniques sim-to-real transfer

**Domain Randomization** : **randomiser massivement** paramètres simulation :
- Textures/couleurs aléatoires
- Éclairage variable
- Paramètres physiques perturbés (friction, masse, raideur)
- Bruit capteurs/actionneurs
- Positions initiales variées

Idée : si agent fonctionne dans toutes ces conditions, fonctionne aussi monde réel (une instanciation parmi d'autres).

**Domain Adaptation** : plutôt que randomiser, **adapter** représentations :
- Sim-to-real via GANs : transformer images simulées ressembler réelles
- Feature alignment : apprentissage représentations invariantes domaine
- Fine-tuning données réelles : affiner politique sur trajectoires réelles

**System Identification** : ajuster paramètres simulateur correspondre robot réel :
- Mesure paramètres physiques réels
- Calibration automatique modèles dynamique
- Création digital twins précis

## Perspectives

Tendance : combinaison approches + foundation models. NVIDIA GR00T utilise Isaac Sim pré-entraîner modèles robotiques généralistes. **Genesis** propose générer automatiquement environnements simulation via IA générative, réduisant coût données synthétiques.
