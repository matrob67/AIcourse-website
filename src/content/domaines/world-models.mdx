## Imagination artificielle : le robot qui pense avant d'agir

Quand vous imaginez attraper un verre d'eau, votre cerveau simule mentalement le mouvement, anticipe le poids, prédit résultat. **World models** robots : « simulateur interne » permettant raisonner sur conséquences actions sans les exécuter physiquement.

<Callout type="info" title="L'analogie humaine">
Votre cerveau simule mentalement les conséquences avant agir. World models donnent robots capacité similaire : planifier en simulant dans modèle appris plutôt que monde réel.
</Callout>

## Fondements : Model-Based vs Model-Free RL

**Model-Free** (PPO, SAC) : agent apprend politique (état → action) directement sans modèle environnement. Simple, nécessite énormément données.

**Model-Based** : agent apprend modèle dynamique environnement, puis l'utilise planifier. Plus sample-efficient, mais modèle peut être inexact.

World models = forme sophistiquée model-based RL où modèle = réseau neuronal profond prédisant observations complexes.

### Cadre général

World model apprend trois fonctions :
1. **Modèle transition** : prédit état suivant étant donné état actuel et action
2. **Modèle observation** : génère observations (images) à partir état latent
3. **Modèle récompense** : prédit récompense associée transition

## Architectures clés

**World Models** (Ha & Schmidhuber, 2018, fondateur) :
- **V (Vision)** : VAE comprimant images en vecteurs latents
- **M (Memory)** : RNN (MDN-RNN) prédisant évolution vecteurs latents
- **C (Controller)** : réseau linéaire choisissant actions

Agent peut être entraîné entièrement « dans son rêve » — interagissant avec world model plutôt qu'environnement réel.

**Dreamer** (Danijar Hafner) : famille world models la plus aboutie :
- **V1** (2020) : apprend politique par backprop à travers world model
- **V2** (2021) : atteint performances surhumaines Atari sans model-free RL
- **V3** (2023) : fonctionne large gamme domaines sans tuning hyperparamètres

<Callout type="tip" title="DreamerV3 la généralisation">
DreamerV3 utilise espace latent discret (symlog predictions), objectif valeur robuste, exploration intrinsèque. Fonctionne Minecraft à locomotion robotique sans changer hyperparamètres — prouesse généralisation algorithmique.
</Callout>

**TD-MPC2** (Temporal Difference Model Predictive Control) : combine world models et contrôle prédictif :
- Apprend modèle latent dynamique
- MPC pour planifier temps réel
- Fonctionne 80+ tâches contrôle continu, single ensemble hyperparamètres
- Adapté tâches robotiques requérant planification précise

## Applications robotique

**Planification manipulation** : robot simule mentalement différentes stratégies saisie avant choisir meilleure. Prédiction trajectoire objet quand poussé. Anticipation collisions. Évaluation stabilité prise.

**Navigation** : prédire conséquences déplacements. Évitement obstacles anticipé. Planification chemin environnements dynamiques. Adaptation terrain.

**World Models vidéo** : génération vidéo (Sora, Runway) = world models implicites. Ces modèles apprennent implicitement lois physique pour générer vidéos réalistes. Plusieurs groupes explorent utiliser générateurs vidéo comme world models robotique.

<Callout type="paper" title="UniSim : le simulateur universel appris">
UniSim (Google DeepMind) : simulateur universel appris. Modèle génératif vidéo qui simule conséquences n'importe quelle action dans n'importe quel environnement. Vision ultime world models : simulateur général appris données, remplaçant simulateurs physiques classiques.
</Callout>

## Avantages

- **Sample efficiency** : apprendre modèle puis planifier plus efficace que model-free RL
- **Sécurité** : tester actions dans modèle avant exécuter monde réel
- **Transfert** : bon world model réutilisable pour nouvelles tâches
- **Interprétabilité** : visualiser ce que robot « imagine »

## Limitations

- **Erreurs modèle** : prédictions divergent réalité horizons longs (erreurs compounding)
- **Complexité** : environnements riches dynamiques difficiles modéliser
- **Coût computationnel** : planification dans modèle ajoute latence
- **Objets nouveaux** : modèle ne peut prédire comportement objets jamais vus

<Callout type="warning" title="Le problème compounding error">
Erreurs prédiction world model s'accumulent. 1% erreur par pas = 50% erreur après 50 pas. World models souvent utilisés planification court terme plutôt que trajectoires longues.
</Callout>

## Perspectives

Convergence entre world models et modèles génératifs (diffusion, flow matching) = direction prometteuse. Foundation models vidéo apprennent représentations riches physique monde pouvant servir base world models robotiques universels.
