## Les outils sans lesquels on ne fait rien

PyTorch règne. JAX pour Google DeepMind. Transformers : l'API universelle. Comprendre les trois = comprendre comment l'IA se code.

<Callout type="info" title="L'importance du choix de framework">
Le choix du framework influence la productivité, les performances et l'accès à l'écosystème. PyTorch domine la recherche, JAX est utilisé par Google pour les plus grands modèles, et Transformers est l'interface standard pour utiliser les modèles pré-entraînés.
</Callout>

## PyTorch

### Présentation

Développé par Meta AI (FAIR), PyTorch est le framework de deep learning le plus utilisé :

- **Graphe dynamique** : le graphe de calcul est construit à la volée (eager execution), facilitant le debugging
- **API Pythonique** : syntaxe naturelle Python, facile à apprendre
- **Autograd** : différentiation automatique transparente
- **Écosystème riche** : torchvision, torchaudio, torchtext, Lightning

### Composants clés

- **torch.Tensor** : structure de données fondamentale (similaire à numpy, mais sur GPU)
- **torch.nn** : modules de réseaux de neurones (Linear, Conv2d, Transformer)
- **torch.optim** : optimiseurs (Adam, SGD, AdamW)
- **torch.distributed** : entraînement distribué multi-GPU et multi-nœuds
- **torch.compile** : compilation JIT pour accélérer le code (PyTorch 2.0+)

### PyTorch 2.0+

La version 2.0 a introduit des améliorations majeures :
- **torch.compile** : compilation du code PyTorch en kernels optimisés
- **Scaled Dot Product Attention** : implémentation native de Flash Attention
- Support amélioré du distributed training (FSDP)

<Callout type="tip" title="PyTorch en production">
Historiquement critiqué pour ses performances en production, PyTorch s'est considérablement amélioré avec torch.compile, TorchServe, et l'export ONNX. La majorité des systèmes de production en IA utilisent désormais PyTorch, éliminant l'ancien argument en faveur de TensorFlow pour le déploiement.
</Callout>

## JAX

### Présentation

Développé par Google, JAX est un framework de calcul numérique haute performance :

- **Transformations fonctionnelles** : jit (compilation), grad (différentiation), vmap (vectorisation), pmap (parallélisation)
- **XLA backend** : compilation vers des kernels optimisés pour GPU et TPU
- **Numpy-compatible** : API similaire à NumPy pour une adoption facile
- **Immuabilité** : approche fonctionnelle pure (pas de mutation d'état)

### Les transformations clés

- **jax.jit** : compilation Just-In-Time pour la performance
- **jax.grad** : différentiation automatique (forward et reverse)
- **jax.vmap** : vectorisation automatique (batch processing)
- **jax.pmap** : parallélisation sur plusieurs devices (multi-GPU/TPU)

### Écosystème JAX

- **Flax** : bibliothèque de réseaux de neurones pour JAX (Google)
- **Optax** : optimiseurs pour JAX
- **Equinox** : interface PyTorch-like pour JAX

### JAX vs PyTorch

| Aspect | PyTorch | JAX |
|--------|---------|-----|
| Paradigme | Impératif (eager) | Fonctionnel (compilé) |
| Debugging | Facile (print, breakpoint) | Plus complexe (code fonctionnel) |
| Performance | Très bon (torch.compile) | Excellent (XLA) |
| Multi-GPU/TPU | FSDP, DDP | pmap, sharding natif |
| Écosystème | Massif | Plus petit mais croissant |
| Adoption | Recherche + industrie | Google, recherche HPC |

<Callout type="paper" title="Qui utilise JAX ?">
Google DeepMind utilise JAX pour ses plus grands modèles (Gemini, AlphaFold). JAX est privilégié quand la performance de calcul est critique, notamment pour l'entraînement distribué sur TPU. En dehors de Google, l'adoption reste plus limitée que PyTorch.
</Callout>

## Hugging Face Transformers

### La bibliothèque universelle

Transformers de Hugging Face est le point d'accès standard aux modèles pré-entraînés :

- **200 000+ modèles** accessibles via une API unifiée
- Support de PyTorch, TensorFlow et JAX
- Pipeline en une ligne pour les tâches courantes
- Intégration avec le Model Hub

### Utilisation typique

Le workflow standard est simple et rapide :
1. Choisir un modèle sur le Hub
2. Le charger avec `AutoModel.from_pretrained()`
3. L'utiliser pour l'inférence ou le fine-tuning

### Bibliothèques complémentaires

- **PEFT** : fine-tuning efficace (LoRA, QLoRA)
- **TRL** : Transformer Reinforcement Learning (RLHF, DPO)
- **Accelerate** : entraînement distribué simplifié
- **Datasets** : chargement et traitement de données
- **Evaluate** : métriques d'évaluation
- **Diffusers** : modèles de diffusion (Stable Diffusion, Flux)

<Callout type="warning" title="Abstraction vs Compréhension">
Transformers rend l'utilisation des modèles très facile (quelques lignes de code), ce qui peut masquer la complexité sous-jacente. Pour la recherche et le développement avancé, il est important de comprendre ce qui se passe sous le capot — les abstractions de haut niveau ne suffisent pas.
</Callout>

## Autres outils essentiels

- **vLLM** : serveur d'inférence optimisé pour les LLMs (PagedAttention)
- **TensorRT-LLM** : optimisation d'inférence par NVIDIA
- **DeepSpeed** : bibliothèque d'entraînement distribué (Microsoft)
- **Megatron-LM** : framework d'entraînement de LLMs à grande échelle (NVIDIA)
- **Weights & Biases (wandb)** : suivi d'expériences et monitoring
- **MLflow** : gestion du cycle de vie ML
