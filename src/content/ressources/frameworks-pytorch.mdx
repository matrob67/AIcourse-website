## Les outils sans lesquels on ne fait rien

Tout le monde parle de GPT, Claude, Gemini... mais derriere chaque modele, il y a un **framework de deep learning**. C'est l'infrastructure invisible. Le moteur sous le capot. Et comprendre ces outils, c'est comprendre comment l'IA se construit, pas juste comment elle s'utilise.

PyTorch regne sur la recherche et l'industrie. JAX est l'arme secrete de Google DeepMind. Transformers (la bibliotheque Hugging Face) est l'API universelle qui donne acces a tout. Maitriser au moins un de ces trois, c'est indispensable pour quiconque veut aller au-dela du simple appel d'API.

<Callout type="info" title="L'importance du choix de framework">
Le choix du framework influence ta productivite, tes performances et ton acces a l'ecosysteme. PyTorch domine la recherche et l'industrie, JAX est utilise par Google pour les plus grands modeles (Gemini, AlphaFold), et Transformers est l'interface standard pour utiliser les modeles pre-entraines.
</Callout>

## PyTorch

### Presentation

Developpe par [Meta AI (FAIR)](https://ai.meta.com/), [PyTorch](https://pytorch.org/) est **le** framework de deep learning le plus utilise au monde. Plus de 90% des papers de recherche en IA utilisent PyTorch. C'est le standard de facto.

Pourquoi ? Parce que coder en PyTorch, c'est **coder en Python**. Pas de magie noire, pas de syntaxe bizarre. Tu ecris du code Python normal, et PyTorch s'occupe de la differentiation automatique et de l'execution GPU.

- **Graphe dynamique** : le graphe de calcul est construit a la volee (eager execution), ce qui rend le debugging trivial. Tu peux mettre des `print()` partout.
- **API Pythonique** : syntaxe naturelle Python, facile a apprendre pour quiconque connait Python
- **Autograd** : differentiation automatique transparente. Tu calcules le forward pass, PyTorch calcule les gradients tout seul.
- **Ecosysteme riche** : [torchvision](https://pytorch.org/vision/), [torchaudio](https://pytorch.org/audio/), torchtext, [Lightning](https://lightning.ai/)

### Composants cles

- **torch.Tensor** : structure de donnees fondamentale (comme numpy, mais sur GPU). [Documentation](https://pytorch.org/docs/stable/tensors.html)
- **torch.nn** : modules de reseaux de neurones (Linear, Conv2d, Transformer, etc.). [Documentation](https://pytorch.org/docs/stable/nn.html)
- **torch.optim** : optimiseurs (Adam, SGD, AdamW). [Documentation](https://pytorch.org/docs/stable/optim.html)
- **torch.distributed** : entrainement distribue multi-GPU et multi-noeuds
- **torch.compile** : compilation JIT pour accelerer le code (PyTorch 2.0+). Un game-changer.

### PyTorch 2.0+

La version 2.0 a ete une revolution silencieuse :
- **[torch.compile](https://pytorch.org/docs/stable/torch.compiler.html)** : compilation du code PyTorch en kernels optimises. Souvent 30-50% de speedup gratuit avec une seule ligne de code.
- **Scaled Dot Product Attention** : implementation native de [Flash Attention](https://arxiv.org/abs/2205.14135). Plus besoin de librairies externes.
- Support ameliore du distributed training ([FSDP](https://pytorch.org/docs/stable/fsdp.html))

Documentation officielle : [pytorch.org/docs](https://pytorch.org/docs/stable/index.html)
GitHub : [pytorch/pytorch](https://github.com/pytorch/pytorch) (85K+ stars)
Tutoriels : [pytorch.org/tutorials](https://pytorch.org/tutorials/)

<Callout type="tip" title="PyTorch en production">
Historiquement critique pour ses performances en production, PyTorch s'est considerablement ameliore avec torch.compile, [TorchServe](https://pytorch.org/serve/), et l'export [ONNX](https://onnx.ai/). La majorite des systemes de production en IA utilisent desormais PyTorch. L'argument "TensorFlow pour la prod, PyTorch pour la recherche" est **mort et enterre**.
</Callout>

## JAX

### Presentation

Developpe par Google, [JAX](https://github.com/google/jax) est un framework de calcul numerique haute performance. Si PyTorch, c'est "Python with superpowers", JAX, c'est "numpy on steroids, compiled to the moon".

L'approche est radicalement differente de PyTorch : tout est **fonctionnel** et **composable**. Tu ecris des fonctions pures, et JAX les transforme en code ultra-optimise.

- **Transformations fonctionnelles** : `jit` (compilation), `grad` (differentiation), `vmap` (vectorisation), `pmap` (parallelisation)
- **XLA backend** : compilation vers des kernels optimises pour GPU et TPU
- **Numpy-compatible** : API similaire a NumPy pour une adoption facile
- **Immutabilite** : approche fonctionnelle pure (pas de mutation d'etat)

### Les transformations cles

Les quatre "superpouvoirs" de JAX :

- **[jax.jit](https://jax.readthedocs.io/en/latest/jax-101/02-jitting.html)** : compilation Just-In-Time. Transforme ta fonction Python en code machine optimise.
- **[jax.grad](https://jax.readthedocs.io/en/latest/jax-101/04-advanced-autodiff.html)** : differentiation automatique (forward et reverse). Plus flexible que PyTorch autograd.
- **[jax.vmap](https://jax.readthedocs.io/en/latest/jax-101/03-vectorization.html)** : vectorisation automatique. Ecris ta fonction pour un seul exemple, `vmap` la rend "batchee" automatiquement.
- **[jax.pmap](https://jax.readthedocs.io/en/latest/jax-101/06-parallelism.html)** : parallelisation sur plusieurs devices (multi-GPU/TPU). Scaling horizontal quasi-gratuit.

### Ecosysteme JAX

- **[Flax](https://github.com/google/flax)** : bibliotheque de reseaux de neurones pour JAX (Google). L'equivalent de `torch.nn`.
- **[Optax](https://github.com/google-deepmind/optax)** : optimiseurs pour JAX. L'equivalent de `torch.optim`.
- **[Equinox](https://github.com/patrick-kidger/equinox)** : interface PyTorch-like pour JAX. Le meilleur des deux mondes.

### JAX vs PyTorch

| Aspect | PyTorch | JAX |
|--------|---------|-----|
| Paradigme | Imperatif (eager) | Fonctionnel (compile) |
| Debugging | Facile (print, breakpoint) | Plus complexe (code fonctionnel) |
| Performance | Tres bon (torch.compile) | Excellent (XLA) |
| Multi-GPU/TPU | FSDP, DDP | pmap, sharding natif |
| Ecosysteme | Massif | Plus petit mais croissant |
| Adoption | Recherche + industrie | Google, recherche HPC |
| Courbe d'apprentissage | Douce | Raide (paradigme fonctionnel) |

Documentation : [jax.readthedocs.io](https://jax.readthedocs.io/)
GitHub : [google/jax](https://github.com/google/jax) (30K+ stars)

<Callout type="paper" title="Qui utilise JAX ?">
Google DeepMind utilise JAX pour ses plus grands modeles ([Gemini](https://deepmind.google/technologies/gemini/), [AlphaFold](https://github.com/google-deepmind/alphafold)). JAX est privilegie quand la performance de calcul est **critique**, notamment pour l'entrainement distribue sur TPU. En dehors de Google, l'adoption reste plus limitee que PyTorch, mais des projets comme [Mistral](https://mistral.ai/) l'utilisent aussi.
</Callout>

## Hugging Face Transformers

### La bibliotheque universelle

[Transformers](https://huggingface.co/docs/transformers) de Hugging Face est **le point d'acces standard** aux modeles pre-entraines. C'est comme un App Store pour les modeles d'IA : tu cherches, tu telecharges, tu utilises.

- **200 000+ modeles** accessibles via une API unifiee sur le [Hugging Face Hub](https://huggingface.co/models)
- Support de PyTorch, TensorFlow et JAX
- Pipeline en une ligne pour les taches courantes
- Integration avec le Model Hub

### Utilisation typique

Le workflow standard est simple et rapide :
1. Cherche un modele sur le [Hub](https://huggingface.co/models)
2. Charge-le avec `AutoModel.from_pretrained()`
3. Utilise-le pour l'inference ou le fine-tuning

Documentation : [huggingface.co/docs/transformers](https://huggingface.co/docs/transformers)
GitHub : [huggingface/transformers](https://github.com/huggingface/transformers) (135K+ stars)

### Bibliotheques complementaires

L'ecosysteme Hugging Face va bien au-dela de Transformers :

- **[PEFT](https://huggingface.co/docs/peft)** : fine-tuning efficace (LoRA, QLoRA). Indispensable pour fine-tuner sans 8 GPU.
- **[TRL](https://huggingface.co/docs/trl)** : Transformer Reinforcement Learning (RLHF, DPO). Pour l'alignement.
- **[Accelerate](https://huggingface.co/docs/accelerate)** : entrainement distribue simplifie. Multi-GPU en changeant 3 lignes de code.
- **[Datasets](https://huggingface.co/docs/datasets)** : chargement et traitement de donnees. Plus rapide et plus efficace que pandas pour les gros datasets.
- **[Evaluate](https://huggingface.co/docs/evaluate)** : metriques d'evaluation standardisees.
- **[Diffusers](https://huggingface.co/docs/diffusers)** : modeles de diffusion (Stable Diffusion, Flux, SDXL). Le framework standard pour la generation d'images.

<Callout type="warning" title="Abstraction vs Comprehension">
Transformers rend l'utilisation des modeles tres facile (quelques lignes de code), ce qui peut masquer la complexite sous-jacente. C'est genial pour le prototypage, mais pour la recherche et le developpement avance, il est **crucial** de comprendre ce qui se passe sous le capot. Les abstractions de haut niveau ne suffisent pas quand tu dois debugger un training qui diverge.
</Callout>

## Autres outils essentiels

Le deep learning moderne, c'est tout un ecosysteme. Voici les outils complementaires les plus importants :

- **[vLLM](https://github.com/vllm-project/vllm)** : serveur d'inference optimise pour les LLMs. PagedAttention pour une utilisation memoire efficace. Le standard pour servir des LLMs.
- **[TensorRT-LLM](https://github.com/NVIDIA/TensorRT-LLM)** : optimisation d'inference par NVIDIA. Performance maximale sur GPU NVIDIA.
- **[DeepSpeed](https://github.com/microsoft/DeepSpeed)** : bibliotheque d'entrainement distribue (Microsoft). ZeRO pour entrainer des modeles plus grands que la memoire GPU.
- **[Megatron-LM](https://github.com/NVIDIA/Megatron-LM)** : framework d'entrainement de LLMs a grande echelle (NVIDIA). Utilise pour entrainer les plus gros modeles.
- **[Weights & Biases (wandb)](https://wandb.ai/)** : suivi d'experiences et monitoring. Comme un dashboard pour tes entrainements.
- **[MLflow](https://mlflow.org/)** : gestion du cycle de vie ML. Open-source, self-hosted.
