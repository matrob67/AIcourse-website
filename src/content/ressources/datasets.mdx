## Les données d'abord, tout le reste ensuite

On parle toujours de la taille des modèles : 7B paramètres, 70B, 405B... Mais la vérité que l'industrie a mis du temps a accepter, c'est que **les données comptent plus que les paramètres**. Un modèle de 7B entraîne sur des données exceptionnelles bat un modèle de 70B entraîne sur du bruit.

C'est la leçon de [Chinchilla](https://arxiv.org/abs/2203.15556) : les meilleures IA ne sont pas les plus grosses, ce sont celles avec les **meilleures données**. Et connaître les datasets, c'est comprendre pourquoi un modèle est bon (ou mauvais) dans un domaine spécifique.

<Callout type="info" title="L{"'"}importance de la curation">
Un bon dataset n'est pas simplement un grand dataset. La qualité des données (filtrage, deduplication, diversite) est souvent **plus importante** que la quantite brute. Les meilleurs datasets sont soigneusement curates pour éliminer le bruit, les duplicats et les contenus toxiques. C'est un travail ingrat mais absolument crucial.
</Callout>

## Datasets de pré-entraînement LLM

C'est le carburant des grands modèles de langage. Des **trillions** de tokens aspires du web, filtres, nettoyes et dedupliques.

### Datasets web

- **[Common Crawl](https://commoncrawl.org/)** : le plus grand crawl du web (~250 milliards de pages). Source brute pour la quasi-totalite des datasets de pré-entraînement. C'est le petrole brut de l'IA : inutilisable tel quel, mais indispensable.
- **[FineWeb](https://huggingface.co/datasets/HuggingFaceFW/fineweb)** (Hugging Face) : 15T tokens filtres depuis Common Crawl avec des pipelines de qualité sophistiques. La référence open-source actuelle pour le pré-entraînement. [Blog post technique](https://huggingface.co/spaces/HuggingFaceFW/blogpost-fineweb-v1).
- **[FineWeb-Edu](https://huggingface.co/datasets/HuggingFaceFW/fineweb-edu)** : sous-ensemble educatif de FineWeb, 1.3T tokens de haute qualité educative. Parfait pour entraîner des modèles orientes "connaissance".
- **[RedPajama v2](https://github.com/togethercomputer/RedPajama-Data)** : 30T tokens avec scoring de qualité, réplique ouverte des données LLaMA. Projet de Together AI.
- **[Dolma](https://huggingface.co/datasets/allenai/dolma)** (Allen AI) : 3T tokens, dataset transparent avec documentation complète de chaque étape de curation. Le paper [Dolma](https://arxiv.org/abs/2402.00159) est une lecture obligatoire si tu t'intéresses à la curation de données.
- **[The Pile](https://pile.eleuther.ai/)** (EleutherAI) : 825 GB de texte diversifie (livres, code, web, articles). Historiquement important, c'est le dataset qui a lance le mouvement open-source des LLMs. [Paper](https://arxiv.org/abs/2101.00027).
- **[C4](https://huggingface.co/datasets/allenai/c4)** (Google) : Common Crawl nettoye, utilise pour entraîner T5. Un classique.

### Datasets de code

- **[The Stack v2](https://huggingface.co/datasets/bigcode/the-stack-v2)** (Hugging Face/BigCode) : 67.5TB de code source de GitHub dans 619 langages. Le plus grand dataset de code open-source. [Paper](https://arxiv.org/abs/2402.19173).
- **[StarCoderData](https://huggingface.co/datasets/bigcode/starcoderdata)** : dataset cure pour l'entraînement de [StarCoder](https://huggingface.co/bigcode/starcoder2-15b).

### Datasets multilingues

- **[CulturaX](https://huggingface.co/datasets/uonlp/CulturaX)** : 6.3T tokens dans 167 langues, nettoye et deduplique. Indispensable pour le multilingual.
- **[mC4](https://huggingface.co/datasets/mc4)** : Common Crawl multilingue dans 101 langues.
- **[OSCAR](https://oscar-project.github.io/documentation/)** : corpus web multilingue filtre et classifie, projet français (Inria).

<Callout type="tip" title="Transparence des données">
La tendance forte est à la transparence. Des projets comme Dolma (Allen AI) et FineWeb (Hugging Face) documentent **chaque étape** de leur pipeline de curation, permettant la reproductibilité et l'audit. À l'inverse, les modèles propriétaires (GPT-4, Claude, Gemini) ne publient pas leurs données. Cette asymétrie est l'un des grands débats actuels de l'IA.
</Callout>

## Datasets de fine-tuning et alignement

Une fois le modèle pré-entraîne, il faut lui apprendre a être **utile**. C'est la que les datasets d'instruction tuning et d'alignement entrent en jeu.

### Instruction tuning

- **[OpenAssistant Conversations (OASST)](https://huggingface.co/datasets/OpenAssistant/oasst2)** : 161 000 messages de conversation humains en 35 langues. Projet communautaire impressionnant. [Paper](https://arxiv.org/abs/2304.07327).
- **[Dolly 2.0](https://huggingface.co/datasets/databricks/databricks-dolly-15k)** (Databricks) : 15 000 instructions ecrites par des employes Databricks. Petit mais de haute qualité.
- **[FLAN Collection](https://github.com/google-research/FLAN)** (Google) : melange de 1800+ tâches NLP formattees comme des instructions. Le paper [Flan-T5](https://arxiv.org/abs/2210.11416) montre l'impact de la diversite des tâches.
- **[UltraChat](https://huggingface.co/datasets/stingning/ultrachat)** : 1.5M conversations synthetiques multi-tours.
- **[ShareGPT](https://huggingface.co/datasets/anon8231489123/ShareGPT_Vicuna_unfiltered)** : conversations reelles avec ChatGPT partagees par les utilisateurs. Un des premiers datasets qui a permis de "cloner" le style conversationnel de ChatGPT.

### Alignement (préférences)

C'est ici qu'on apprend au modèle a être **helpful** (utile) et **harmless** (inoffensif) :

- **[HH-RLHF](https://huggingface.co/datasets/Anthropic/hh-rlhf)** (Anthropic) : paires de réponses classees par des humains (helpful, harmless). Le dataset fondateur du RLHF. [Paper](https://arxiv.org/abs/2204.05862).
- **[UltraFeedback](https://huggingface.co/datasets/openbmb/UltraFeedback)** : 64K prompts avec réponses de 17 modèles, annotees par GPT-4.
- **[Nectar](https://huggingface.co/datasets/berkeley-nest/Nectar)** : 183K conversations avec classements de réponses.

## Datasets de vision

### Classification

- **[ImageNet](https://www.image-net.org/)** : 14M images, 1000 classes. **Le** benchmark fondateur de la vision par ordinateur. Le paper original [ImageNet (Deng et al., 2009)](https://arxiv.org/abs/1409.0575) et la competition ImageNet ont lance la révolution deep learning en 2012.
- **ImageNet-21K** : version etendue avec 14M images et 21 841 classes.
- **[CIFAR-10/100](https://www.cs.toronto.edu/~kriz/cifar.html)** : 60K petites images (32x32), 10 ou 100 classes. Le "Hello World" de la vision par ordinateur.

### Détection d'objets

- **[COCO](https://cocodataset.org/)** (Common Objects in Context) : 330K images, 80 catégories, annotations de segmentation et de détection. Le benchmark standard.
- **[Open Images](https://storage.googleapis.com/openimages/web/index.html)** (Google) : 9M images avec annotations de détection, segmentation et relations.

### Segmentation

- **[ADE20K](https://groups.csail.mit.edu/vision/datasets/ADE20K/)** : 25K images avec segmentation sémantique dense (150 classes).
- **[SA-1B](https://segment-anything.com/)** (Meta) : 11M images avec 1.1 milliard de masques de segmentation. C'est le dataset de [SAM (Segment Anything)](https://arxiv.org/abs/2304.02643), un des modèles de vision les plus influents.

## Datasets multimodaux

### Image-texte

- **[LAION-5B](https://laion.ai/blog/laion-5b/)** : 5.85 milliards de paires image-texte crawlees du web. C'est **le** dataset qui a rendu possible Stable Diffusion et la génération d'images open-source. [Paper](https://arxiv.org/abs/2210.08402).
- **LAION-400M** : sous-ensemble de 400M de paires filtrees.
- **[DataComp](https://www.datacomp.ai/)** : benchmark pour la curation de datasets multimodaux. [Paper](https://arxiv.org/abs/2304.14108).
- **[CC3M/CC12M](https://github.com/google-research-datasets/conceptual-captions)** : Conceptual Captions, paires image-legende de Google.

### Video

- **[WebVid](https://github.com/m-bain/webvid)** : 10M clips video-texte du web.
- **[InternVid](https://github.com/OpenGVLab/InternVideo/tree/main/Data/InternVid)** : 7M videos avec descriptions détaillées.

## Datasets d'évaluation

Comment savoir si ton modèle est bon ? Avec des benchmarks. Voici les principaux :

### NLP général

- **[GLUE / SuperGLUE](https://super.gluebenchmark.com/)** : suites de tâches NLP (classification, inference, QA). Les benchmarks classiques.
- **[MMLU](https://huggingface.co/datasets/cais/mmlu)** : 57 domaines de QCM academiques. Le benchmark de référence pour mesurer les connaissances générales d'un LLM. [Paper](https://arxiv.org/abs/2009.03300).
- **HellaSwag, WinoGrande, ARC** : raisonnement et bon sens. Disponibles sur [Hugging Face](https://huggingface.co/datasets).

### Code

- **[HumanEval](https://github.com/openai/human-eval)** : 164 problèmes Python (OpenAI). Le benchmark standard pour évaluer la génération de code.
- **[MBPP](https://github.com/google-research/google-research/tree/master/mbpp)** : 974 problèmes Python (Google).
- **[SWE-bench](https://www.swebench.com/)** : 2 294 issues GitHub reelles. Le benchmark le plus réaliste pour évaluer la capacité d'un modèle à résoudre de vrais bugs. [Paper](https://arxiv.org/abs/2310.06770).

### Mathématiques

- **[GSM8K](https://huggingface.co/datasets/openai/gsm8k)** : 8 792 problèmes d'arithmetique. Simple mais discriminant.
- **[MATH](https://github.com/hendrycks/math)** : 12 500 problèmes de competition mathématique. Beaucoup plus difficile. [Paper](https://arxiv.org/abs/2103.03874).

<Callout type="warning" title="Enjeux ethiques et legaux">
Les grands datasets web (Common Crawl, LAION) contiennent inevitablement des contenus problématiques : texte toxique, images protegees par le copyright, données personnelles. Le filtrage ne les éliminé **jamais** complètement. Les proces en cours (artistes vs Stability AI, NYT vs OpenAI) pourraient redéfinir complètement les règles d'utilisation de ces données. A suivre de très près.
</Callout>

## Ou trouver des datasets

- **[Hugging Face Hub](https://huggingface.co/datasets)** : 200 000+ datasets avec viewer intégré. Le premier endroit ou chercher.
- **[Kaggle Datasets](https://www.kaggle.com/datasets)** : datasets et competitions
- **[Papers with Code Datasets](https://paperswithcode.com/datasets)** : datasets associes aux benchmarks et papers
- **[Google Dataset Search](https://datasetsearch.research.google.com/)** : moteur de recherche de datasets
- **[Registry of Open Data on AWS](https://registry.opendata.aws/)** : datasets heberges sur AWS
