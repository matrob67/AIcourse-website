## Les données d'abord

Model size ? Survalué. Données ? Tout. Les meilleures IA = meilleures données, pas plus de paramètres. Connaître les datasets c'est connaître l'IA.

<Callout type="info" title="L'importance de la curation">
Un bon dataset n'est pas simplement un grand dataset. La qualité des données (filtrage, déduplication, diversité) est souvent plus importante que la quantité brute. Les meilleurs datasets sont soigneusement curatés pour éliminer le bruit, les duplicats et les contenus toxiques.
</Callout>

## Datasets de pré-entraînement LLM

### Datasets web

- **Common Crawl** : le plus grand crawl du web (~250 milliards de pages). Source brute pour la plupart des datasets de pré-entraînement.
- **FineWeb** (Hugging Face) : 15T tokens filtrés depuis Common Crawl avec des pipelines de qualité sophistiqués. Référence open-source.
- **FineWeb-Edu** : sous-ensemble éducatif de FineWeb, 1.3T tokens de haute qualité éducative.
- **RedPajama v2** : 30T tokens avec scoring de qualité, réplique ouverte des données LLaMA.
- **Dolma** (Allen AI) : 3T tokens, dataset transparent avec documentation complète de la curation.
- **The Pile** (EleutherAI) : 825 GB de texte diversifié (livres, code, web, articles), historique.
- **C4** (Google) : Common Crawl nettoyé, utilisé pour T5.

### Datasets de code

- **The Stack v2** (Hugging Face/BigCode) : 67.5TB de code source de GitHub dans 619 langages.
- **StarCoderData** : dataset curé pour l'entraînement de StarCoder.

### Datasets multilingues

- **CulturaX** : 6.3T tokens dans 167 langues, nettoyé et dédupliqué.
- **mC4** : Common Crawl multilingue dans 101 langues.
- **OSCAR** : corpus web multilingue filtré et classifié.

<Callout type="tip" title="Transparence des données">
La tendance est à la transparence. Des projets comme Dolma (Allen AI) et FineWeb (Hugging Face) documentent chaque étape de leur pipeline de curation, permettant la reproductibilité et l'audit. Les modèles propriétaires (GPT-4, Claude) ne publient pas leurs données.
</Callout>

## Datasets de fine-tuning et alignement

### Instruction tuning

- **OpenAssistant Conversations (OASST)** : 161 000 messages de conversation humains en 35 langues.
- **Dolly 2.0** (Databricks) : 15 000 instructions écrites par des employés Databricks.
- **FLAN Collection** (Google) : mélange de 1800+ tâches NLP formattées comme des instructions.
- **UltraChat** : 1.5M conversations synthétiques multi-tours.
- **ShareGPT** : conversations réelles avec ChatGPT partagées par les utilisateurs.

### Alignement (préférences)

- **HH-RLHF** (Anthropic) : paires de réponses classées par des humains (helpful, harmless).
- **UltraFeedback** : 64K prompts avec réponses de 17 modèles, annotées par GPT-4.
- **Nectar** : 183K conversations avec classements de réponses.

## Datasets de vision

### Classification

- **ImageNet** : 14M images, 1000 classes. Le benchmark fondateur de la vision par ordinateur.
- **ImageNet-21K** : version étendue avec 14M images et 21 841 classes.
- **CIFAR-10/100** : 60K petites images (32x32), 10 ou 100 classes.

### Détection d'objets

- **COCO (Common Objects in Context)** : 330K images, 80 catégories, annotations de segmentation et de détection.
- **Open Images** (Google) : 9M images avec annotations de détection, segmentation et relations.

### Segmentation

- **ADE20K** : 25K images avec segmentation sémantique dense (150 classes).
- **SA-1B** (Meta) : 11M images avec 1.1 milliard de masques de segmentation (dataset de SAM).

## Datasets multimodaux

### Image-texte

- **LAION-5B** : 5.85 milliards de paires image-texte crawlées du web. Utilisé pour Stable Diffusion.
- **LAION-400M** : sous-ensemble de 400M de paires filtrées.
- **DataComp** : benchmark pour la curation de datasets multimodaux.
- **CC3M/CC12M** : Conceptual Captions, paires image-légende de Google.

### Vidéo

- **WebVid** : 10M clips vidéo-texte du web.
- **InternVid** : 7M vidéos avec descriptions détaillées.

## Datasets d'évaluation

### NLP général

- **GLUE / SuperGLUE** : suites de tâches NLP (classification, inférence, QA).
- **MMLU** : 57 domaines de QCM académiques.
- **HellaSwag, WinoGrande, ARC** : raisonnement et bon sens.

### Code

- **HumanEval** : 164 problèmes Python (OpenAI).
- **MBPP** : 974 problèmes Python (Google).
- **SWE-bench** : 2 294 issues GitHub réelles.

### Mathématiques

- **GSM8K** : 8 792 problèmes d'arithmétique.
- **MATH** : 12 500 problèmes de compétition mathématique.

<Callout type="warning" title="Enjeux éthiques et légaux">
Les grands datasets web (Common Crawl, LAION) contiennent inévitablement des contenus problématiques : texte toxique, images protégées par le copyright, données personnelles. Le filtrage ne les élimine jamais complètement. Les procès en cours (artistes vs Stability AI, NYT vs OpenAI) pourraient redéfinir les règles d'utilisation de ces données.
</Callout>

## Où trouver des datasets

- **Hugging Face Hub** : 200 000+ datasets avec viewer intégré
- **Kaggle** : datasets et compétitions
- **Papers with Code** : datasets associés aux benchmarks
- **Google Dataset Search** : moteur de recherche de datasets
- **Registry of Open Data on AWS** : datasets hébergés sur AWS
