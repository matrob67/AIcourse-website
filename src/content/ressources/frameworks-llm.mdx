## Plus qu'une IA

Un LLM seul = parleur sympa. Ajoutez une base de données ? Un outil de calcul ? Une source de vérité ? Maintenant c'est utile. Les frameworks font le glue.

LangChain règne. LlamaIndex pour RAG. Haystack pour production. DSPy pour optimisation. Choisissez votre poison.

<Callout type="info" title="Le besoin d'orchestration">
Une application LLM typique ne se limite pas à appeler une API. Elle doit : récupérer des documents pertinents (RAG), gérer des conversations multi-tours, appeler des outils externes, gérer les erreurs et les retries, structurer les sorties, et chaîner plusieurs étapes de raisonnement.
</Callout>

## LangChain

### Présentation

LangChain est le framework d'orchestration LLM le plus populaire, créé par Harrison Chase en 2022 :

- **Abstraction des LLMs** : interface unifiée pour OpenAI, Anthropic, local models, etc.
- **Chains** : composition séquentielle d'opérations LLM
- **Agents** : LLMs qui décident quels outils utiliser et quand
- **Retrieval** : intégration avec des bases vectorielles pour le RAG
- **LangGraph** : framework d'agents basé sur des graphes d'état (plus flexible que les chains)

### Composants clés

- **LLM/Chat Models** : wrappers pour tous les fournisseurs de modèles
- **Prompts** : templates de prompts avec variables
- **Output Parsers** : extraction structurée des réponses
- **Tools** : intégration d'outils (recherche web, calcul, APIs)
- **Vector Stores** : intégration avec Pinecone, Weaviate, Chroma, FAISS

### LangSmith et LangGraph

- **LangSmith** : plateforme de monitoring et debugging des applications LLM
- **LangGraph** : framework d'agents stateful avec graphes de flux, plus contrôlable que les agents LangChain classiques

<Callout type="tip" title="LangChain vs LangGraph">
LangChain classique utilise des "chains" séquentielles et des "agents" autonomes. LangGraph offre plus de contrôle avec des graphes d'état explicites, permettant des workflows complexes avec boucles, branchements conditionnels et parallélisme. Pour les applications de production, LangGraph est recommandé.
</Callout>

## LlamaIndex

### Présentation

LlamaIndex (anciennement GPT Index) se spécialise dans le **RAG** et la connexion des LLMs aux données :

- **Data Connectors** : ingestion de données depuis 160+ sources (PDF, bases de données, APIs, Slack, Notion)
- **Indexing** : création d'index vectoriels et structurés sur les données
- **Query Engine** : moteur de requête combinant recherche et génération
- **Agents** : agents de recherche d'information capables de naviguer dans les données

### Points forts

- **Spécialisation RAG** : le framework le plus complet pour le retrieval-augmented generation
- **Data agents** : agents spécialisés dans l'interrogation de données structurées et non-structurées
- **Multi-modal** : support des documents avec images
- **LlamaParse** : parsing intelligent de documents complexes (tableaux, graphiques)

## Haystack

### Présentation

Haystack, développé par deepset (Allemagne), est un framework NLP orienté **pipelines** :

- **Pipelines** : composition modulaire de composants (retrievers, readers, generators)
- **Production-ready** : conçu pour le déploiement en production
- **Flexible** : support de nombreux backends (Elasticsearch, FAISS, Weaviate)
- **Évaluation** : outils intégrés d'évaluation des pipelines RAG

### Haystack 2.0

La version 2.0 a été réécrite avec une architecture de pipelines plus flexible :
- Composants connectables via input/output typés
- Support natif des LLMs (OpenAI, Anthropic, local)
- Intégration avec les modèles Hugging Face
- Pipeline builder visuel

<Callout type="paper" title="Haystack vs LangChain">
Haystack se distingue par son approche plus structurée et production-ready. Là où LangChain offre une grande flexibilité (parfois au prix de la complexité), Haystack impose une structure de pipeline claire qui facilite le déploiement et la maintenance.
</Callout>

## DSPy

### Présentation

DSPy (Declarative Self-improving Language Programs), créé par Omar Khattab (Stanford), adopte une approche radicalement différente :

- **Programmation déclarative** : on déclare la structure du programme, pas les prompts
- **Optimisation automatique** : les prompts et les exemples few-shot sont optimisés automatiquement
- **Modules composables** : ChainOfThought, ReAct, RAG comme modules composables
- **Compilateurs** : transforment un programme déclaratif en prompts optimisés

### Le paradigme DSPy

Au lieu d'écrire manuellement des prompts, DSPy :
1. Définit des **signatures** (entrées/sorties attendues)
2. Compose des **modules** (CoT, RAG, multi-hop)
3. **Compile** le programme avec un optimiseur qui trouve les meilleurs prompts et exemples
4. Évalue sur un jeu de données de développement

### Avantages

- **Pas de prompt engineering manuel** : l'optimiseur trouve les meilleurs prompts
- **Robustesse** : les programmes sont moins fragiles que les prompts manuels
- **Portabilité** : changement de modèle sans réécrire les prompts
- **Reproductibilité** : les programmes sont des objets Python vérifiables

<Callout type="warning" title="Maturité des frameworks">
L'écosystème des frameworks LLM évolue très rapidement. Les API changent fréquemment, de nouveaux frameworks apparaissent, et les meilleures pratiques se redéfinissent constamment. Il est important de suivre les mises à jour et de ne pas s'attacher trop rigidement à un seul framework.
</Callout>

## Choisir le bon framework

| Besoin | Framework recommandé |
|--------|---------------------|
| RAG et recherche | LlamaIndex |
| Agents complexes | LangGraph |
| Pipeline de production | Haystack |
| Optimisation de prompts | DSPy |
| Prototypage rapide | LangChain |
| Application simple | Appel direct à l'API |

## Autres outils notables

- **Semantic Kernel** (Microsoft) : framework d'orchestration pour .NET et Python
- **CrewAI** : framework d'agents multi-rôles collaboratifs
- **AutoGen** (Microsoft) : framework d'agents conversationnels multi-agents
- **Instructor** : extraction structurée de données depuis les LLMs
- **Outlines** : génération structurée (JSON, regex) avec des LLMs
