## Le rêve des challengers : battre NVIDIA

Les GPU NVIDIA dominent, mais plusieurs startups ont pris des **paris architecturaux radicaux**. Pas pour remplacer NVIDIA (trop dur), mais pour dominer des niches spécifiques : inférence ultra-fast ou entraînement sans parallélisme complexe.

<Callout type="info" title="Marché énorme, mais concentré">
$50 milliards de marché de l'IA en 2024. NVIDIA prend 80-90%. Les alternatives partagent le reste, mais pour le 10-20% libre, c'est quand même de la vraie concurrence.
</Callout>



## Groq LPU : l'inférence déterministe

### Le concept : pas de caches, tout prévu

Le **LPU** (Language Processing Unit) dit : "pourquoi les GPU sont complexes ? Éliminons tout non-déterminisme." Pas de caches dynamiques, pas de prédiction de branche. **Tout est planifié à la compilation.**

Le compilateur Groq fait tout le planning : chaque opération, chaque mouvement de données. C'est comme du VLSI scheduling extrême.

### Architecture réelle

- **Tensor Streaming Processor** : les données coulent dans un pipeline
- **SRAM on-chip uniquement** : 230 Mo, aucune HBM
- **Pas de caches** : tout est prédictible
- **Performance** : ~750 TOPS (INT8)
- **Latence** : ultra-basse, prévisible

### Le résultat incroyable

Groq a montré : >500 tokens/sec sur Llama 2 70B. C'est fou. La latence est elle-même 100-500ms au lieu de secondes. Pourquoi ? Le modèle complet tient en SRAM distribué. Zéro goulot d'étranglement mémoire HBM.

<Callout type="tip" title="Le tradeoff">
Vous devez **distribuer le modèle sur des centaines de LPU** pour le faire rentrer. Llama 70B = centaines de puces. Et c'est optimisé pour l'inférence, pas l'entraînement. Niche : très spécifique.
</Callout>

## Cerebras : le pari suprême, une wafer entière

Cerebras a fait le **choix le plus radical** : utiliser la **wafer complète** de silicium comme une seule puce.

### Les nombres

**WSE-3 (2024)**:
- 4 trillions de transistors
- 900 000 cœurs AI
- 44 Go SRAM on-chip
- 214 Pb/s bande passante fabric interne

Comparez au H100 : ~800 mm² vs 46 225 mm² (wafer complet).

### Architecture : une grille de cœurs

Pas de hiérarchie classique. Une **grille 2D de cœurs**, chacun avec sa mémoire SRAM locale, connectés par un fabric ultra-rapide. Les données circulent directement entre cœurs voisins, pas via une mémoire centrale.

<Callout type="paper" title="L'avantage wafer-scale">
Élimine le goulot d'étranglement mémoire. 44 Go SRAM avec centaines de Po/s de bande passante = ratio compute/mémoire radicalement meilleur. Les modèles qui tiennent en SRAM sont servis à des vitesses impossibles ailleurs.
</Callout>

### Cas d'usage

- **Entraînement sans parallélisme complexe** : un CS-3 = ce qu'il faudrait distribuer sur 50 GPU normaux
- **Inférence haute perf** : pour les modèles qui tiennent en mémoire

## Graphcore IPU : le calcul massivement parallèle

L'**IPU** (Intelligence Processing Unit) anglais, basé sur **BSP** (Bulk Synchronous Parallel).

### Architecture

- 1 472 **tiles** (Mk2), chacun pouvant faire 6 threads
- SRAM distribuée (~900 Mo on-chip)
- Fabric all-to-all entre tiles
- Modèle MIMD (Multiple Instruction, Multiple Data)

### Le modèle BSP : phases alternées

**Calcul** : chaque tile travaille sur ses données. **Communication** : tiles échangent via le fabric. **Synchronisation** explicite. C'est très différent des GPU où la communication et calcul se chevauchent.

<Callout type="warning" title="Graphcore : l'avenir incertain">
Techniquement innovant, mais des difficultés commerciales. Acquise par SoftBank en 2024. Le statut de l'IPU reste flou. Ça illustre comment l'écosystème CUDA c'est trop puissant : même une bonne tech peut échouer face à la dépendance.
</Callout>

## Les autres

### AMD MI300X
- 192 Go HBM3 (énorme !)
- ROCm ecosystem (compatible CUDA via HIP)
- Principal challenger GPU

### Intel Gaudi 3
- 128 Go HBM2e
- Écosystème Intel
- Prix compétitif

### SambaNova SN40L
- Dataflow reconfigurable
- Jusqu'à 1.5 To de mémoire
- Pour inférence d'énormes modèles en entreprise

## Comment choisir

- **Écosystème** : NVIDIA=imbattable grâce à CUDA
- **Cas d'usage** : inférence pure (Groq), entraînement simplifié (Cerebras), général (NVIDIA/AMD)
- **Budget** : AMD/Intel souvent 30-40% moins cher
- **Réduction du lock-in** : alternatives réduisent la dépendance
- **Disponibilité** : NVIDIA/AMD chez tous les cloud, autres non

La vraie réponse : NVIDIA domine sur la généralité, mais pour des niches (inférence ultra-rapide, entraînement simplifié) les alternatives brillent. Et ça pousse l'innovation.
