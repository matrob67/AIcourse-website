## Pourquoi le cloud pour entraîner des modèles ?

Un GPU H100 coûte 40 000 dollars. Vous n'allez pas l'acheter pour un side project. Le cloud : vous louez les GPU à l'heure, vous payez que ce que vous utilisez. 30 dollars par heure est cher ? Oui. Mais seulement pour 2 heures ? C'est 60 dollars au lieu de 40 000.

Les trois géants (AWS, Google, Microsoft) proposent des plateformes qui gèrent tout le cycle de vie : préparation data, entraînement, évaluation, deployment, monitoring. C'est complexe mais ça marche.

<Callout type="info" title="Qui a le plus de part de marché?">
AWS : ~31% du cloud global. Azure : ~25%. Google Cloud : ~11%. Mais en IA spécifiquement, c'est plus nuancé. Google a les TPU exclusifs. Azure a OpenAI. AWS a la plus grande base d'utilisateurs.
</Callout>



## AWS SageMaker : l'écosystème maximum

**SageMaker** c'est le fourre-tout AWS pour le ML. C'est venu en 2017 et depuis, c'est devenu énorme.

### Ce qu'il offre

- **Studio** : IDE web basé sur Jupyter. Écrivez du code.
- **Training** : entraînement distribué sur multi-GPU/nœud
- **Processing** : pré/post-traitement à l'échelle
- **Endpoints** : déployer en temps réel (inference)
- **Pipelines** : orchestration (CI/CD natif)
- **Feature Store** : stockage centralisé des features
- **Model Registry** : gestion du cycle de vie des modèles
- **Ground Truth** : annotation via workforce humaine

### Avantages

- **Écosystème AWS** : intègre nativement S3, Lambda, Compute, IAM, et mille autres services
- **Derniers GPU** : H100, A100, toutes les dernières puces NVIDIA
- **HyperPod** : si un GPU crash en entraînement, le job continue automatiquement (tolérance aux pannes)
- **Mature** : bien documenté, beaucoup d'exemples

### Limites

- **Complexité** : faut apprendre l'écosystème AWS en plus de la plateforme
- **Coûts** : peuvent exploser vite si vous faites pas attention
- **Lock-in** : une fois dans SageMaker, c'est dur de partir vers un autre cloud

<Callout type="tip" title="JumpStart : le raccourci">
SageMaker JumpStart c'est un catalogue de modèles pré-entraînés deployables en un clic. Llama, Mistral, Stable Diffusion, tout est là. Pas besoin de tout faire vous-même.
</Callout>



## Google Cloud Vertex AI : l'expertise IA

**Vertex AI** c'est Google qui dit "on sait faire l'IA, voilà notre plateforme". C'est la fusion de plusieurs services (AI Platform, AutoML). Elle bénéficie de 20 ans de recherche Google en ML.

### Ce qu'il offre

- **Workbench** : notebooks managés (Jupyter)
- **Training** : custom ou distribué, GPU et TPU
- **Prediction** : endpoints avec auto-scaling
- **Pipelines** : orchestration Kubeflow Pipelines (plus léger qu'AWS)
- **Feature Store** : managé
- **Model Garden** : catalogue de modèles (Gemini, Claude, Llama)
- **Search & Conversation** : RAG managé

### Avantages

- **TPU** : exclusif Google, très bon marché pour l'entraînement
- **Gemini natif** : le modèle de Google intégré partout
- **BigQuery ML** : entraîner des modèles directement en SQL
- **Kubeflow** : orchestration robuste sur Kubernetes
- **Multimodal** : support texte, image, vidéo, audio

### Limites

- **Écosystème moins complet** : moins de services que AWS
- **Documentation** : parfois en retard sur les features
- **GPUs** : moins de options que AWS (amélioration en cours)

<Callout type="paper" title="Model Garden c'est smart">
Au lieu de vous imposer les modèles Google, Model Garden propose Google (Gemini) mais aussi Claude d'Anthropic et Llama de Meta. Vous n'êtes pas forcé d'utiliser les modèles Google pour utiliser Vertex AI.
</Callout>



## Azure Machine Learning : le partenaire Microsoft

**Azure ML** intègre tout l'écosystème Microsoft (Office 365, Teams, Power BI) et bénéficie du partenariat stratégique avec OpenAI.

### Ce qu'il offre

- **Studio** : interface web
- **Compute** : instances managées
- **Endpoints** : deployment managed et serverless
- **Pipelines** : orchestration
- **Designer** : drag-and-drop (no-code/low-code)
- **Azure OpenAI** : accès aux modèles OpenAI (GPT-4, DALL-E, Whisper)
- **Prompt Flow** : dev et eval d'applis LLM

### Avantages

- **Azure OpenAI** : accès exclusif à GPT-4 en cloud d'entreprise avec SLA et conformité
- **Microsoft ecosystem** : Power BI, Office, Teams, Dynamics (très integré)
- **Enterprise** : conformité, security, gestion des identités (Azure AD) mature
- **Responsible AI** : dashboard pour fairness et explainabilité
- **Prompt Flow** : outil unique pour dev d'applis LLM avec eval intégré

### Limites

- **Interface** : moins intuitive que les concurrents
- **Innovation** : les trucs cool arrivent parfois plus tard
- **Lock-in Microsoft** : utile si vous êtes déjà in the ecosystem, sinon c'est heavy

## Comparaison : qui choisir ?

| Critère | AWS | GCP | Azure |
|---|---|---|---|
| **Maturité** | Très élevée | Élevée | Élevée |
| **GPU** | H100, A100 | H100, A100 | H100, A100 |
| **Modèles** | Llama, Mistral | Gemini, Claude | GPT-4 |
| **Orchestration** | SageMaker Pipelines | Kubeflow | Azure Pipelines |
| **Enterprise** | Très bon | Bon | Excellent |
| **Prix GPU** | $$$ | $$ (TPU good) | $$$ |
| **Lock-in** | Élevé | Modéré | Élevé |

<Callout type="warning" title="Lock-in c'est réel">
Chaque plateforme crée une dépendance. Les formats de données, les SDK, les pipelines sont spécifiques. Pour minimiser le risque, utilisez des outils open source (MLflow, Kubeflow, DVC) et des formats standards (ONNX, Docker) qui marchent partout.
</Callout>

## Comment choisir en vrai

**Vous utilisez déjà AWS ?** → SageMaker
**Vous utilisez déjà Microsoft ?** → Azure ML + OpenAI
**Vous voulez la meilleure tech IA** → Vertex AI (TPU, Gemini)
**Vous ne savez pas ?** → Multi-cloud strategy. Utilisez chacun pour ses forces.

Beaucoup de grandes boîtes utilisent les trois. AWS pour la plupart des trucs, GCP pour l'IA, Azure pour Microsoft Office.

L'important : ne vous perdez pas dans l'outil. Le vrai travail c'est vos données et vos modèles, pas la plateforme.
