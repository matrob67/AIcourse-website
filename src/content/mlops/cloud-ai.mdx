## Pourquoi le cloud pour entraîner des modèles ?

Un GPU H100 coûte environ 40 000 dollars. Tu ne vas pas l'acheter pour un side project. Le cloud change la donne : tu loues les GPU à l'heure, tu ne paies que ce que tu utilises. 30 dollars par heure, ça paraît cher ? Oui. Mais seulement pendant 2 heures, c'est 60 dollars au lieu de 40 000. C'est comme louer une Ferrari pour une journée au lieu de l'acheter.

Les trois géants du cloud (AWS, Google, Microsoft) proposent des plateformes qui gèrent tout le cycle de vie ML : préparation des données, entraînement, évaluation, déploiement, monitoring. C'est complexe, parfois frustrant, mais ça fonctionne. Et surtout, ça t'évite de gérer toi-même des serveurs GPU (ce qui est un cauchemar en soi).

<Callout type="info" title="Parts de marché cloud en 2025">
AWS : ~31% du cloud global. Azure : ~25%. Google Cloud : ~11%. Mais en IA spécifiquement, la donne est plus nuancée. Google a les TPU exclusifs (voir le chapitre dédié). Azure a un partenariat exclusif avec OpenAI. AWS a la plus grande base d'utilisateurs existants. Chacun a ses atouts uniques.
</Callout>

## AWS SageMaker : l'écosystème maximum

**SageMaker** c'est le couteau suisse ML d'Amazon, lancé en 2017. Depuis, il n'a pas arrêté de grossir : aujourd'hui c'est une plateforme tentaculaire avec des dizaines de sous-services.

### Ce qu'il offre

- **Studio** : un IDE web basé sur Jupyter pour écrire et exécuter du code
- **Training** : entraînement distribué sur plusieurs GPU et plusieurs machines
- **Processing** : pré/post-traitement de données à grande échelle
- **Endpoints** : déploiement d'APIs d'inférence en temps réel avec auto-scaling
- **Pipelines** : orchestration de workflows ML (CI/CD natif)
- **Feature Store** : stockage centralisé et versionné des features
- **Model Registry** : gestion du cycle de vie des modèles (staging, production, archivage)
- **Ground Truth** : annotation de données via une workforce humaine managée

### Avantages

- **Écosystème AWS immense** : intègre nativement S3, Lambda, IAM (Identity and Access Management), CloudWatch, et les mille autres services AWS. Si ton entreprise est déjà sur AWS, tout se connecte naturellement.
- **Derniers GPU disponibles** : H100, A100, et toutes les dernières puces NVIDIA dès leur sortie
- **HyperPod** : si un GPU crash pendant un entraînement de plusieurs jours, le job reprend automatiquement depuis le dernier checkpoint. Pour des entraînements de modèles massifs, ça évite de perdre des milliers de dollars.
- **Maturité** : beaucoup de documentation, d'exemples, et une grande communauté

### Limites

- **Complexité** : il faut apprendre l'écosystème AWS en plus de la plateforme ML. La courbe d'apprentissage est raide.
- **Coûts cachés** : les instances GPU, le stockage, le réseau, les endpoints... la facture peut exploser si tu ne surveilles pas. Un endpoint d'inférence oublié allumé pendant le week-end, ça fait mal.
- **Lock-in** (dépendance fournisseur) : une fois ton pipeline construit sur SageMaker, migrer vers un autre cloud demande un effort considérable

<Callout type="tip" title="JumpStart : le raccourci">
SageMaker JumpStart est un catalogue de modèles pré-entraînés déployables en quelques clics : Llama, Mistral, Stable Diffusion, et bien d'autres. Pas besoin de tout construire toi-même. C'est le point d'entrée le plus rapide si tu veux tester un modèle sans te soucier de l'infrastructure.
</Callout>

## Google Cloud Vertex AI : l'expertise IA native

**Vertex AI** c'est la plateforme ML de Google, née de la fusion de plusieurs services plus anciens (AI Platform, AutoML). Elle bénéficie de 20 ans de recherche Google en ML : TensorFlow, les TPU, et l'expérience de servir des milliards de requêtes par jour.

### Ce qu'il offre

- **Workbench** : notebooks managés (Jupyter) avec GPU pré-configurés
- **Training** : entraînement custom ou distribué, sur GPU et TPU
- **Prédiction** : endpoints d'inférence avec auto-scaling intelligent
- **Pipelines** : orchestration basée sur Kubeflow Pipelines
- **Feature Store** : managé et intégré à BigQuery
- **Model Garden** : catalogue de modèles variés (Gemini, Claude, Llama, Mistral)
- **Search & Conversation** : RAG managé pour construire des chatbots avec vos propres données

### Avantages

- **TPU** : exclusifs à Google, souvent moins chers que les GPU NVIDIA pour l'entraînement à grande échelle. C'est un vrai avantage compétitif.
- **Gemini natif** : le modèle flagship de Google intégré partout dans la plateforme
- **BigQuery ML** : entraîner des modèles directement en SQL, sans écrire de Python. Idéal pour les data analysts qui veulent faire du ML sans changer d'outil.
- **Kubeflow** : orchestration robuste et open source sur Kubernetes
- **Model Garden ouvert** : au lieu de vous forcer à utiliser uniquement les modèles Google, le catalogue propose aussi Claude (Anthropic), Llama (Meta), et d'autres

### Limites

- **Écosystème global moins complet** : AWS a plus de services annexes (mais Google rattrape son retard)
- **Documentation** : parfois en retard sur les dernières features
- **Moins de choix GPU** : historiquement moins d'options NVIDIA que AWS (en amélioration)

<Callout type="paper" title={"Model Garden : l'approche ouverte de Google"}>
Au lieu de t'imposer ses propres modèles, Google Vertex AI propose un catalogue diversifié : Gemini (Google), Claude (Anthropic), Llama (Meta), Mistral, et d'autres. Tu n'es pas forcé d'utiliser les modèles Google pour profiter de l'infrastructure Google. C'est une approche plus ouverte que ce qu'on attend d'un géant du cloud.
</Callout>

## Azure Machine Learning : le monde Microsoft

**Azure ML** s'intègre dans tout l'écosystème Microsoft (Office 365, Teams, Power BI, Active Directory) et bénéficie du partenariat stratégique avec OpenAI. Si votre entreprise vit dans l'univers Microsoft, Azure ML est le choix naturel.

### Ce qu'il offre

- **Studio** : interface web pour gérer vos expériences, modèles et endpoints
- **Compute** : instances managées avec GPU, auto-scaling
- **Endpoints** : déploiement managé et serverless
- **Pipelines** : orchestration de workflows ML
- **Désigner** : une interface drag-and-drop pour construire des pipelines sans code
- **Azure OpenAI Service** : accès aux modèles OpenAI (GPT-5.4, DALL-E, Whisper) avec les garanties entreprise
- **Prompt Flow** : un outil dédié au développement et à l'évaluation d'applications LLM

### Avantages

- **Azure OpenAI** : accès exclusif à GPT-5.4 dans un cloud d'entreprise avec SLA (Service Level Agreement, garantie de disponibilité), conformité réglementaire, et résidence des données. Pour les entreprises qui veulent GPT-5.4 sans envoyer leurs données chez OpenAI directement, c'est la solution.
- **Écosystème Microsoft** : Power BI pour la visualisation, Office pour les rapports, Teams pour la collaboration, Azure AD pour la gestion des identités. Tout est intégré.
- **Enterprise-ready** : conformité (SOC2, HIPAA, RGPD), sécurité, audit logs. Azure ML coche toutes les cases compliance des grandes entreprises.
- **Responsible AI Dashboard** : outils intégrés pour mesurer l'équité, l'explicabilité et la fiabilité de tes modèles
- **Prompt Flow** : un outil unique pour développer, tester et évaluer des applications LLM avec des métriques intégrées

### Limites

- **Interface** : moins intuitive que les concurrents, parfois labyrinthique
- **Innovation ML** : les dernières fonctionnalités IA arrivent parfois avec un peu de retard par rapport à GCP
- **Lock-in Microsoft** : très intéressant si tu es déjà dans l'écosystème, moins pertinent sinon

## Comparaison : qui choisir ?

| Critère | AWS SageMaker | GCP Vertex AI | Azure ML |
|---|---|---|---|
| **Maturité** | Très élevée | Élevée | Élevée |
| **GPU disponibles** | H100, A100, tous | H100, A100 + TPU | H100, A100 |
| **Modèles phares** | Llama 4, Mistral | Gemini 3.1 Pro, Claude | GPT-5.4 (exclusif) |
| **Orchestration** | SageMaker Pipelines | Kubeflow Pipelines | Azure Pipelines |
| **Enterprise/Compliance** | Très bon | Bon | Excellent |
| **Coût GPU** | $$$ | $$ (TPU avantageux) | $$$ |
| **Lock-in** | Élevé | Modéré | Élevé |

<Callout type="warning" title="Le lock-in cloud est un vrai risque">
Chaque plateforme crée une dépendance progressive. Les formats de données, les SDK, les pipelines sont spécifiques. Plus tu investis dans un cloud, plus il est coûteux d'en changer. Pour minimiser le risque : utilise des outils open source portables (MLflow, Kubeflow, DVC), des formats standards (ONNX pour les modèles, Docker pour les environnements), et garde une couche d'abstraction entre ton code et les services cloud.
</Callout>

## Comment choisir en pratique

**Tu utilises déjà AWS pour le reste ?** SageMaker est le choix naturel. L'intégration avec tes services existants justifie le choix.

**Tu es dans l'univers Microsoft (Office, Teams, AD) ?** Azure ML + Azure OpenAI. La synergie est réelle.

**Tu veux la meilleure infrastructure IA pure ?** Vertex AI avec les TPU et Gemini. Google a un avantage technologique en IA.

**Tu ne sais pas et tu pars de zéro ?** Commence par celui qui t'offre le plus de crédits gratuits. Les trois proposent des programmes de crédits pour démarrer.

Beaucoup de grandes entreprises utilisent les trois clouds en parallèle : AWS pour l'infrastructure générale, GCP pour l'IA et le data, Azure pour l'écosystème Office. Le multi-cloud, c'est complexe à gérer, mais ça évite de mettre tous ses œufs dans le même panier.

L'important : ne te perds pas dans les outils. Le vrai travail, c'est tes données et tes modèles, pas la plateforme qui les héberge.
