## Pourquoi ça vous concerne

Vous entraînez un modèle avec des données personnelles ? Vous avez des utilisateurs européens ? Le **RGPD** s'applique. Vous avez un système d'IA à haut risque ? L'**EU AI Act** s'applique.

Ne pas s'y conformer = amendes jusqu'à 20 millions € (RGPD) ou 35 millions € (AI Act), plus les dégâts réputationnels.

<Callout type="info" title="C'est aussi un plus business">
Les clients enterprise exigent de plus en plus : "Avez-vous la conformité RGPD ? ISO 27001 ? SOC2 ?" C'est un prérequis pour les gros contrats. La gouvernance c'est pas juste légal, c'est compétitif.
</Callout>

## RGPD : les bases

Le **RGPD** = droit européen sur les données personnelles. Clairement applicable si vous traitez des données de residents européens.

### Les principes

- **Licéité** : vous avez une base légale ? Consentement, intérêt légitime, etc.
- **Finalité limitée** : données collectées pour une finalité ≠ réutilisées pour une autre sans base légale
- **Minimisation** : collecter que ce qu'il faut
- **Exactitude** : les données doivent être correctes
- **Rétention limitée** : pas de stockage infini
- **Sécurité** : protégez contre les accès non-autorisés

### Les droits des individus

- **Droit d'accès** : "quelles données vous avez sur moi ?"
- **Rectification** : "corrigez ma donnée"
- **Effacement** (droit à l'oubli) : "supprimez mes données"
- **Portabilité** : "donnez-moi mes données en format structuré"

### Le problème du machine unlearning

Le droit à l'oubli pose un défi : si quelqu'un demande la suppression, comment faire "oublier" ça au modèle ? Le "machine unlearning" existe mais pas de solution satisfaisante. **La prévention est meilleure** : ne passez pas de données personnelles dans l'entraînement.

<Callout type="warning" title="Le droit à l'oubli c'est VRAI">
Et c'est dur à implémenter. Prévénez : anonymisez les données avant l'entraînement, ou ne les mettez pas du tout.
</Callout>



## EU AI Act : le nouveau cadre

L'**AI Act** (2024, appliqué progressivement 2025-2027) c'est le **premier cadre légal global pour l'IA**.

### Classification par risque

1. **Risque inacceptable** (INTERDIT) : scoring social, manipulation subliminale, reconnaissance faciale en temps réel dans les espaces publics
2. **Haut risque** : recrutement, crédit, justice, santé, éducation. Obligations strictes.
3. **Risque limité** : chatbots, deepfakes. Obligations de transparence.
4. **Risque minimal** : aucune obligation spécifique

### Obligations pour les modèles fondation (GPAI)

Pour GPT-4, Claude, Llama, etc. :

- **Documentation** : description technique, données d'entraînement
- **Respect droit d'auteur** : politique pour les données d'entraînement
- **Résumé des données** : publication des données d'entraînement utilisées
- **Risque systémique** (>10^25 FLOPS): évaluation de modèle, tests adversariaux, signalement incidents

En pratique : les plus gros modèles (GPT-4, Claude) sont concernés. Les modèles 7B-70B généralement sous le seuil.

<Callout type="paper" title="Le seuil de risque systémique">
10^25 FLOPS d'entraînement = GPT-4 level. C'est ÉNORME. La plupart des modèles n'atteindront pas ce seuil. Mais si vous entraînez un monstre, attention.
</Callout>

## Traçabilité des données (Data Lineage)

### Qu'est-ce que c'est

Le **data lineage** trace le chemin complet : d'où les données viennent, ce qui leur est arrivé, où elles sont utilisées.

### Ce qu'il faut tracer

- **Provenance** : web scraping ? Base de données ? Données achetées ?
- **Transformations** : nettoyage ? Filtrage ? Augmentation ?
- **Consentement** : base légale pour utilisation
- **Accès** : qui y a accédé, quand
- **Utilisation** : dans quels modèles

### Outils

- **Apache Atlas** : gouvernance et lineage
- **OpenLineage** : standard ouvert
- **Amundsen / DataHub** : catalogues de données
- **dbt** : lineage pour SQL

<Callout type="tip" title="Model Cards et Data Cards">
Documentez vos modèles et datasets. HuggingFace les a popularisées. L'AI Act pourrait les rendre obligatoires pour certains systèmes. C'est pas juste du virtue signaling, c'est pratique.
</Callout>

## Bonne gouvernance de données pour l'IA

### Inventaire

- Cataloguez toutes les sources de données
- Documentez la base légale
- Identifiez les données personnelles
- Classifiez par sensibilité

### Protection

- **PII detection** : scannez auto vos datasets pour trouver emails, numéros de téléphone, etc.
- **PII removal** : supprimez avant l'entraînement
- **Pseudonymisation** : remplacez les identifiants
- **Differential privacy** : techno math pour garantir extraction impossible

### Qualité

- Tests automatisés pour schéma, distributions, complétude
- Monitoring continu en production
- Data cards pour documenter

### Org

- Nommez un DPO (Data Protection Officer)
- Formez les équipes ML
- Intégrez conformité dans CI/CD
- Audits réguliers

<Callout type="warning" title="Les amendes réelles">
RGPD : jusqu'à 20 millions € ou 4% du chiffre d'affaires global.
AI Act : jusqu'à 35 millions € ou 7% du chiffre d'affaires.

Ce ne sont pas des bluffs. Des boîtes ont payé des millions pour RGPD.
</Callout>
