## Trois modes pour le prix d'un

Voici un problème que chaque chercheur en NLP affronte : à l'entraînement, on veut traiter toute la séquence en parallèle (c'est rapide). À l'inférence, on génère token par token (on veut que ce soit efficace). Et parfois, on veut un compromis — traiter un prompt long de manière semi-parallèle avant de générer.

La plupart des architectures choisissent un mode et s'y tiennent. **RetNet**, proposé par Microsoft Research, a une ambition plus grande : supporter nativement **trois modes de calcul** qui sont mathématiquement équivalents.

## Le mécanisme de Retention

### L'idée centrale : attention avec décroissance exponentielle

RetNet remplace l'attention par la **retention** — un mécanisme qui ressemble à l'attention mais avec une différence clé : les poids d'attention sont multipliés par un facteur de décroissance exponentielle selon la distance entre les tokens.

```
Retention(X) = (Q * K_transposé ⊙ D) * V
```

Où `D` est une matrice de masquage causal avec décroissance :

```
D[n,m] = gamma^(n-m)  si n >= m (token n est après token m)
D[n,m] = 0            si n < m  (masquage causal)
```

Le paramètre `gamma` (entre 0 et 1) contrôle la vitesse de décroissance. Avec `gamma = 0,99`, un token à 100 positions de distance aura un poids multiplié par `0,99^100 ≈ 0,37` — soit moins de la moitié du poids original.

<Callout type="info" title="Intuition de la retention">
Imaginez lire un texte et mémoriser les phrases précédentes : les phrases juste au-dessus sont très fraîches dans votre esprit, celles de trois paragraphes plus haut un peu moins, et celles du début du chapitre encore moins. La retention encode mathématiquement cette décroissance naturelle de l'attention avec la distance — c'est un biais inductif raisonnable pour la plupart des textes.
</Callout>

### Encodage de position dans l'espace complexe

RetNet utilise une rotation complexe similaire à **RoPE** (Rotary Position Embedding) pour encoder les positions :

```
Q = (X * W_Q) ⊙ Theta
K = (X * W_K) ⊙ Theta_conjugué
```

Où `Theta` contient des éléments `exp(i*n*theta)` — des nombres complexes qui tournent selon la position `n`. Cette formulation encode l'information positionnelle directement dans le mécanisme de retention, sans encodage positionnel séparé.

## Les trois paradigmes de calcul

C'est la vraie valeur ajoutée de RetNet.

### Mode parallèle : pour l'entraînement

En mode parallèle, la retention se calcule comme une multiplication matricielle standard, similaire à l'attention :

```
Retention_parallèle(X) = (Q * K_transposé ⊙ D) * V
```

Complexité : O(L² d), identique à l'attention standard. Parfaitement parallélisable sur GPU.

### Mode récurrent : pour l'inférence token par token

En mode récurrent, la retention se reformule comme une récurrence avec un **état matriciel** `s` :

```
s_n = gamma * s_(n-1) + k_n_transposé * v_n   # mise à jour de l'état
o_n = q_n * s_n                                 # sortie
```

L'état `s_n` est une **matrice de taille d x d** qui accumule les associations clé-valeur pondérées par la décroissance. Chaque nouveau token se calcule en O(d²), indépendamment de la longueur de la séquence.

<Callout type="tip" title="Équivalence exacte entre les modes">
Contrairement à certaines approximations (comme l'attention linéaire qui approxime le softmax), les trois modes de RetNet sont **mathématiquement identiques**. Le résultat numérique est exactement le même, que vous utilisiez le mode parallèle, récurrent, ou chunkwise. On peut s'entraîner en mode parallèle et inférer en mode récurrent sans aucune perte de précision — ce n'est pas une approximation.
</Callout>

### Mode chunkwise : le compromis

Le mode chunkwise divise la séquence en morceaux (chunks) de taille B :

- **Intra-chunk** : calcul parallèle en mode parallèle au sein de chaque morceau
- **Inter-chunk** : propagation récurrente de l'état entre morceaux

```
Retention_chunk(X_i) = (Q_i * K_i_transposé ⊙ D) * V_i + (Q_i * S_i) ⊙ xi
```

Ce mode est particulièrement utile pour traiter un long prompt (phase de prefill) : plus rapide que la récurrence pure, moins gourmand en mémoire que le mode parallèle sur une très longue séquence.

## Multi-Scale Retention : plusieurs échelles de temps

Comme le Multi-Head Attention utilise plusieurs têtes, RetNet utilise plusieurs têtes de retention avec des **taux de décroissance différents** :

```
gamma_h appartient à {1 - 2^(-5), 1 - 2^(-6), ..., 1 - 2^(-5-h)}
```

<Callout type="paper" title="Multi-Scale Retention">
Différentes valeurs de `gamma` permettent aux têtes de se spécialiser : une tête avec `gamma = 0,999` (décroissance très lente) capture les dépendances à longue portée — elle "se souvient" bien des tokens lointains. Une tête avec `gamma = 0,9` (décroissance rapide) se concentre sur les interactions locales. Ensemble, elles couvrent plusieurs échelles temporelles simultanément.
</Callout>

### Architecture complète d'un bloc RetNet

Chaque bloc RetNet contient :
1. Multi-Scale Retention (MSR) avec normalisation GroupNorm
2. Couche feed-forward avec gate SwiGLU
3. Connexions résiduelles et normalisation

## Comparaison avec les architectures concurrentes

| Propriété | Transformer | RetNet | RWKV | Mamba |
|-----------|-------------|--------|------|-------|
| Mode entraînement | Parallèle | Parallèle | Parallèle | Scan parallèle |
| Inférence par token | O(L) (KV cache) | O(1) état fixe | O(1) état fixe | O(1) état fixe |
| Taille de l'état | KV cache = O(L*d) | Matrice O(d²) | Vecteur O(d) | Vecteur O(N*d) |
| Mode chunkwise natif | Non | Oui | Possible | Oui (Mamba-2) |

L'état matriciel d x d de RetNet est plus grand que l'état vectoriel de RWKV ou Mamba — ce qui lui donne potentiellement plus de capacité de mémorisation, au prix d'une mémoire d'inférence plus élevée.

## Forces et limitations

### Ce que RetNet fait bien

- **Triple paradigme** : flexibilité unique, on choisit le mode selon le contexte d'utilisation
- **Pas de KV cache** : mémoire d'inférence fixe, indépendante de la longueur du contexte
- **Équivalence mathématique** : pas de compromis de précision entre les modes
- **Multi-échelle** : les différents gamma capturent différentes portées temporelles

<Callout type="warning" title="Le coût de l'état matriciel">
L'état récurrent de RetNet est une matrice d x d par tête. Pour une dimension cachée d = 4 096, chaque tête stocke une matrice de 16 millions d'éléments. Multiplié par 32 têtes et 32 couches, c'est ~16 milliards d'éléments en précision float16 — soit ~32 GB juste pour l'état. Beaucoup plus que RWKV ou Mamba, même si c'est toujours fixe en longueur de séquence.
</Callout>

### Ce que RetNet fait moins bien

- **Décroissance fixe** : les taux gamma sont fixes, pas data-dependent (contrairement à Mamba). Le modèle ne peut pas décider dynamiquement de mémoriser plus longtemps un token important.
- **Pas de sélectivité** : c'est un système LTI (temps-invariant), comme S4 avant Mamba.
- **Adoption limitée** : peu de modèles de production utilisent RetNet, contrairement à Mamba ou RWKV.

## Héritage de RetNet

RetNet a influencé plusieurs développements. Le framework SSD de Mamba-2, qui unifie les vues parallèle et récurrente des SSM, est conceptuellement proche du triple paradigme de RetNet. Le concept de "calcul chunkwise" est devenu un critère important pour évaluer les nouvelles architectures.

<Callout type="tip" title="Où se situe RetNet dans l'écosystème ?">
RetNet occupe un espace intermédiaire entre les Transformers (expressivité maximale, coût élevé) et les RNN classiques (efficacité maximale, expressivité limitée). Il offre un compromis pragmatique avec une garantie d'équivalence mathématique entre ses modes, ce qui le rend attrayant pour les déploiements qui doivent optimiser à la fois l'entraînement et l'inférence.
</Callout>
