## L'observation clé

Pas tous les poids ne sont égaux. Si vous regardez **ce qui entre** dans une couche (les activations), vous pouvez deviner quels **poids** sont critiques. C'est simple mais puissant.

<ArxivLink id="2306.00978" title="AWQ: Activation-Aware Weight Quantization (Lin et al., 2023)" />

**L'idée en une phrase :** Si un canal d'activation est toujours gros, les poids correspondants sont importants. Protégez-les.

<Callout type="info" title="L'intuition">
C'est comme identifier les piliers critiques d'un bâtiment en voyant la charge qu'ils supportent. Les gros piliers doivent rester forts, les petits peuvent être plus fragiles.
</Callout>

## Comment ça marche

### Étape 1 : Mesurer la saillance

Pour chaque canal de poids, mesurez la **magnitude moyenne des activations** qui le traversent :

```
saillance(canal_j) = moyenne(|activation_j|)
```

Les canaux à forte activation = poids importants.

### Étape 2 : Protéger par scaling

Pour chaque canal important, appliquez un **facteur multiplicateur** avant la quantification :

```
Poids protégés = Poids originaux × scaling_factor
```

Ensuite, vous quantifiez les poids protégés. Les grands nombres sont mieux quantifiés qu'les petits (plus d'étapes de quantification utilisées).

Au passage de l'entrée, vous divisez par le même facteur pour compenser. Mathématiquement, le résultat est identique, mais la quantification est plus précise.

### Étape 3 : Trouver le meilleur scaling

La magie : tester quelques valeurs de scaling et voir laquelle minimise l'erreur finale. Grid search rapide.

<Callout type="paper" title="Pourquoi c'est plus rapide">
Contrairement à GPTQ qui doit inverser une grosse matrice mathématique (très coûteux), AWQ ne fait que mesurer des nombres et tester des valeurs. Résultat : 10-100× plus rapide.
</Callout>



## Avantages

### 1. Qualité supérieure
AWQ produit **meilleure perplexité** que GPTQ à 4 bits, surtout sur les gros modèles (30B+).

### 2. Calibration rapide
Quelques minutes au lieu d'heures. Vous pouvez quantifier en temps réel.

### 3. Robustesse
Moins sensible au choix du dataset de calibration. Les magnitudes d'activation sont une propriété stabile du modèle.

### 4. Kernels optimisés
L'équipe AWQ a créé des kernels CUDA spécialisés. Supportés par vLLM, TGI, et autres frameworks.

## Résultats

| Modèle | Méthode | Bits | Perplexité |
|---|---|---|---|
| LLaMA-7B | FP16 | 16 | 5.68 |
| LLaMA-7B | RTN | 4 | 6.24 |
| LLaMA-7B | GPTQ | 4 | 5.85 |
| LLaMA-7B | **AWQ** | 4 | **5.78** |

AWQ gagne légèrement sur GPTQ, surtout sur les gros modèles.

<Callout type="tip" title="AWQ brille sur les gros modèles">
Sur des modèles 30B+, l'avantage d'AWQ est notable. Sur 7B, c'est marginal. Choisissez AWQ si vous avez un gros modèle et peu de temps.
</Callout>

## Utilisation pratique

```python
from awq import AutoAWQForCausalLM

model = AutoAWQForCausalLM.from_pretrained("mistralai/Mistral-7B")
tokenizer = AutoTokenizer.from_pretrained("mistralai/Mistral-7B")

model.quantize(tokenizer, quant_config={
    "w_bit": 4,
    "q_group_size": 128,
})
model.save_quantized("./mistral-awq")
```

### Avec vLLM (inférence)

```python
from vllm import LLM
model = LLM("./mistral-awq", quantization="awq")
outputs = model.generate(["Bonjour"])
```

## AWQ dans l'écosystème

- **vLLM** : support natif avec kernels optimisés
- **TGI** : support natif
- **TensorRT-LLM** : support via conversion
- **Transformers HF** : support via autoawq

C'est devenu le format de quantification préféré pour la production sur GPU NVIDIA.

<Callout type="warning" title="GPU NVIDIA only">
AWQ n'est optimisé que pour GPU NVIDIA. Pour CPU ou Mac, utilisez GGUF. Les deux formats ciblent des cas d'usage différents.
</Callout>

## Évolutions récentes

### AWQ + FP8
Sur H100 (qui supporte FP8 natif), combinez AWQ (poids 4-bit) avec quantification FP8 des activations. Compression maximale, vitesse maximale.

### Intégration dans les pipelines
AWQ devient le format par défaut dans les systèmes de déploiement production, grâce à son bon équilibre : qualité décente, calibration rapide, inférence très rapide.
