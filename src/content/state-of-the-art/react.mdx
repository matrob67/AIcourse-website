## Le problème : le raisonnement seul hallucine

Le Chain-of-thought (CoT) a été une avancée majeure : demander au LLM de "réfléchir à haute voix" améliore ses réponses. Mais il y a un problème fondamental. Quand le LLM raisonne dans le vide, sans jamais vérifier ses affirmations contre le monde réel, il peut construire des raisonnements parfaitement cohérents mais complètement faux. Il hallucine avec conviction.

Prenons un exemple : vous demandez "Qui a écrit le livre le plus vendu en 2023 ?" Le LLM peut raisonner "Le livre le plus vendu est probablement..." et inventer une réponse plausible. Sans accès à une source fiable, il ne peut que deviner.

**ReAct** (Reasoning + Acting) résout ce problème en combinant réflexion ET action : le LLM réfléchit, puis **agit** (appelle un outil), observe le résultat réel, et réfléchit à nouveau avec cette nouvelle information.

<ArxivLink id="2210.03629" title="ReAct: Synergizing Reasoning and Acting in Language Models (Yao et al., 2022)" />

<Callout type="info" ’>
La réflexion seule peut halluciner : le LLM fabrique des faits. L'action seule est aveugle : l'agent agit sans comprendre pourquoi. ReAct les combine pour obtenir le meilleur des deux mondes : un agent qui réfléchit ET qui vérifie contre la réalité.
</Callout>

## Le cycle : Thought, Action, Observation

Voici ReAct en action sur un exemple concret :

> **Q : Quel âge a le président français actuel ?**
>
> **Thought 1 :** Je dois d'abord chercher qui est le président actuel de la France.
> **Action 1 :** search("président France 2024")
> **Observation 1 :** Emmanuel Macron est président de la France depuis 2017.
>
> **Thought 2 :** Maintenant je connais le président. Pour son âge, j'ai besoin de sa date de naissance.
> **Action 2 :** search("Emmanuel Macron date naissance")
> **Observation 2 :** Emmanuel Macron est né le 21 décembre 1977.
>
> **Thought 3 :** Né en 1977, en 2024 il a 46 ans. J'ai toutes les infos nécessaires.
> **Action 3 :** finish("Emmanuel Macron a 46 ans.")

Remarquez la structure : le LLM ne se contente pas d'agir mécaniquement. Il **explique** ce qu'il fait et pourquoi à chaque étape, ce qui rend le processus transparent et débugable.

### Les trois composants

**Thought** (pensée) : le LLM verbalise son raisonnement. Que sait-il ? Que lui manque-t-il ? Quelle est la prochaine étape logique ? C'est le "cerveau" de l'agent.

**Action** (action) : un appel concret à un outil externe avec des paramètres spécifiques. Recherche web, calculatrice, API, base de données, n'importe quel outil auquel le LLM a accès.

**Observation** (observation) : le résultat réel retourné par l'outil, réinjecté dans le contexte du LLM. C'est l'ancrage dans la réalité, le moment où le LLM confronte ses hypothèses aux faits.

## Pourquoi ça marche si bien

**Ancrage dans la réalité** : les observations forcent le LLM à se confronter aux faits concrets. Chaque affirmation peut être vérifiée, ce qui réduit drastiquement les hallucinations. Le LLM ne "devine" plus, il cherche et vérifie.

**Traçabilité complète** : vous voyez exactement pourquoi le LLM a pris chaque décision. Si le résultat final est faux, vous pouvez remonter le fil : est-ce que le Thought 2 était mal formulé ? Est-ce que l'outil a retourné une mauvaise info ? C'est un avantage énorme pour le debugging.

**Adaptabilité naturelle** : les questions simples se résolvent en 1-2 étapes, les questions complexes en 5-10. Le LLM ajuste automatiquement la profondeur de son raisonnement selon la difficulté. Pas besoin de fixer à l'avance le nombre d'étapes.

<Callout type="paper" title="Résultats empiriques">
ReAct surpasse le CoT seul sur HotPotQA (questions nécessitant de croiser plusieurs sources) et FEVER (vérification factuelle). L'avantage principal : beaucoup moins d'erreurs factuelles grâce à l'ancrage par les observations réelles.
</Callout>

## Implémentation simple

```python
from langchain.agents import create_react_agent
from langchain.tools import Tool

tools = [
    Tool(name="search",
         func=search_fn,
         description="Recherche web"),
    Tool(name="calc",
         func=calc_fn,
         description="Calcul mathématique"),
]

agent = create_react_agent(llm, tools, prompt)
result = agent.invoke({
    "input": "Population France au carré ?"
})
```

Le framework LangChain implémente ReAct nativement. Vous définissez les outils disponibles, le LLM décide quand et comment les utiliser. En coulisses, le prompt système structure le format Thought/Action/Observation.

## Variantes et extensions

**ReAct + auto-réflexion** : après chaque observation, le LLM évalue explicitement si le résultat est satisfaisant ou s'il faut reformuler la recherche.

> **Thought :** Le résultat n'est pas assez précis, je vais reformuler ma requête avec des termes plus spécifiques.

**ReAct + planning** : avant de commencer les actions, le LLM élabore un plan complet, puis le suit étape par étape. Utile pour les tâches complexes avec des dépendances entre étapes.

> **Plan :** 1) Chercher X &rarr; 2) Chercher Y &rarr; 3) Combiner les résultats &rarr; 4) Formuler la réponse

**ReWOO** (Reasoning Without Observation) : approche opposée. Le LLM planifie toutes ses actions d'un coup, les exécute en parallèle, puis synthétise les résultats. Plus rapide (moins d'allers-retours), mais moins adaptatif : si la première action retourne un résultat inattendu, le plan ne s'ajuste pas.

<Callout type="tip" title="ReAct est le point de départ par défaut">
Pour la plupart des agents LLM, ReAct est la bonne architecture de départ. Simple à implémenter, facile à débugger, suffisamment flexible pour la majorité des cas. N'introduisez des alternatives plus complexes (planning, MCTS, multi-agents) que si ReAct montre des limitations concrètes sur votre cas d'usage.
</Callout>

## Les pièges à éviter

**Boucles infinies** : l'agent répète les mêmes actions en boucle sans progresser. C'est le piège le plus courant. Solution : limitez le nombre d'itérations (typiquement 5-10 max) et détectez les répétitions pour forcer une sortie.

**Explosion du coût en tokens** : chaque itération ajoute le Thought, l'Action et l'Observation au contexte. Après 10 itérations, le contexte devient énorme et coûteux. Sur des requêtes complexes, le coût peut être 10-20x celui d'un simple appel.

**Erreurs en cascade** : une mauvaise observation au début (outil qui retourne du bruit, API qui plante) contamine toute la chaîne de raisonnement. Le LLM construit ses Thoughts suivants sur des fondations fausses.

<Callout type="warning" title="La qualité des outils est déterminante">
ReAct est aussi fiable que ses outils. Un moteur de recherche qui retourne du contenu non pertinent, une API qui plante silencieusement, ou un outil mal documenté peuvent saboter tout le raisonnement. Investissez dans des outils robustes avec des messages d'erreur clairs et des réponses structurées.
</Callout>

## L'héritage de ReAct

ReAct est devenu le paradigme fondamental des agents LLM modernes. LangChain, LlamaIndex, AutoGen, CrewAI : tous implémentent une variante de ReAct comme architecture de base. Même les agents les plus sophistiqués (avec planning, mémoire à long terme, multi-agents) construisent sur le cycle Thought-Action-Observation introduit par ReAct. C'est le "HTTP des agents LLM" : le protocole de base sur lequel tout le reste s'appuie.
