## Après o1 : comment on améliore l'amélioration

o1 a brisé les records. Ensuite ? OpenAI a lancé o3 et o4-mini.

La nomenclature ? o2 a été sauté (confusion avec l'opérateur télécom O2). Mais la progression est claire : chaque version est significativement meilleure.

<Callout type="info" title="L'accélération">
Le rythme d'innovation en modèles de raisonnement est incroyable. o1 (2024) → o3 (début 2025) → o4-mini (mi-2025). En 2026, OpenAI continue à pousser cette lignée. Chaque version apporte des gains réels.
</Callout>

## o3 : plus profond, plus multimodal

o3 garde la chaîne de pensée invisible de o1, mais ajoute beaucoup.

### Améliorations

- **Raisonnement plus profond** : peut résoudre des problèmes qui demandent 100+ étapes logiques
- **Tool use en réflexion** : peut utiliser des outils (web search, code execution) pendant qu'il réfléchit, pas seulement avant/après
- **Multimodal nativement** : images dans le raisonnement — comprend et raisonne sur du contenu visuel
- **Meilleure calibration** : sait mieux quand il est certain vs incertain

### Le résultat révolutionnaire : ARC-AGI

ARC-AGI est un benchmark conçu spécifiquement pour le raisonnement abstrait humain — très difficile.

o3 avec un budget de compute élevé obtient des scores **proches du niveau humain**.

Personne ne pensait que c'était possible si tôt.

### Niveaux de compute ajustables

o3 introduit la notion qu'on peut **demander plus ou moins de réflexion** :

- **Low** : réflexion rapide, moins de tokens
- **Medium** : balance standard
- **High** : réflexion approfondie, plein de tokens

Vous choisissez le niveau selon votre problème.

<Callout type="paper" title="L'adaptativité : une avancée clé">
Au lieu de "un seul niveau de réflexion pour tout", avoir Low/Medium/High permet aux problèmes faciles de ne pas gaspiller du compute, tandis que les difficiles en reçoivent assez. C'est vers une vraie intelligence adaptative.
</Callout>

## o4-mini : le raisonnement économique

Pendant que o3 explore la profondeur, o4-mini explore l'efficacité.

o4-mini est conçu pour être **rapide et bon marché**, optimisé spécifiquement pour les problèmes STEM (Science, Technologie, Ingénierie, Maths).

### Positionnement

- Beaucoup moins cher que o3
- Latence beaucoup meilleure
- Performance compétitive sur maths et code
- Parfait pour les applications haute volume

### Forces

- **Maths et code** : presque aussi bon que o3 à une fraction du prix
- **Latence** : réponses plus rapides
- **Intégration** : API identique, facile de migrer de GPT-4o

### Limitations

- Connaissances générales moins vastes
- Raisonnement moins approfondi sur très complexe
- Pas vraiment pensé pour créatif ou conversationnel

## Comparaison : la gamme complète

| Modèle | Coût relatif | Maths | Code | Connaissance | Latence |
|---|---|---|---|---|---|
| GPT-4o | 1x | Bon | Bon | Excellent | Rapide |
| o1-mini | 3x | Très bon | Très bon | Moyen | Moyen |
| o1 | 10x | Excellent | Excellent | Bon | Lent |
| o4-mini | 4x | Excellent | Excellent | Bon | Moyen |

<Callout type="tip" title="Guide décisionnel simple">
Pour la plupart des gens : **GPT-4o**.
Pour maths/code pas cher : **o4-mini**.
Pour raisonnement ultra-complexe : **o3**.
Testez GPT-4o d'abord — montez que si nécessaire.
</Callout>

## L'écosystème des modèles de raisonnement

OpenAI ne joue pas seule.

### Autres modèles de raisonnement propriétaires
- **Google Gemini 2.0 Flash Thinking** : vision + raisonnement
- **Anthropic Claude avec extended thinking** : raisonnement visible

### Open-source
- **DeepSeek-R1** : gratuit, MIT license, compétitif avec o1
- **Qwen-QwQ** (Alibaba) : raisonnement open-source
- **Magistral** (Mistral) : raisonnement européen

Le raisonnement n'est plus monopole d'OpenAI.

## Coin technique : le paradigme "raisonnement comme service"

Les modèles de raisonnement changent la facturation :

Avant : vous payez pour la génération de tokens output.

Maintenant : vous payez aussi pour la "réflexion" — le nombre de tokens que le modèle utilise pour penser.

```
Coût = (input tokens) + (reasoning tokens) + (output tokens)
```

Le coût monte avec la difficulté. Un problème facile = peu de tokens reasoning. Un PhD-level = beaucoup.

### Composition automatique de modèles

Nouveau pattern : router automatique.

```
if problem_looks_simple:
    use_gpt4o()
else:
    use_o3()
```

Optimise le coût global.

<Callout type="warning" title="Les modèles de raisonnement ne sont pas universels">
o3 n'est pas meilleur que GPT-4o partout. Sur conversation naturelle, rédaction créative, instructions simples — GPT-4o gagne. o3 surpense sur les problèmes simples.

Utilisez le bon outil pour le bon job.
</Callout>

## La question clé

Avant : "Quel modèle est le meilleur ?"

Maintenant : "Quel modèle, à ce budget, pour ce problème, dans ce délai, est optimal ?"

Les modèles de raisonnement d'OpenAI ont transformé ça en question d'optimisation plutôt qu'une course simple.
