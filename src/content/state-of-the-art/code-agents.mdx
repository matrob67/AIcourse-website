## Pourquoi le code c'est le terrain idéal pour les agents

Contrairement à la rédaction (subjective) ou l'analyse (ambiguë), le code offre un **feedback objectif** :
- Ça compile ou pas
- Les tests passent ou pas
- Le linter se plaint ou pas

L'agent peut **vérifier son propre travail** automatiquement. Boucle feedback efficace et rapide.

<Callout type="info" title="Le feedback est clé">
Code c'est parfait pour les agents. Pas d'ambiguïté. « Marche ou marche pas ». Ça permet une itération serrée jusqu'à succès.
</Callout>



## Les systèmes pionniers

**Devin** (Cognition Labs, 2024) : présenté comme le « premier ingénieur IA autonome »

- Interface complète (terminal, éditeur, navigateur)
- Résout des issues GitHub de bout en bout
- Score SWE-bench : 13.86% (époque record)
- Planification à long terme, décomposition complexe

**Claude Code** (Anthropic) : CLI pour développement assisté

- Accès direct terminal et système de fichiers
- Compréhension de codebases entiers
- Peut exécuter commandes, tests, créer commits
- Boucle ReAct : lire/écrire fichiers, exécuter commands

**Cursor** : IDE VS Code + IA

- **Tab completion** : suggestions contextuelles
- **Cmd+K** : édition par instruction naturelle
- **Chat** : conversation avec le codebase (indexé)
- **Composer** : modifications multi-fichiers coordonnées
- Support GPT-4, Claude, Mistral

## Architecture typique

### Outils disponibles

1. **File operations** : lire, écrire, éditer
2. **Terminal** : shell commands, installer dépendances
3. **Search** : grep, find, semantic search
4. **Browser** : docs, Stack Overflow
5. **Git** : branches, commits, PRs

### Boucle de développement

```
1. Comprendre → lire l'issue, explorer code
2. Planifier → identifier fichiers à modifier
3. Implémenter → écrire/modifier code
4. Tester → exécuter tests, compiler
5. Itérer → erreurs ? corriger et retester
6. Finaliser → commit, PR, docs
```

<Callout type="tip" title="Contexte = roi">
La qualité dépend principalement de la **compréhension du contexte**. Architecture, conventions, dépendances. Investissez dans l'exploration du code avant d'écrire.
</Callout>

## Stratégies de génération

**Génération complète** : nouveau fichier entier. Adapté pour nouvelles fonctionnalités.

**Édition** : modifier parties d'un fichier existant. Pour corrections et améliorations.

**Diff-based** : générer un patch. Plus efficace en tokens.

**Code-first agents** (Smolagents) : LLM écrit du code Python exécuté directement. Plus expressif et naturel que JSON.

## Benchmarks

### SWE-bench

Benchmark de référence : résoudre des issues GitHub réelles dans des projets open-source populaires.

| Agent | Score (%) |
|---|---|
| GPT-4 direct | ~1.7 |
| Claude 3.5 Sonnet + tools | ~49 |
| Claude Code | ~72 |
| OpenAI Codex Agent | ~69 |

### HumanEval et MBPP

Génération de fonctions individuelles. Moins réaliste mais utiles pour comparer capacités de base.

<Callout type="paper" title="L'écart diminue">
Les meilleurs agents résolvent maintenant des bugs réels dans de vrais projets. Les problèmes complexes (refactoring architectural, patterns) restent difficiles.
</Callout>

## Défis spécifiques

**Compréhension à grande échelle** : codebases = des centaines de milliers de lignes. L'agent doit naviguer intelligemment.

**Tests et validation** : un agent fiable doit écrire et exécuter des tests. Sans ça, risque de régressions silencieuses.

**Sécurité** : accès terminal + fichiers = risques majeurs. Sandboxing, permissions limitées, revue humaine essentiels.

<Callout type="warning" title="Revue humaine indispensable">
Même les meilleurs agents produisent du code incorrect ou suboptimal. La revue humaine reste essentielle, surtout pour la production. Les agents accélèrent, ne remplacent pas l'humain.
</Callout>

## L'avenir des agents de code

Probablement l'application IA la plus mature et impactante. Direction : gérer des tâches toujours plus complexes — refactoring massive, migrations de systèmes, collaboration avec équipes sur projets entiers.

Les agents de code aujourd'hui : capacités de junior. Demain : probablement senior engineer assistant.

Reste qu'une revue humaine reste indispensable pour la qualité et la sécurité.
