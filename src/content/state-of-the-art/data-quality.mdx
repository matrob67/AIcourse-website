## Le mantra : "garbage in, garbage out"

Les données d'entraînement sont le facteur le plus déterminant de la qualité d'un LLM. Plus que la taille du modèle, plus que le nombre de tokens.

Tu peux avoir un modèle énorme, mais si tu l'entraînes sur du contenu spam, dupliqué et bas de gamme, le résultat sera mauvais. À l'inverse, un modèle plus petit entraîné sur des données soigneusement filtrées peut surperformer.

## Le pipeline de traitement des données

```
Données brutes (100 TB)
  ↓ Extraction (nettoyage, conversion formats)
  ↓ Filtrage linguistique (détection langue)
  ↓ Déduplication (30-50% éliminés)
  ↓ Filtrage qualité (classifier-based)
  ↓ Filtrage sécurité (toxicité, PII)
  ↓ Mélange et proportionnement
  ↓ Données finales (5-15 TB, 5-15T tokens)
```

Le ratio de réduction est **5-20x**. Cela souligne l'ampleur du filtrage nécessaire.

<Callout type="info" title="Échelle du travail">
Pour un dataset de 5T tokens : tu dois traiter ~100 TB de données brutes à travers ce pipeline. Ça prend des semaines en GPU/CPU massives. C'est pourquoi les datasets publics et bien documentés (FineWeb, DCLM) sont si précieux.
</Callout>



## Déduplication : 30-50% des données sont dupliquées

### Pourquoi c'est important

Les données web contiennent énormément de contenu dupliqué : articles repris par multiple sites, templates récurrents, boilerplate HTML répété, etc.

Conséquences de la non-déduplication :
- **Mémorisation** : le modèle mémorise le contenu dupliqué au lieu d'apprendre des patterns généralisables
- **Surreprésentation** : certains contenus polluent disproportionnément les gradients
- **Contamination des benchmarks** : les données de test finissent dans les données d'entraînement

### Technique 1 : Déduplication exacte (hashing)

Compute une signature cryptographique (MD5, SHA-256) de chaque document. Les documents identiques ont la même signature.

**Rapide**, mais ne capture que les duplications exactes.

### Technique 2 : Déduplication approximative (fuzzy)

Détecte les documents similaires mais non-identiques.

**MinHash + LSH** : signature Jaccard de chaque document (ensemble de n-grams). Documents avec similarité > 80% sont détectés comme dupliqués.

**Efficacité** : capture les documents très similaires (même article légèrement modifié).

<Callout type="paper" title="Impact mesuré de la déduplication">
Études montrent que la déduplication améliore les performances de 10% en perplexité et gains mesurables sur les benchmarks. L'effet est plus grand que l'ajout du même volume de données nouvelles non-filtrées.
</Callout>

## Filtrage de qualité : classifier-based

### Approche heuristique simple

Des règles basiques éliminent les documents de basse qualité :
- **Longueur minimale** : < 50 mots = rejeté
- **Ratio alphabétique** : trop de symboles/chiffres = suspect
- **Répétition** : trop de n-grams répétés = contenu généré automatiquement
- **Mots courants** : trop de stop words = contenu faible

Rapide, élimine les pires cas, mais laisse beaucoup de contenu médiocre.

### Approche moderne : classificateur neural

Entraîne un modèle binaire pour distinguer le contenu "haut de gamme" du contenu "bas de gamme" :

- **Exemples positifs** : Wikipedia, livres, articles peer-reviewed
- **Exemples négatifs** : web aléatoire, spam, contenu auto-généré

Le classificateur score chaque document de 0 à 1. Tu filtres au seuil (ex: garder seulement > 0.7).

**Utilisé par** : GPT-3, LLaMA, Qwen, la majorité des LLM modernes.

<Callout type="tip" title="Ranking > hard filtering">
Au lieu de filtrer dur (garder/rejeter), attribue un score 0-1 à chaque document et sur-échantillonne les hauts scores. Ça évite de rejeter des documents utiles qui sont juste sous le seuil.
</Callout>

## Filtrage de sécurité

### Détoxification

Réduction (pas élimination complète) des contenus toxiques :

- **Classificateurs de toxicité** : entraînés sur des datasets de toxicité
- **Listes de mots** : filtrage rapide (mais brutal)
- **Stratégie** : viser un taux cible de toxicité (ex: < 0.5%) au lieu de zéro

Un filtrage zéro élimine trop de données légitimes (discussions nuancées sur des sujets sensibles, documentation légale, contenu médical, etc.).

### Informations personnelles (PII)

Suppression ou anonymisation :
- Numéros de téléphone, emails, adresses
- Noms de personnes privées
- Numéros d'identification (SSN, cartes d'identité)

<Callout type="warning" title="Compromis filtrage/qualité">
Un filtrage trop agressif dégrade la qualité. Par exemple, filtrer tous les contenus mentionnant des termes médicaux ou juridiques potentiellement sensibles élimine les connaissances légitimes dans ces domaines. Le filtrage de sécurité doit être calibré, pas excessif.
</Callout>

## Mélange des données : proportionnement des sources

### Sources typiques et proportions

| Source | Proportion | Qualité | Notes |
|--------|-----------|---------|--------|
| Web (Common Crawl) | 60-80% | Variable | Volume massif |
| Code (GitHub) | 5-15% | Haute | Vérifiable |
| Livres | 3-8% | Très haute | Peu accessible |
| Articles scientifiques | 2-5% | Très haute | PubMed, arXiv |
| Wikipedia | 2-5% | Haute | Couvrir varié |
| Conversations | 1-3% | Variable | Reddit, Discord |
| Données multilingues | Variable | Variable | Dépend objectif |

### Optimisation du mélange

Le choix des proportions est critique. Approches :

1. **Heuristique** : basé sur l'expérience et l'intuition
2. **Scaling law-guided** : faire des expériences sur des petits modèles pour optimiser
3. **DoReMi** : algorithme qui optimise les proportions en fonction de la loss sur une validation set diversifiée

<Callout type="info" title="Transparence accrue">
La communauté open-source pousse vers plus de transparence. Les datasets publics comme FineWeb (HuggingFace), DCLM (AI2) documentent exactement leurs pipelines de filtrage, proportions, sources. Ça améliore la reproductibilité et la collaboration.
</Callout>

## Datasets publics de référence

- **The Pile** : 800 GB, 22 sources diversifiées
- **RedPajama** : reproduit l'entraînement de LLaMA
- **FineWeb** (HuggingFace) : 15T tokens de données web filtrées
- **DCLM** (AI2) : dataset curé avec documentation complète

## Résumé

La qualité des données est le multiplicateur principal de performance. Les trois pilliers :

1. **Déduplication** : élimine 30-50% des données dupliquées (gain significatif)
2. **Filtrage de qualité** : classifier-based, élimine les bas de gamme
3. **Mélange optimisé** : proportions des sources

Investir dans un pipeline de données robuste apporte plus de retour qu'augmenter la taille du modèle ou le volume de données brutes. C'est le travail invisible mais crucial pour les LLM de pointe.
