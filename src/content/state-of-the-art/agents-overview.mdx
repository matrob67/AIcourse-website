## Qu'est-ce qu'un agent ?

Un agent IA, c'est un LLM qui **agit** sur le monde. Pas juste un chatbot qui répond — un système qui :
- Utilise des **outils** (appels API, exécution de code, recherche web)
- Maintient une **mémoire** (contexte court et long terme)
- Poursuit des **objectifs** en plusieurs étapes
- Observe les résultats et s'adapte

<Callout type="info" title="La définition minimaliste">
Agent = LLM + capacité d'agir (outils) + boucle feedback (observe → décide → agit → observe). Sans la boucle, c'est juste un modèle.
</Callout>

## Architecture simple

**Le cerveau** : le LLM qui raisonne et décide.

**Les outils** : recherche web, exécution de code, appels API, manipulation de fichiers.

**La mémoire** : court terme (conversation), long terme (base de connaissances).

**La boucle** :
```
Objectif → [Réfléchir] → [Agir] → [Observer] → [Réfléchir] → ... → Résultat
```

1. Objectif arrive
2. Agent réfléchit : quel outil appeler ?
3. Appelle l'outil
4. Observe le résultat
5. Décide : suivant ou terminer ?

## Niveaux d'autonomie

| Niveau | Description | Exemple |
|---|---|---|
| L1 : Chat | Conversation | ChatGPT standard |
| L2 : Tool-use | Utilise des outils sur demande | GPT-4 avec plugins |
| L3 : Autonome | Planifie et exécute seul | Devin, Claude Code |
| L4 : Multi-agent | Agents collaborent | AutoGen, CrewAI |



## Patterns de raisonnement courants

**ReAct** : Reasoning + Acting (le plus populaire). Alterne réflexion et action.


**Plan-and-Execute** : Planifie tout d'abord, puis exécute séquentiellement.

**Réflexion** : Auto-évaluation et correction itérative.

<Callout type="tip" title="Commencez simple">
La plupart des cas d'usage se résolvent avec un agent simple + quelques bons outils. N'ajoutez de la complexité (multi-agents) que si le simple échoue.
</Callout>

## Types d'outils

- **Recherche** : web, RAG, base de données
- **Computation** : calculatrice, exécution de code
- **Communication** : email, Slack
- **Fichiers** : lire, écrire, éditer
- **APIs** : météo, finance, CRM

Un bon outil a :
- Nom descriptif et description claire
- Paramètres bien typés avec descriptions
- Messages d'erreur informatifs
- Output structuré et facile à parser

## Défis réels

**Fiabilité** : chaque étape peut échouer. 10 étapes à 95% de succès chacune = 60% de succès total.

```
0.95^10 = 0.60
```

**Coût** : chaque itération consomme des tokens. 10 étapes × 2000 tokens × 10$/M tokens = 0.20$ par requête — pas gratuit.

**Sécurité** : un agent qui peut exécuter du code ou accéder aux APIs = risques sérieux. Sandboxing et permissions essentiels.

<Callout type="warning" title="Fiabilité composée">
Si chaque étape réussit à 95%, un workflow de 10 étapes échoue 40% du temps. La robustesse de chaque composant est critique. Investissez dans des outils solides, des prompts clairs, et des mécanismes de recovery.
</Callout>
