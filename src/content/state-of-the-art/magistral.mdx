## Mistral entre dans le jeu du raisonnement

o1 a lancé la mode. DeepSeek-R1 l'a démocratisée. Maintenant Mistral, l'européenne, propose **Magistral**.

Fidèle à sa philosophie, Mistral propose quelque chose que ni OpenAI ni DeepSeek n'offrent de manière claire : une chaîne de pensée **visible**.

<Callout type="info" title="Magistral : transparent par design">
Contrairement à o1 (pensée cachée), Magistral **vous montre comment il pense**. Les reasoning tokens sont visibles. Vous pouvez regarder le processus de raisonnement complet et voir où il s'est trompé.
</Callout>

## Architecture et entraînement

### Pipeline de Mistral

1. **Pré-entraînement** : base Mistral Large (déjà très bon)
2. **SFT sur raisonnement** : entraînement supervisé sur chaînes de pensée haute qualité
3. **Reinforcement learning** : optimisation pour correction du raisonnement
4. **Alignement** : RLHF/DPO pour correspond à préférences humaines

### La chaîne de pensée structurée

Magistral produit un format structuré :

```xml
<thinking>
Le problème demande de trouver...

Étape 1 : Identifier les variables
  x = inconnu
  3x + 15 = 42

Étape 2 : Poser l'équation
  3x + 15 = 42

Étape 3 : Résoudre
  3x = 42 - 15 = 27
  x = 27 / 3 = 9

Vérification : 3(9) + 15 = 27 + 15 = 42 ✓
</thinking>

La réponse est x = 9.
```

Clair, structuré, vérifiable.



## Les variantes

### Magistral Small

Optimisé pour **rapidité et efficacité** :
- Taille modeste, déployable en local
- Latence basse
- Bon pour STEM quotidien
- Bon rapport coût/bénéfice

### Magistral Medium

L'équilibre :
- Raisonnement plus approfondi
- Problèmes plus complexes
- Bonne latence via API
- Bon prix

### Magistral Large

Le flagship :
- Meilleure performance sur benchmarks
- Chaînes de raisonnement longues et détaillées
- Comparable aux meilleurs du marché (o1, R1)
- Plus cher

<Callout type="tip" title="Guide : choisir votre Magistral">
**Small** = prototyping et tâches STEM courantes.
**Medium** = production quotidienne, meilleur rapport coût/qualité.
**Large** = problèmes les plus difficiles, budget ample.
</Callout>

## Coin technique : pourquoi la transparence compte

### Avantages du raisonnement visible

Contrairement à o1 (boîte noire des thinking tokens) :

- **Debugging** : vous voyez exactement où ça déraille
- **Confiance** : vous pouvez lire le raisonnement, décider si c'est logique
- **Apprentissage** : utile en éducation, comprendre le processus
- **Audit** : traçabilité complète pour applications régulées

Vous n'êtes pas juste "accepter la réponse", vous pouvez **vérifier la chaîne**.

### Contrôle du raisonnement

API Mistral permet de tweaker :
- Limiter nombre de thinking tokens
- Forcer ou désactiver le mode raisonnement
- Accéder aux tokens de réflexion dans la réponse

Vous avez contrôle granulaire.
