## Le problème des benchmarks automatiques

Les benchmarks (MMLU, HumanEval) testent des choses étroites. Une question, une bonne réponse. Pas de nuance.

Mais dans la vraie vie, les utilisateurs demandent des trucs ouverts. "Explique comment fonctionne la photosynthèse." Dix réponses peuvent être bonnes.

**LMArena** résout ça : votes humains directs.

<Callout type="info" title="Le concept simple">
Vous posez une question. Deux modèles anonymes répondent en parallèle. Vous votez pour le meilleur (ou un tie). Votre vote met à jour leur score ELO. Répétez avec 2 millions de votes, et vous avez un classement robuste.
</Callout>

## Le système ELO expliqué

**ELO** : système de classement créé pour les échecs par Arpad Elo.

Chaque modèle a un score (entre 800-1400). Après un vote :

- Vous gagnez contre un modèle mieux classé ? +50 points.
- Vous gagnez contre un modèle moins bon ? +5 points.
- Les scores s'ajustent jusqu'à l'équilibre.

**Propriétés utiles** :

Une différence de 50 points ELO = ~57% de probabilité de victoire. C'est linéaire et interprétable.

Les scores convergen vers l'équilibre avec assez de votes. 2 millions de votes c'est robuste.

<Callout type="tip" title="Le vote aveugle change tout">
Vous ne savez pas qui vous évaluez. ChatGPT vs Llama vs Claude ? Les identités sont cachées. Ça élimine les biais de marque. Un modèle open-source peut battre une marque établie si les réponses sont meilleures.
</Callout>

## Les classements (état mars 2025)

Le classement change constamment, mais les tendances :

- **Modèles de raisonnement** (o1, o3, Gemini 2.5 Pro) dominent
- **Claude et GPT-4o** sont très proches pour les tâches conversationnelles
- **Modèles open-source** (DeepSeek, LLaMA) progressent vite
- **Gemini 2.5 Pro** très compétitif

LMArena propose aussi des classements spécialisés :

- **Coding** : tâches de programmation
- **Math** : raisonnement mathématique
- **Hard Prompts** : questions difficiles et complexes
- **Instruction Following** : suivre des instructions précises
- **Style Control** : ajustement pour éliminer les biais de style
- **Vision Arena** : modèles multimodaux (image + texte)

## Méthodologie : comment ça marche

**Interface web** : chat.lmsys.org

Vous écrivez ce que vous voulez. Deux modèles anonymes répondent côte à côte. Vous votez : A, B, tie, ou "les deux sont mauvais".

Les identités sont révélées après le vote pour la transparence.

**Contrôle qualité** :

- Détection de spam et votes automatisés
- Diversité des questions (pas de sur-représentation)
- Nombre minimum de votes avant classement
- Intervalles de confiance publiés

<Callout type="paper" title="Corrélation avec les benchmarks : c'est pas parfait">
ELO corrèle modérément avec MMLU/HumanEval mais capture des dimensions supplémentaires : style, nuance, utilité. Un modèle peut exceller sur MMLU (connaissances factuelles) mais être mal classé sur Arena s'il répond de manière sèche ou mal formatée.
</Callout>

## Les biais qui restent

**Longueur** : les utilisateurs préfèrent les réponses longues, même si elles ne sont pas plus correctes. Style Control tente de corriger ça.

**Population** : les utilisateurs d'Arena ne représentent pas tous les cas d'usage. Questions biaisées vers les tâches conversationnelles. Moins de questions enterprise, techniques, spécialisées.

**Manipulation** : tentatives de votes coordonnés, bots. LMSYS a des contre-mesures mais c'est un risque continu.

## Ce que LMArena mesure vraiment

**Qualité conversationnelle perçue** : oui.

**Utilité en production** : pas vraiment. Un modèle peut être excellent en conversation libre mais médiocre en analyse de documents ou raisonnement complexe structuré.

<Callout type="warning" title="Arena c'est pour la conversation">
LMArena est excellent pour évaluer les assistants conversationnels. Mais si vous avez besoin d'un modèle pour de la programmation, de l'analyse scientifique ou des tâches enterprise spécialisées, regardez aussi HumanEval, GPQA, SWE-bench. Le classement Arena seul ne dit qu'une partie de l'histoire.
</Callout>

## Impact industrie

LMArena est devenu **la** référence pour les comparaisons de LLMs. Les entreprises citent leur position Arena dans les communications. Les nouveaux modèles sont jugés sur Arena.

C'est l'évaluation la plus proche de la satisfaction utilisateur réelle.
