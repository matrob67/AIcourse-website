## La révision stupéfiante

Janvier 2022. OpenAI entraîne GPT-3. 175 milliards de paramètres. 300 milliards de tokens.

Mars 2022. DeepMind publie **Chinchilla** (Training Compute-Optimal LLMs).

Verdict : GPT-3 est 4-5x trop gros pour sa quantité de données.

Pour le même budget compute, un modèle de 70B (Chinchilla) avec 1.4 trillion de tokens surpasse GPT-3.

**Ça a choqué l'industrie.**

<Callout type="info" title="Le ratio optimal : 20 tokens par paramètre">
Pour le même budget de compute, si vous doublez la taille du modèle, doublez aussi les données. Ratio optimal : environ 20 tokens par paramètre. Les modèles avant ? Sous-exploités de données.
</Callout>

## La preuve

**Chinchilla** (70B, 1.4T) vs **Gopher** (280B, 300B)

Même budget compute. Chinchilla gagne presque partout.

Un modèle 4x plus petit. Correctement entraîné. Bat un modèle 4x plus gros. Mal entraîné.

```
GPT-3: 175B paramètres, 300B tokens = sous-entraîné
Gopher: 280B paramètres, 300B tokens = très sous-entraîné
Chinchilla: 70B paramètres, 1.4T tokens = optimal
```

## Impact immédiat

Tout change après Chinchilla.

**LLaMA** (Meta, 2023) : 7B-65B paramètres entraînés sur 1-1.4T tokens. Directement inspiré par Chinchilla.

**Mistral** : petits modèles, données massives.

**Gemma** : compacts, bien entraînés.

La course aux **données** démarre.

<Callout type="tip" title="Les données sont devenues aussi stratégiques que le compute">
Avant Chinchilla : maximiser la taille du modèle. Après Chinchilla : maximiser la quantité ET qualité des données. Les entreprises qui avaient des données gagnent.
</Callout>

## Le mur des données

Mais Chinchilla révèle un problème.

Pour un modèle de 1 trillion de paramètres, vous avez besoin de 20 trillion de tokens.

Il n'y a probablement que 15 trillion de tokens de texte de qualité sur internet.

Alors ? Deux solutions :

1. **Sur-entraîner** : LLaMA 3 (8B) sur 15T tokens (ratio 1875x du Chinchilla)
2. **Données synthétiques** : générer des données avec des LLMs

<Callout type="paper" title="Inference-optimal vs Compute-optimal">
Chinchilla optimise le coût d'**entraînement**. Mais dans la production, le coût **d'inférence** domine. Un petit modèle sur-entraîné peut être plus efficace qu'un gros modèle compute-optimal. C'est pourquoi les entreprises font du sur-entraînement.
</Callout>

## Ce que ça change

La course à la taille s'arrête. La course à la donnée commence.

Les équipes pivot vers :
- Pipelines de filtrage web sophistiqués
- Datasets curatés (FineWeb, RedPajama)
- Données synthétiques

**Les données sont devenues aussi critiques que les GPU.**

<Callout type="warning" title="Le paradoxe Chinchilla">
Chinchilla est le paper le plus influent que personne ne suit à la lettre. Il dit : ratio 20:1. Les modèles actuels ? 50:1, 100:1, 1000:1. Pourquoi ? Inference cost. Vous préférez un petit modèle rapide et cheap, correctement entraîné, qu'un gros modèle parfaitement compute-optimal.
</Callout>
