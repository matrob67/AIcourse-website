## Comment une entreprise de jeux vidéo contrôle l'IA mondiale

Il y a 10 ans, NVIDIA faisait les GPU pour les jeux vidéo. Puis les chercheurs ont découvert que les GPU étaient parfaits pour le machine learning.

Aujourd'hui : NVIDIA vaut plus de 3 000 milliards de dollars. Pourquoi ? Elle contrôle l'infrastructure sur laquelle toute l'IA mondiale repose.

<Callout type="info" title="Le monopole de facto">
NVIDIA : 80-90% du marché des GPU pour l'entraînement IA. Pas juste le hardware, mais l'écosystème logiciel CUDA. Migration vers AMD ? Ça casse tout le code. L'effet de verrouillage est total.
</Callout>



## Le hardware : la génération H

**H100** (2023) : la puce phare.

- 80 milliards de transistors
- 80 GB de mémoire HBM3
- Coût : 30 000-40 000 dollars par GPU
- Pénurie : les délais de livraison atteignent 6-12 mois

**H200** : amélioration. HBM3e (141 GB) pour les très gros modèles.

**Blackwell (B100/B200)** (2024) : nouvelle génération.

- 2-4x plus rapide que H100
- Architecture Transformer Engine (support FP4 pour l'inférence)
- NVLink 5e génération (1,8 TB/s)
- Décompression hardware pour RAG

**Infrastructure** : vous n'avez pas juste besoin d'une GPU, vous en besoin 1000s. NVIDIA fournit :

- **NVLink** : connexion ultra-rapide entre GPU (900 GB/s)
- **NVSwitch** : commutateur pour 8+ GPU
- **InfiniBand** : réseau inter-nœud (400 Gb/s)
- **DGX SuperPOD** : cluster clé en main

<Callout type="tip" title="La pénurie et les prix">
La demande de GPU dépasse l'offre. Les prix sur le marché secondaire explosent. Les entreprises commandent des années à l'avance. Google paierait 1 milliard par an en GPU H100. C'est immense.
</Callout>

## L'écosystème logiciel : le vrai lock-in

**CUDA** : la vraie arme.

Lancée en 2006. Toute l'IA repo dessus. PyTorch, TensorFlow, JAX — tous utilisent CUDA.

Des millions de développeurs formés à CUDA. Migrer ? Ça casse le code. C'est un switch coûteux.

### Les bibliothèques

**cuDNN** : réseau de neurones optimisé.

**TensorRT** : optimisation pour la production.

**TensorRT-LLM** : optimisation spécifique aux LLMs. Quantification, batching, paged attention. Les astuces qui font tourner les modèles vite.

**cuBLAS** : algèbre linéaire GPU.

**Triton** : langage de programmation GPU. Plus accessible que CUDA.

### Les frameworks

**NeMo** : fine-tuning et développement de LLMs.

**RAPIDS** : data science accélérée (pandas, scikit-learn sur GPU).

**Isaac** : robotique et simulation.

## NVIDIA crée ses propres modèles

Pas juste fournisseur. Aussi créateur.

**Nemotron** : famille de LLMs. 8B à 340B. Concurrence à Meta et Open.

**GR00T** : modèle fondation pour robots humanoïdes.

**VILA** : vision-langage.

**Edify** : génération d'images et vidéos.

**DGX Cloud** : accès à des clusters GPU pré-configurés.

<Callout type="paper" title="NVIDIA dans la recherche">
NVIDIA contribue aussi à la recherche. Flash Attention a été optimisé sur leurs GPU. Megatron-LM c'est leur framework pour entraîner LLMs à grande échelle. Leurs chercheurs publient.
</Callout>

## La compétition

**AMD** : MI300X rivalise avec H100. 192 GB. Mais ROCm (l'alternative à CUDA) est loin d'être mature. Faible adoption.

**Intel** : Gaudi 3. Rattrapage.

**Google** : TPUs. Excellents pour Google. Pas disponibles ailleurs (sauf Google Cloud).

**Amazon** : Trainium/Inferentia. Custom. Pour AWS clients.

**Startups** : Cerebras (wafer-scale), Groq (inférence ultra-rapide), SambaNova. Intéressants mais non-mainstream.

NVIDIA reste dominant.

<Callout type="warning" title="Les risques géopolitiques">
NVIDIA vend beaucoup à la Chine. Mais les restrictions d'exportation (sanctions US vers la Chine) limitent H100/H200. DeepSeek a utilisé H800 (version dégradée). Ces restrictions pourraient s'étendre. Impact durable possible.
</Callout>
