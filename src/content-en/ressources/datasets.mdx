## Data First, Everything Else Second

Everyone talks about model sizes: 7B parameters, 70B, 405B... But the truth that the industry took a while to accept is that **data matters more than parameters**. A 7B model trained on exceptional data beats a 70B model trained on noise.

That's the lesson from [Chinchilla](https://arxiv.org/abs/2203.15556): the best AIs aren't the biggest, they're the ones with the **best data**. And knowing the datasets means understanding why a model is good (or bad) in a specific domain.

<Callout type="info" title={"The importance of curation"}>
A good dataset isn't simply a large dataset. Data quality (filtering, deduplication, diversity) is often **more important** than raw quantity. The best datasets are carefully curated to eliminate noise, duplicates, and toxic content. It's thankless work but absolutely crucial.
</Callout>

## LLM Pre-training Datasets

This is the fuel for large language models. **Trillions** of tokens scraped from the web, filtered, cleaned, and deduplicated.

### Web Datasets

- **[Common Crawl](https://commoncrawl.org/)**: the largest web crawl (~250 billion pages). The raw source for virtually all pre-training datasets. It's the crude oil of AI: unusable as-is, but indispensable.
- **[FineWeb](https://huggingface.co/datasets/HuggingFaceFW/fineweb)** (Hugging Face): 15T tokens filtered from Common Crawl with sophisticated quality pipelines. The current open-source reference for pre-training. [Technical blog post](https://huggingface.co/spaces/HuggingFaceFW/blogpost-fineweb-v1).
- **[FineWeb-Edu](https://huggingface.co/datasets/HuggingFaceFW/fineweb-edu)**: educational subset of FineWeb, 1.3T high educational quality tokens. Perfect for training knowledge-oriented models.
- **[RedPajama v2](https://github.com/togethercomputer/RedPajama-Data)**: 30T tokens with quality scoring, an open replica of LLaMA data. A Together AI project.
- **[Dolma](https://huggingface.co/datasets/allenai/dolma)** (Allen AI): 3T tokens, a transparent dataset with complete documentation of every curation step. The [Dolma](https://arxiv.org/abs/2402.00159) paper is mandatory reading if you're interested in data curation.
- **[The Pile](https://pile.eleuther.ai/)** (EleutherAI): 825 GB of diverse text (books, code, web, articles). Historically important, it's the dataset that launched the open-source LLM movement. [Paper](https://arxiv.org/abs/2101.00027).
- **[C4](https://huggingface.co/datasets/allenai/c4)** (Google): cleaned Common Crawl, used to train T5. A classic.

### Code Datasets

- **[The Stack v2](https://huggingface.co/datasets/bigcode/the-stack-v2)** (Hugging Face/BigCode): 67.5TB of GitHub source code in 619 languages. The largest open-source code dataset. [Paper](https://arxiv.org/abs/2402.19173).
- **[StarCoderData](https://huggingface.co/datasets/bigcode/starcoderdata)**: curated dataset for training [StarCoder](https://huggingface.co/bigcode/starcoder2-15b).

### Multilingual Datasets

- **[CulturaX](https://huggingface.co/datasets/uonlp/CulturaX)**: 6.3T tokens in 167 languages, cleaned and deduplicated. Essential for multilingual work.
- **[mC4](https://huggingface.co/datasets/mc4)**: multilingual Common Crawl in 101 languages.
- **[OSCAR](https://oscar-project.github.io/documentation/)**: filtered and classified multilingual web corpus, a French project (Inria).

<Callout type="tip" title="Data transparency">
The strong trend is toward transparency. Projects like Dolma (Allen AI) and FineWeb (Hugging Face) document **every step** of their curation pipeline, enabling reproducibility and auditing. Conversely, proprietary models (GPT-4, Claude, Gemini) don't publish their data. This asymmetry is one of the great ongoing debates in AI.
</Callout>

## Fine-tuning and Alignment Datasets

Once the model is pre-trained, you need to teach it to be **useful**. That's where instruction tuning and alignment datasets come in.

### Instruction Tuning

- **[OpenAssistant Conversations (OASST)](https://huggingface.co/datasets/OpenAssistant/oasst2)**: 161,000 human conversation messages in 35 languages. An impressive community project. [Paper](https://arxiv.org/abs/2304.07327).
- **[Dolly 2.0](https://huggingface.co/datasets/databricks/databricks-dolly-15k)** (Databricks): 15,000 instructions written by Databricks employees. Small but high quality.
- **[FLAN Collection](https://github.com/google-research/FLAN)** (Google): mix of 1,800+ NLP tasks formatted as instructions. The [Flan-T5](https://arxiv.org/abs/2210.11416) paper shows the impact of task diversity.
- **[UltraChat](https://huggingface.co/datasets/stingning/ultrachat)**: 1.5M synthetic multi-turn conversations.
- **[ShareGPT](https://huggingface.co/datasets/anon8231489123/ShareGPT_Vicuna_unfiltered)**: real conversations with ChatGPT shared by users. One of the first datasets that made it possible to "clone" ChatGPT's conversational style.

### Alignment (Preferences)

This is where you teach the model to be **helpful** and **harmless**:

- **[HH-RLHF](https://huggingface.co/datasets/Anthropic/hh-rlhf)** (Anthropic): pairs of responses ranked by humans (helpful, harmless). The foundational dataset of RLHF (Reinforcement Learning from Human Feedback). [Paper](https://arxiv.org/abs/2204.05862).
- **[UltraFeedback](https://huggingface.co/datasets/openbmb/UltraFeedback)**: 64K prompts with responses from 17 models, annotated by GPT-4.
- **[Nectar](https://huggingface.co/datasets/berkeley-nest/Nectar)**: 183K conversations with response rankings.

## Vision Datasets

### Classification

- **[ImageNet](https://www.image-net.org/)**: 14M images, 1,000 classes. **The** foundational benchmark for computer vision. The original paper [ImageNet (Deng et al., 2009)](https://arxiv.org/abs/1409.0575) and the ImageNet competition launched the deep learning revolution in 2012.
- **ImageNet-21K**: extended version with 14M images and 21,841 classes.
- **[CIFAR-10/100](https://www.cs.toronto.edu/~kriz/cifar.html)**: 60K small images (32x32), 10 or 100 classes. The "Hello World" of computer vision.

### Object Detection

- **[COCO](https://cocodataset.org/)** (Common Objects in Context): 330K images, 80 categories, segmentation and detection annotations. The standard benchmark.
- **[Open Images](https://storage.googleapis.com/openimages/web/index.html)** (Google): 9M images with detection, segmentation, and relationship annotations.

### Segmentation

- **[ADE20K](https://groups.csail.mit.edu/vision/datasets/ADE20K/)**: 25K images with dense semantic segmentation (150 classes).
- **[SA-1B](https://segment-anything.com/)** (Meta): 11M images with 1.1 billion segmentation masks. This is the dataset for [SAM (Segment Anything)](https://arxiv.org/abs/2304.02643), one of the most influential vision models.

## Multimodal Datasets

### Image-Text

- **[LAION-5B](https://laion.ai/blog/laion-5b/)**: 5.85 billion image-text pairs crawled from the web. **The** dataset that made Stable Diffusion and open-source image generation possible. [Paper](https://arxiv.org/abs/2210.08402).
- **LAION-400M**: filtered subset of 400M pairs.
- **[DataComp](https://www.datacomp.ai/)**: benchmark for multimodal dataset curation. [Paper](https://arxiv.org/abs/2304.14108).
- **[CC3M/CC12M](https://github.com/google-research-datasets/conceptual-captions)**: Conceptual Captions, image-caption pairs from Google.

### Video

- **[WebVid](https://github.com/m-bain/webvid)**: 10M video-text clips from the web.
- **[InternVid](https://github.com/OpenGVLab/InternVideo/tree/main/Data/InternVid)**: 7M videos with detailed descriptions.

## Evaluation Datasets

How do you know if your model is good? With benchmarks. Here are the main ones:

### General NLP

- **[GLUE / SuperGLUE](https://super.gluebenchmark.com/)**: NLP task suites (classification, inference, QA). The classic benchmarks.
- **[MMLU](https://huggingface.co/datasets/cais/mmlu)** (Massive Multitask Language Understanding): 57 domains of academic multiple-choice questions. The reference benchmark for measuring an LLM's general knowledge. [Paper](https://arxiv.org/abs/2009.03300).
- **HellaSwag, WinoGrande, ARC**: reasoning and common sense. Available on [Hugging Face](https://huggingface.co/datasets).

### Code

- **[HumanEval](https://github.com/openai/human-eval)**: 164 Python problems (OpenAI). The standard benchmark for evaluating code generation.
- **[MBPP](https://github.com/google-research/google-research/tree/master/mbpp)**: 974 Python problems (Google).
- **[SWE-bench](https://www.swebench.com/)**: 2,294 real GitHub issues. The most realistic benchmark for evaluating a model's ability to solve real bugs. [Paper](https://arxiv.org/abs/2310.06770).

### Mathematics

- **[GSM8K](https://huggingface.co/datasets/openai/gsm8k)**: 8,792 arithmetic problems. Simple but discriminating.
- **[MATH](https://github.com/hendrycks/math)**: 12,500 math competition problems. Much harder. [Paper](https://arxiv.org/abs/2103.03874).

<Callout type="warning" title="Ethical and legal challenges">
Large web datasets (Common Crawl, LAION) inevitably contain problematic content: toxic text, copyrighted images, personal data. Filtering **never** completely eliminates them. Ongoing lawsuits (artists vs Stability AI, NYT vs OpenAI) could completely redefine the rules for using this data. Keep a very close eye on this.
</Callout>

## Where to Find Datasets

- **[Hugging Face Hub](https://huggingface.co/datasets)**: 200,000+ datasets with built-in viewer. The first place to look.
- **[Kaggle Datasets](https://www.kaggle.com/datasets)**: datasets and competitions
- **[Papers with Code Datasets](https://paperswithcode.com/datasets)**: datasets associated with benchmarks and papers
- **[Google Dataset Search](https://datasetsearch.research.google.com/)**: dataset search engine
- **[Registry of Open Data on AWS](https://registry.opendata.aws/)**: datasets hosted on AWS
