## The Tools Without Which Nothing Gets Done

Everyone talks about GPT, Claude, Gemini... but behind every model, there's a **deep learning framework**. It's the invisible infrastructure. The engine under the hood. And understanding these tools means understanding how AI is built, not just how it's used.

PyTorch reigns over research and industry. JAX is Google DeepMind's secret weapon. Transformers (the Hugging Face library) is the universal API that gives access to everything. Mastering at least one of these three is essential for anyone who wants to go beyond simple API calls.

<Callout type="info" title={"The importance of framework choice"}>
Your framework choice influences your productivity, performance, and access to the ecosystem. PyTorch dominates research and industry, JAX is used by Google for the largest models (Gemini, AlphaFold), and Transformers is the standard interface for using pre-trained models.
</Callout>

## PyTorch

### Overview

Developed by [Meta AI (FAIR)](https://ai.meta.com/), [PyTorch](https://pytorch.org/) is **the** most widely used deep learning framework in the world. Over 90% of AI research papers use PyTorch. It's the de facto standard.

Why? Because coding in PyTorch is **coding in Python**. No black magic, no weird syntax. You write normal Python code, and PyTorch handles automatic differentiation and GPU execution.

- **Dynamic graph**: the computation graph is built on the fly (eager execution), making debugging trivial. You can put `print()` statements everywhere.
- **Pythonic API**: natural Python syntax, easy to learn for anyone who knows Python
- **Autograd**: transparent automatic differentiation. You compute the forward pass, PyTorch calculates the gradients on its own.
- **Rich ecosystem**: [torchvision](https://pytorch.org/vision/), [torchaudio](https://pytorch.org/audio/), torchtext, [Lightning](https://lightning.ai/)

### Key Components

- **torch.Tensor**: fundamental data structure (like numpy, but on GPU). [Documentation](https://pytorch.org/docs/stable/tensors.html)
- **torch.nn**: neural network modules (Linear, Conv2d, Transformer, etc.). [Documentation](https://pytorch.org/docs/stable/nn.html)
- **torch.optim**: optimizers (Adam, SGD, AdamW). [Documentation](https://pytorch.org/docs/stable/optim.html)
- **torch.distributed**: distributed training across multi-GPU and multi-node setups
- **torch.compile**: JIT compilation to speed up code (PyTorch 2.0+). A game-changer.

### PyTorch 2.0+

Version 2.0 was a quiet revolution:
- **[torch.compile](https://pytorch.org/docs/stable/torch.compiler.html)**: compiles PyTorch code into optimized kernels. Often 30-50% free speedup with a single line of code.
- **Scaled Dot Product Attention**: native implementation of [Flash Attention](https://arxiv.org/abs/2205.14135). No need for external libraries anymore.
- Improved distributed training support ([FSDP](https://pytorch.org/docs/stable/fsdp.html))

Official documentation: [pytorch.org/docs](https://pytorch.org/docs/stable/index.html)
GitHub: [pytorch/pytorch](https://github.com/pytorch/pytorch) (85K+ stars)
Tutorials: [pytorch.org/tutorials](https://pytorch.org/tutorials/)

<Callout type="tip" title="PyTorch in production">
Historically criticized for its production performance, PyTorch has improved considerably with torch.compile, [TorchServe](https://pytorch.org/serve/), and [ONNX](https://onnx.ai/) export. The majority of AI production systems now use PyTorch. The argument "TensorFlow for prod, PyTorch for research" is **dead and buried**.
</Callout>

## JAX

### Overview

Developed by Google, [JAX](https://github.com/google/jax) is a high-performance numerical computing framework. If PyTorch is "Python with superpowers," JAX is "numpy on steroids, compiled to the moon."

The approach is radically different from PyTorch: everything is **functional** and **composable**. You write pure functions, and JAX transforms them into ultra-optimized code.

- **Functional transformations**: `jit` (compilation), `grad` (differentiation), `vmap` (vectorization), `pmap` (parallelization)
- **XLA backend**: compilation to optimized kernels for GPU and TPU (Tensor Processing Unit)
- **Numpy-compatible**: API similar to NumPy for easy adoption
- **Immutability**: pure functional approach (no state mutation)

### Key Transformations

JAX's four "superpowers":

- **[jax.jit](https://jax.readthedocs.io/en/latest/jax-101/02-jitting.html)**: Just-In-Time compilation. Transforms your Python function into optimized machine code.
- **[jax.grad](https://jax.readthedocs.io/en/latest/jax-101/04-advanced-autodiff.html)**: automatic differentiation (forward and reverse). More flexible than PyTorch autograd.
- **[jax.vmap](https://jax.readthedocs.io/en/latest/jax-101/03-vectorization.html)**: automatic vectorization. Write your function for a single example, `vmap` makes it batched automatically.
- **[jax.pmap](https://jax.readthedocs.io/en/latest/jax-101/06-parallelism.html)**: parallelization across multiple devices (multi-GPU/TPU). Nearly free horizontal scaling.

### JAX Ecosystem

- **[Flax](https://github.com/google/flax)**: neural network library for JAX (Google). The equivalent of `torch.nn`.
- **[Optax](https://github.com/google-deepmind/optax)**: optimizers for JAX. The equivalent of `torch.optim`.
- **[Equinox](https://github.com/patrick-kidger/equinox)**: PyTorch-like interface for JAX. The best of both worlds.

### JAX vs PyTorch

| Aspect | PyTorch | JAX |
|--------|---------|-----|
| Paradigm | Imperative (eager) | Functional (compiled) |
| Debugging | Easy (print, breakpoint) | More complex (functional code) |
| Performance | Very good (torch.compile) | Excellent (XLA) |
| Multi-GPU/TPU | FSDP, DDP | pmap, native sharding |
| Ecosystem | Massive | Smaller but growing |
| Adoption | Research + industry | Google, HPC research |
| Learning curve | Gentle | Steep (functional paradigm) |

Documentation: [jax.readthedocs.io](https://jax.readthedocs.io/)
GitHub: [google/jax](https://github.com/google/jax) (30K+ stars)

<Callout type="paper" title="Who uses JAX?">
Google DeepMind uses JAX for its largest models ([Gemini](https://deepmind.google/technologies/gemini/), [AlphaFold](https://github.com/google-deepmind/alphafold)). JAX is preferred when computational performance is **critical**, especially for distributed training on TPU. Outside of Google, adoption remains more limited than PyTorch, but projects like [Mistral](https://mistral.ai/) use it too.
</Callout>

## Hugging Face Transformers

### The Universal Library

[Transformers](https://huggingface.co/docs/transformers) by Hugging Face is **the standard access point** for pre-trained models. It's like an App Store for AI models: you search, download, and use.

- **200,000+ models** accessible via a unified API on the [Hugging Face Hub](https://huggingface.co/models)
- Support for PyTorch, TensorFlow, and JAX
- One-line pipeline for common tasks
- Integration with the Model Hub

### Typical Usage

The standard workflow is simple and fast:
1. Search for a model on the [Hub](https://huggingface.co/models)
2. Load it with `AutoModel.from_pretrained()`
3. Use it for inference or fine-tuning

Documentation: [huggingface.co/docs/transformers](https://huggingface.co/docs/transformers)
GitHub: [huggingface/transformers](https://github.com/huggingface/transformers) (135K+ stars)

### Complementary Libraries

The Hugging Face ecosystem goes well beyond Transformers:

- **[PEFT](https://huggingface.co/docs/peft)** (Parameter-Efficient Fine-Tuning): efficient fine-tuning (LoRA, QLoRA). Essential for fine-tuning without 8 GPUs.
- **[TRL](https://huggingface.co/docs/trl)**: Transformer Reinforcement Learning (RLHF, DPO). For alignment.
- **[Accelerate](https://huggingface.co/docs/accelerate)**: simplified distributed training. Multi-GPU by changing 3 lines of code.
- **[Datasets](https://huggingface.co/docs/datasets)**: data loading and processing. Faster and more efficient than pandas for large datasets.
- **[Evaluate](https://huggingface.co/docs/evaluate)**: standardized evaluation metrics.
- **[Diffusers](https://huggingface.co/docs/diffusers)**: diffusion models (Stable Diffusion, Flux, SDXL). The standard framework for image generation.

<Callout type="warning" title="Abstraction vs Understanding">
Transformers makes model usage very easy (a few lines of code), which can mask the underlying complexity. It's great for prototyping, but for research and advanced development, it's **crucial** to understand what's happening under the hood. High-level abstractions aren't enough when you need to debug a training run that's diverging.
</Callout>

## Other Essential Tools

Modern deep learning is a whole ecosystem. Here are the most important complementary tools:

- **[vLLM](https://github.com/vllm-project/vllm)**: optimized inference server for LLMs. PagedAttention for efficient memory usage. The standard for serving LLMs.
- **[TensorRT-LLM](https://github.com/NVIDIA/TensorRT-LLM)**: inference optimization by NVIDIA. Maximum performance on NVIDIA GPUs.
- **[DeepSpeed](https://github.com/microsoft/DeepSpeed)**: distributed training library (Microsoft). ZeRO for training models larger than GPU memory.
- **[Megatron-LM](https://github.com/NVIDIA/Megatron-LM)**: large-scale LLM training framework (NVIDIA). Used to train the largest models.
- **[Weights & Biases (wandb)](https://wandb.ai/)**: experiment tracking and monitoring. Like a dashboard for your training runs.
- **[MLflow](https://mlflow.org/)**: ML lifecycle management. Open-source, self-hosted.
