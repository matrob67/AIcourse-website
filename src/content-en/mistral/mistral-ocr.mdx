## Mistral OCR: Reading Documents Like a Human

You have 10,000 paper invoices to digitize. Classic OCR (Optical Character Recognition) tools (Google Vision, Tesseract) read them and return raw text: characters in bulk, with no structure, no understanding. "Here's some text, figure it out yourself."

But what you actually need is the invoice number, the date, the supplier name, each line item, and the amounts. Mistral OCR doesn't just read: it **understands** the document. It knows that "2024-001-FR" in the top right is an invoice number, not a product code.

### Classic OCR vs Intelligent OCR: Two Different Worlds

**Classic OCR (Tesseract, Google Vision)**:
- What it does: recognize characters in an image and produce raw text
- What it doesn't do: understand what it reads
- The result: a long stream of unstructured text, impossible to use automatically without massive post-processing

**Mistral OCR (vision + LLM)**:
- What it does: recognize characters AND understand the document
- It identifies the document type (invoice, contract, form), its structure (headers, tables, fields), and the meaning of each element
- The result: structured data (JSON, Markdown) directly usable by an application

The difference is fundamental. Classic OCR does character recognition. Mistral OCR does **document comprehension**.

<Callout type="info" title="The Combination That Changes Everything">
Mistral OCR combines two capabilities: vision (reading the image pixels) and generative artificial intelligence (understanding what is read and structuring it). On complex documents, it's up to 10 times more useful than classic OCR, because the result is directly usable without post-processing.
</Callout>



### What Mistral OCR Can Do

**Extraction with preserved structure.** Not just raw text: headings, paragraphs, lists, sections are preserved. The result is in Markdown or JSON, ready to be integrated into a document or database.

**Table comprehension.** Mistral OCR sees a table and understands it: it identifies headers, columns, rows, and cells. It returns valid CSV or structured JSON, not a text mess.

**Form matching.** A form with "Name: ___" and the handwritten answer below? Mistral OCR matches the field with its value. You get structured key-value pairs.

**Handwriting reading.** Not just printed text. Handwritten notes, annotations, signatures: Mistral OCR reads them (with variable accuracy depending on legibility).

**Multi-page continuity.** A table that starts on page 2 and ends on page 3? Mistral OCR recognizes it's a single continuing table and reconstructs it correctly.

### API Integration

Mistral OCR is accessible via La Plateforme, integrated into the multimodal endpoints.

**Usage process**:
1. Send the image or PDF of the document to the API (in base64 or via URL)
2. Specify in the prompt the type of extraction you want (structured text, invoice data, table content, output format)
3. The model returns the extracted data in the requested format (Markdown, JSON)

**Supported input formats**: JPEG, PNG, PDF (multi-page), TIFF, and other common image formats.

**Resolution recommendations**: 150 DPI minimum for standard text documents, 300 DPI for documents with small text or fine details. Higher resolution gives better results but also increases file size and processing time.

### Quality and Accuracy by Document Type

Mistral OCR accuracy varies depending on image quality and document type:

**High-quality printed documents**: very high accuracy (above 98% at the character level), comparable to the best dedicated OCR systems. This is the ideal case.

**Average-quality scanned documents**: good accuracy (above 95%), with appreciable robustness against scan artifacts (noise, slight tilt, stains).

**Handwritten documents**: variable accuracy depending on handwriting legibility, generally between 80% and 95%. Neat handwriting gives good results, doctor's handwriting less so.

**Multilingual documents**: good support for documents mixing multiple languages, thanks to the multilingual capabilities of Mistral models. A form with fields in French, English, and German is handled correctly.

<Callout type="warning" title="Human Verification Remains Necessary">
Despite the spectacular progress of intelligent OCR, human verification remains recommended for critical use cases: accounting, legal, medical. An OCR error on an invoice amount or a medication dosage can have significant consequences. The safest approach is a workflow that combines automated OCR with selective human validation on critical elements.
</Callout>
