## Mistral Vibe: AI That Comes to You in the Terminal

Developers live in the terminal. `cd`, `git`, `npm`, `vim`: that's their natural habitat. Why force them to open a browser, copy-paste code into a web chatbot, then copy the response back into their editor? It's absurd.

Vibe flips the logic: instead of leaving your terminal to go find the AI, the AI comes to your terminal. You type "refactor this file," and Mistral does it. Directly. Without switching windows.

### Why a CLI Tool?

Developers who use Neovim, tmux, and zero GUI exist. There are a lot of them. And they don't want a VS Code extension.

**The name "Vibe"** references the concept of "vibe coding": that development approach where the developer describes what they want in natural language and lets the AI handle the implementation. You describe the intent, the AI writes the code. It's a paradigm shift in how we program.

<Callout type="info" title="CLI-first, But Not CLI-only">
Vibe complements IDE integrations like Codestral in VS Code. The two cover different audiences: Vibe for pure terminal users (Neovim, tmux, headless systems), IDE integrations for those who prefer a graphical environment. Same AI engine, two interfaces.
</Callout>

### Key Features

#### Natural Language File Manipulation

Vibe can create, modify, and organize files following instructions in English or French.

**Creation:** "Create a React component for a login form with email validation and error handling."

**Modification:** "Add error handling to the file utils/api.ts, with clear messages and automatic retry."

**Refactoring:** "Refactor the files in the src/services directory to use TypeScript classes instead of standalone functions."

**Batch operations:** "Rename all .jsx files to .tsx and update imports across the entire project." An operation that would take 30 minutes by hand gets done in 10 seconds.

#### Semantic Code Search

Forget `grep`. Vibe uses Mistral models' semantic understanding to find what you're actually looking for, even if you don't know the exact name.

**By intent:** "Find all functions that make external API calls." Vibe doesn't search for the word "API," it understands the concept of external calls and finds the matching functions, even if they use `fetch`, `axios`, or a custom wrapper.

**By question:** "Where is email validation implemented?" Instead of grepping "email" across 500 files, you get a direct answer.

**By impact:** "Which files would be impacted if I modify the User structure?" Vibe analyzes dependencies and gives you the list of files to check.

#### Project Understanding

Vibe can analyze an entire project and answer architectural questions. This is particularly useful when joining an existing project.

**Architecture:** "Explain the structure of this project and the main patterns used." Vibe walks the file tree, reads key files, and gives you an overview.

**Documentation:** "Generate documentation for this module's public APIs." Instead of writing docs by hand, Vibe infers them from the code.

**Onboarding:** "Walk me through the main files to understand the authentication flow." The equivalent of a senior colleague giving you a code tour, but available at 3 AM.

<Callout type="tip" title="Automatic Context">
Vibe automatically analyzes the project structure (files, folders, configuration, dependencies) to provide contextualized responses. It understands the project's conventions (naming, patterns, style) and follows them when generating or modifying code. No need to explain everything.
</Callout>

#### Native Git Integration

Vibe integrates with Git to simplify version control operations.

**Smart commits:** "Create a commit with a message describing the current changes." Vibe analyzes the diff and generates a descriptive, conventional commit message.

**Branches:** "Create a feature branch for adding the notifications system." It creates the branch, names it correctly according to the project's conventions, and switches to it.

**Review:** "Analyze the changes from the last commit and suggest improvements." An automatic code review before pushing.

**Conflict resolution:** "Help me resolve the merge conflicts in this file." Vibe understands the intent of both branches and proposes an intelligent resolution.

### Technical Architecture

**Local client.** The CLI tool runs on your machine and has direct access to the file system. It reads project files, analyzes directory structure, and can execute shell commands.

**API communication.** Requests are sent to the Mistral API (La Plateforme) for processing by the language models. Project context is injected into prompts to get relevant, specific responses.

**Local execution.** File modifications are made locally. The developer can preview changes before applying them: no surprises, no unwanted modifications.

### Installation and Configuration

**Installation:** available via npm, pip, or as a precompiled binary depending on your platform.

**Configuration:** a Mistral API key is required, configurable via environment variable (`MISTRAL_API_KEY`) or configuration file.

**Customization:** a project configuration file lets you define the preferred model, code conventions, files to ignore, and style preferences.

### Three Interaction Modes

#### Conversational Mode

An interactive chat in the terminal, with project context loaded automatically. You ask questions, Vibe answers taking your code into account. Ideal for exploration and open-ended questions.

#### Command Mode

Direct actions for specific tasks:
- `vibe fix "fix the bug in auth.py"`
- `vibe explain "what does this regex do?"`
- `vibe create "an Express middleware for rate limiting"`

Each command is an atomic action. Fast, precise, no unnecessary conversation.

#### Pipe Mode

Integration into shell pipelines, the strength of a well-designed CLI tool:
- `cat error.log | vibe "explain these errors and suggest fixes"`
- `git diff | vibe "summarize these changes for a commit message"`
- `curl api.example.com/data | vibe "convert this JSON to a TypeScript type"`

Pipe mode transforms Vibe into a composable building block in any shell workflow.

<Callout type="warning" title="Security and Code Confidentiality">
Vibe sends code fragments to the Mistral API for processing. Even though Mistral doesn't use them to train its models, the code transits through their servers. For projects containing sensitive proprietary code, two options: use an on-premise deployment of Mistral models, or configure Vibe to exclude certain files/folders from being sent.
</Callout>

### Advanced Use Cases

**Automating repetitive tasks.** Adding unit tests to an entire module, updating documentation after refactoring, migrating patterns (class components to React hooks): Vibe automates the tedious tasks that take hours by hand.

**Automated code review.** Integrated into a CI/CD (Continuous Integration / Continuous Deployment) pipeline, Vibe analyzes pull requests, identifies potential issues (bugs, style violations, security problems), and suggests fixes. A first level of review before the human review.

**Pair programming solo.** Vibe lets the solo developer benefit from a "partner" who questions architectural choices, suggests alternatives, and identifies potential problems. It's not a replacement for human pair programming, but a valuable complement when working alone.

Mistral Vibe fits into Mistral's vision of providing AI tools integrated at every stage of the development workflow: from the terminal to the IDE, from writing code to putting it in production.
