## Mistral Large 3: The Flagship

**Mistral Large 3** is Mistral's most powerful model. Launched in 2025, it represents the culmination of everything the team has learned since Mistral 7B. The numbers speak for themselves: **675 billion total parameters**, but only **41 billion active** per token thanks to the MoE (Mixture of Experts) architecture.

With a **256K token** context, multimodal capabilities (text + images), and a #2 ranking on LMArena (the most reputable human-based ranking for open models), Mistral Large 3 proves that Europe can compete with the best proprietary models.

### Architecture: The Power of MoE at Scale

| Parameter | Value |
|-----------|-------|
| Total | 675B |
| Active per token | 41B |
| Architecture | MoE (Mixture of Experts) |
| Max context | 256,000 tokens |
| Multimodal | Yes (text + image) |
| Training GPUs | NVIDIA Hopper (H100/H200) |

To understand the 675B/41B ratio, here's an analogy: imagine a university with 675 professors specialized in different fields. When a student asks a question, instead of consulting all 675 professors (which would take forever), the system routes the question to the 41 most relevant professors. The result: the quality of a 675-expert team, for the compute cost of 41.

Compare this with Mixtral 8x7B which had a 3.6:1 ratio (total vs active experts). Mistral Large 3 goes up to 16:1: many more experts, finer routing, more precise specializations.

<Callout type="info" title="The power of MoE at massive scale">
675B total parameters = enormous knowledge capacity, like a giant library. 41B active parameters = reasonable compute cost, because the model only consults a fraction of that library for each question. That's the whole advantage of MoE: the capacity of a gigantic model without the corresponding compute cost.
</Callout>

### Context: 256K Tokens, or How to Swallow Entire Books

**256K tokens** is about 200,000 words, or **2 to 3 complete books** loaded into memory simultaneously. Concretely, this means you can send the model:

- An entire software project (dozens of code files)
- The complete GDPR documentation
- A 500-page annual financial report with its appendices
- Multiple legal contracts at the same time for comparison

All in context, in a single request. No more tedious splitting into chunks.

How is this technically possible? Thanks to a cocktail of techniques: SWA (Sliding Window Attention, inherited from Mistral 7B), hierarchical attention (fine local + sparse global), KV cache compression, and progressive training on increasingly longer sequences.

### Multimodal: Text + Images

Mistral Large 3 isn't limited to text. It **understands images**: photos, charts, diagrams, screenshots. In practice, this opens up concrete use cases:

- **Analyzing a financial chart** and extracting key trends
- **Reading a document scan** (intelligent OCR)
- **Interpreting a software architecture diagram**
- **Solving a math problem** from a photo of the problem statement

The crucial point: text performance remains top-notch. Many multimodal models regress on text when vision is added. Mistral Large 3 avoids this trap.

### Performance: In the Big Leagues

**LMArena**: ranked #2 among open-weight models. LMArena works through blind human judgment: users compare two responses without knowing which model generated them. It's the most reliable benchmark because it measures what truly matters: human preference.

**Academic benchmarks**: on MMLU (general knowledge), HumanEval (code), and MATH (mathematics), Mistral Large 3 is competitive with 2024 proprietary models (GPT-4o, Claude 3.5 Sonnet). In 2026, the competition is against GPT-5.4, Claude Sonnet 4.6, and Gemini 3.1 Pro, but the gap between open and proprietary models continues to close.

### Training: The Brute Force of Hopper GPUs

Trained on NVIDIA H100/H200 GPUs equipped with HBM3e memory (4.8 TB/s bandwidth). To train 675B parameters, you need massive four-level parallelism:

- **Data parallelism**: the same data across multiple GPUs
- **Tensor parallelism**: a single tensor distributed across multiple GPUs
- **Pipeline parallelism**: the model's layers on different GPUs
- **Expert parallelism**: MoE experts distributed across different GPUs

<Callout type="paper" title={"Estimated training cost"}>
Community estimates place the training cost between 50 and 150 million dollars in GPU hours. This is an unofficial figure, but it gives an idea of the investment needed to reach this level of performance.
</Callout>

### Use Cases: Where Large 3 Excels

**Complex documents**: with 256K context and vision, you can analyze a complete financial report with its charts and tables in a single request. No more complex RAG pipeline needed.

**Multi-step reasoning**: synthesizing information from multiple sources, comparative analysis, solving complex problems.

**AI agents**: advanced function calling combined with long context enables building sophisticated agents capable of planning and executing complex tasks.

**Multilingual deployments**: Mistral Large 3 excels in many languages, making it a natural choice for international organizations.

### Availability

- **La Plateforme**: direct API access (`mistral-large-latest`)
- **Le Chat**: consumer conversational interface
- **Clouds**: Azure AI Studio, AWS Bedrock, GCP Model Garden
- **Open-weight**: available on Hugging Face for self-hosting

<Callout type="warning" title="Self-hosted deployment: not for everyone">
675B parameters, even quantized, require a cluster of high-end GPUs (multiple H100s or equivalent). For the vast majority of users, API access via La Plateforme or public clouds is the most pragmatic and economical solution.
</Callout>
