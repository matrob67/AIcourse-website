## La Plateforme: Your Gateway to Mistral

**La Plateforme** is Mistral's API. Launched in February 2024, it lets any developer access all Mistral models through code, with a free tier to get started and per-token pricing to scale.

The key point: the API is **compatible with the OpenAI format**. If you already have code that calls ChatGPT, switching to Mistral comes down to changing two lines (the endpoint URL and the API key). The rest of your code stays the same.

### Available Models

|--------|----------|----------|------------|
| Mistral Small | `mistral-small-latest` | Variable | No |
| Mistral Medium | `mistral-medium-latest` | Variable | No |
| Mistral Large | `mistral-large-latest` | 256K | Yes |
| Codestral | `codestral-latest` | Variable | No |
| Magistral | `magistral-latest` | Variable | No |

All models, one API, one account. The `-latest` endpoints always point to the current version of the model, which saves you from managing versions manually.

<Callout type="info" title="OpenAI Compatibility: Migrate in 2 Minutes">
The Mistral API follows the same format as OpenAI (same message structure, same parameters). Migrating from ChatGPT? Just change the base URL and the API key, that's it. Frameworks like LangChain and LlamaIndex support Mistral natively, so migration is even simpler if you're using them.
</Callout>

### Free Tier: Get Started Without a Credit Card

You can **test for free** all entry-level models without providing a credit card:

- Limits: fewer requests per day than a paid tier
- Access: entry-level models (Small, embeddings)
- No SLA (Service Level Agreement, meaning no guaranteed uptime), no priority support

It's perfect for prototypes, students, personal projects, and exploring the ecosystem.

**Get started in 2 minutes**: go to console.mistral.ai, create an account, generate an API key, fire off your first request. It's that simple.

### Pricing: Pay for What You Use

The business model is simple: you pay per token consumed, with no fixed subscription.

**The pricing logic**:
- Input tokens (your prompt) cost less than output tokens (the generated response). This makes sense: generating text is more expensive than reading it.
- More powerful models cost more per token (Large > Medium > Small)
- Batch processing gets 25 to 50% discounts

Prices evolve, but Mistral is generally competitive compared to OpenAI and Anthropic. And La Plateforme directly is almost always cheaper than going through public clouds (Azure, AWS, GCP) which add their own margin.

### Rate Limits: How Many Requests Per Minute

- **Free**: strict limits (enough for dev, not for production)
- **Paid**: higher limits, adjustable on request
- **Enterprise**: virtually unlimited

The HTTP headers of each response indicate your remaining consumption, which makes it easy to manage on the code side.

### Key API Features

**Chat Completions**: the main endpoint. You send a message history (system, user, assistant), and Mistral generates the continuation of the conversation.

Common parameters you'll use:
- `temperature`: controls creativity (0 = deterministic and factual, 1 = creative and varied)
- `max_tokens`: limits the response length
- `top_p`: nucleus sampling to control diversity
- `tools`: define functions the model can call (function calling)
- `response_format`: force the model to respond in structured JSON

**Streaming**: receive the response token by token via SSE (Server-Sent Events, a real-time server-to-client communication protocol). Essential for real-time user experience: the user sees the response appearing progressively instead of waiting for the entire generation to finish.

**Embeddings**: convert text into vectors for semantic search. The Mistral Embed endpoint produces 1024-dimensional vectors.

**Fine-tuning**: create custom models trained on your own data, directly from the API.



### Official and Community SDKs

- **Python**: `pip install mistralai` (official SDK)
- **JavaScript/TypeScript**: `npm install @mistralai/mistralai` (official SDK)
- **Go, Rust, Java**: community SDKs available

<Callout type="warning" title="Keep Your API Keys Secret">
Golden rule: never put an API key in public code. Not in frontend HTML, not in a GitHub repo, not in a committed config file. Use environment variables. Rotate your keys regularly. A compromised API key can generate thousands of euros in charges within hours.
</Callout>

### Dashboard and Monitoring

The dashboard at console.mistral.ai provides a comprehensive view of your usage:
- **Real-time consumption**: tokens consumed, costs per model, per day
- **Call history**: latency, error rates, volumes
- **Configurable alerts**: get notified if your consumption exceeds a threshold

### Production Best Practices

**Retry with exponential backoff**: if you hit a rate limit (429 error), don't immediately send a new request. Wait 1 second, then 2, then 4, then 8. Exponential backoff prevents flooding the API.

**Smart caching**: if the same question comes up regularly with the same expected answer, cache the result. It reduces costs and latency.

**Appropriate timeout**: configure the timeout based on the use case. A short chat = short timeout (10 seconds). A long generation = more generous timeout (60 seconds).

**Fallback between models**: if Large is too slow or unavailable, automatically switch to Small. Your application's resilience shouldn't depend on a single model.
