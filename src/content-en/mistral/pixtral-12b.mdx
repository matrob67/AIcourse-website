## Pixtral 12B: When Mistral Learns to See

Until Pixtral, Mistral models only understood text. "Explain this article." "Summarize this code." That's fine, but the real world isn't just text.

An invoice is text mixed with a logo, a price table, and sometimes a scanned stamp. A financial report is text interwoven with bar charts. An error screenshot is an image with text inside it. Until now, you needed one tool per modality: an OCR for text in images, a table extractor, and an LLM to understand it all.

Pixtral 12B unifies everything. You give it text, images, or both together, and it understands the whole thing like a human would: in a single glance.

### Why Vision Has Become Essential

It's estimated that 60% of enterprise queries could benefit from a visual component. Traditional OCR manages with clean printed text, but it struggles with complex layouts, charts, or handwritten text.

A human analyst looks at an X-ray (image) while reading the radiologist's report (text). Their brain integrates both naturally. Pixtral does the same: it fuses visual and textual modalities in a single reasoning process.

<Callout type="info" title="Multimodality: a standard, no longer a luxury">
GPT-4V, Claude, Gemini: all major models can now process images. In 2024-2025, vision is no longer a differentiator, it's a prerequisite. With Pixtral, Mistral joins this standard while offering an accessible open-weight model.
</Callout>

### Architecture: How to Teach Vision to an LLM

Pixtral 12B adopts the typical modular architecture of modern vision-language models, composed of three building blocks.

#### The Vision Encoder (the model's "eyes")

The vision encoder is a Vision Transformer (ViT) that slices each image into small tiles (patches) of 14x14 or 16x16 pixels. Each tile is transformed into a "visual token," exactly like a word is transformed into a text token. All the tiles then pass through attention layers so the model understands the spatial relationships between image regions.

A key strength of Pixtral's encoder: it supports variable resolutions. A small icon and a high-resolution A4 document are processed adaptively, without forcing a brutal resize that would lose details.

#### The Projection Module (the "translator")

Imagine the vision encoder speaks "the language of images" and the language model speaks "the language of words." The projection module is the translator between the two: a neural network (MLP) that transforms visual embeddings into embeddings compatible with the LLM's representation space. This is a critical step, because all visual information must pass through this bottleneck.

#### The Language Model (the "brain")

The backbone is the classic Mistral architecture, with all its optimizations (GQA for Grouped-Query Attention, attention optimizations). Visual tokens are inserted into the input sequence alongside text tokens, and the language model processes everything in a unified manner, as if images and text were part of the same "discourse."

<Callout type="paper" title={"The modular vs native approach"}>
The "vision encoder + projection + LLM" architecture was popularized by LLaVA (Large Language and Vision Assistant). Other approaches exist, like Fuyu which processes pixels directly without a separate encoder. But the modular approach remains dominant because it allows reusing pre-trained components: a good existing vision encoder + a good existing LLM = a good VLM, without retraining everything from scratch.
</Callout>

### What Pixtral Can Do

**Understand a scene.** Describe the content of a photo, identify objects, people, actions, and spatial relationships. It understands context and can infer what isn't explicitly visible ("this person seems surprised").

**Analyze visual documents.** This is probably the most useful enterprise use case. Invoices, administrative forms, reports with charts: Pixtral extracts structured information by understanding the visual hierarchy and relationships between elements. No more complex OCR pipeline needed.

**Interpret charts and tables.** Bar charts, line graphs, pie charts, cross-reference tables: Pixtral reads them and can answer quantitative questions. "Which month had the highest revenue?" "What's the trend between 2022 and 2024?"

**Reason visually.** Beyond description, the model compares elements, infers causes, and solves simple visual problems. It's the difference between "describing what you see" and "understanding what it means."

### Text Quality Wasn't Sacrificed

One of the classic pitfalls of multimodal models: when adding vision, text performance degrades. The model "forgets" how to write well because it spent too much time learning to see. This is called catastrophic forgetting.

<Callout type="tip" title={"The challenge of multimodal training"}>
Mistral developed training techniques that preserve text performance while grafting visual capabilities. The result: Pixtral 12B maintains text performance comparable to a text-only model of similar size. You can use the same model for purely textual tasks without any penalty. One model, two modalities, zero compromise.
</Callout>

### Concrete Production Use Cases

**Automated document processing.** Companies receive massive volumes of visual documents daily: invoices, purchase orders, bank statements. Pixtral automates information extraction, classification, and summarization, replacing fragile OCR pipelines with a single API call.

**Visual assistance.** An assistant integrating Pixtral can help a user understand a stock chart, analyze a medical image (with appropriate guardrails), or decipher a technical schematic.

**E-commerce.** Automatic product image analysis, description generation from photos, visual comparison between products: all direct applications for online retail.

### Deployment: Accessible and Flexible

Pixtral 12B is available as open-weight, allowing self-hosted deployment. With only 12 billion parameters, it runs on a single high-end GPU, making it far more accessible than competing multimodal giants.

The model is also available via Mistral's La Plateforme and cloud partners, with an API that accepts images encoded in base64 or via URL.

<Callout type="warning" title="Current limitations of vision">
Like all current vision-language models, Pixtral 12B isn't infallible. It can hallucinate visual content (describe an object that isn't there), misinterpret ambiguous images, or have difficulty with very small text in images. For critical use cases (medical, legal), human validation remains essential.
</Callout>

Pixtral 12B marks an important milestone in the evolution of the Mistral lineup, laying the multimodal foundations that will be extended in subsequent models, notably Mistral Large 3.
